### Modulator Editor 视图

在 Modulator Editor（调制器编辑器）中，可定义所选调制器的特性。在将调制器作用于 RTPC 之前，这些特性将决定调制器的输出。有关如何对调制器使用 RTPC 的详细信息，请参阅[*使用 RTPC*](../../../../04-与游戏互动/05-使用-RTPC/00-使用-RTPC.md "使用 RTPC")。

|  |  |
| --- | --- |
| [备注] | 备注 |
| Modulator 自身的一些属性也可以通过 RTPC，用 Game parameter 或其他 Modulator 进行控制。 |

|  |  |
| --- | --- |
| [备注] | 备注 |
| LFO 和 Envelope 调制器的 RTPC 光标并不能指示其所在时间点的具体值，因为该值是由调制器的内部属性决定的。 |

这些调制器设置也可另存为 ShareSet，也就是说用户可以在工程的多个对象之间共享这些设置。

|  |  |
| --- | --- |
| [注意] | 注意 |
| LFO 和 Envelope（包络）调制器的处理时间取决于其 RTPC 用法。对于大部分属性，将在每个音频控制帧对调制器进行估值。不过，对于 **Voice Volume**（声部音量）属性，将在每个音频采样帧对这些调制器进行估值。请有选择地使用这些调制器，因为它们会占用相当大一部分平台内存和 CPU 资源。 |
