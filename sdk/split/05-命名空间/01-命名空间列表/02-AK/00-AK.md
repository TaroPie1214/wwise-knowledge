## 详细描述

Definition of data structures for [AkAudioObject](struct_ak_audio_object.html)

Utility functions

Audiokinetic namespace

Can be used to track individual parameter changes to avoid costly computations when they remain constant This class is designed to use only the lower bit information of the parameter IDs

Ambisonics configurations (corresponding to [AkChannelConfig::eConfigType](struct_ak_channel_config_a598c4a4489e9fb252e7022ae99135674.html#a598c4a4489e9fb252e7022ae99135674 "Channel config type (AkChannelConfigType).") == AK\_ChannelConfigType\_Ambisonic). Convention: X points towards the front, and XYZ follow a right-hand rule, so Y is the side vector (pointing to the left). Channel presence and ordering are predefined according to the number of channels. The ordering convention is ACN, with the mapping of components to number of channels detailed below (source: <https://en.wikipedia.org/wiki/Ambisonic_data_exchange_formats>). Normalization natively used in [Wwise](namespace_a_k_1_1_wwise.html) is SN3D.

| 1 | 0 | 0 | mono |  |
| 4 | 1 | 1 | first-order full sphere | WYZX |
| 9 | 2 | 2 | second-order full sphere | WYZXVTRSU |
| 16 | 3 | 3 | third-order full sphere | WYZXVTRSUQOMKLNP |
