版本说明 2016.2.1

以下各节列举并描述Wwise版本2016.2.0与版本2016.2.1之间的变化。

# 平台 SDK 的变化

- **PS4 SDK**：已更新至 SDK 4.008.101.
- **Xbox One XDK**：已更新至 XDK 160802 （2016 年 8 月 QFE 2）。

# 漏洞修复

- **WG-30670** 已解决：File Packager 无法打开非常大的 SoundBanksInfo.xml 文件。
- **WG-30913** 已解决：Random Containers 中 Blend Containers 的 SoundBanksInfo.xml 中的 `DurationMax` 值过大。
- **WG-31008** 已解决：当 Blend Container 在播放时，关闭 Game Simulator 会导致 ASSERT（断言）。
- **WG-31100** 已解决：（Xbox One）如果音高改变，在匮乏出现后暂停声音会导致 XMA 断言。
- **WG-31214** 已解决：当更改 State 时，在 Switch 音轨组回放过程中发生断言。
- **WG-31349** 已解决：当原始素材需要缩混时，可能导致 SoundBank 完好性问题。
- **WG-31532** 已解决：在工程载入时，混合音轨会重置所有的参数值范围区域。
- **WG-31537** 已解决：（Xbox One）在回收旧声部时，未知 XMA 声部状态导致 XMA 断言。
- **WG-31617** 已解决：在 `AK::DSP::ApplyGainRamp` 内发生罕见崩溃。
- **WG-31630** 已解决：（Windows）当在 Windows 7 上创建工程时崩溃。
- **WG-31658** 已解决：当没有采集并且勾选了‘follow capture time（跟随捕获时间）’时，光标位置不更新。
- **WG-31662** 已解决：当远程连接时无法创建 RTPC。
- **WG-31675** 已解决：多个选择数值会由于选择对象的顺序不同而改变。
- **WG-31676** 改善了当 WwiseCLI 无法启动时的错误报告。
- **WG-31685** （Android） 在设备上设置最大采样率为 48 kHz 会错误报告成更高的数值。
- **WG-31697** 已解决：无法为 Randomizer 范围输入小数点数。
- **WG-31699** 已解决：使用制表符分割的文件时，Nested Switch Containers 会显示错误的 Switch 指派。
- **WG-31700** （Mac，iOS，tvOS）集成演示：将 Sound Engine 内存池增加到 2 MB 以解决偶尔发生的崩溃。
- **WG-31704** 已解决：单独播放的 Game Object 的 solo 状态在播放完后不可恢复。
- **WG-31711** 已解决：（iOS）如果采样率设为 0，在启动时会发生崩溃。
- **WG-31726** 已解决：在载入某些 PROF 文件后，时间轴的开始部分会出现差距。
- **WG-31761** 已解决：当使用 API Profiler 并且 `StartOutputCapture()` 或 `StartProfilerCapture()` 会被调用时，会发生崩溃。
- **WG-31768** 已解决：当缓冲仍在进行时，`AK::SoundEngine::GetBufferStatusForPinnedEvent()` 和 `AK::IAkStreamMgr::GetBufferStatusForPinnedFile()` 返回的值可能会显示 100。
- **WG-31772** 已解决：当批量添加/移除 Watches 时，Game Object Explorer 视图非常缓慢。
- **WG-31783** 已解决：（Android）如果硬件不支持重采样，一些 Android 4.2 或更老系统的设备会崩溃。
- **WG-31787** 已解决：（Android）Vorbis在 Android x86 上扭曲（ARM 没有问题）。
- **WG-31798** 已解决：AkSpeakerConfig.h 内发生警告。
- **WG-31805** 已解决：当对象在回放时，实时编辑这个对象的输出总线会导致不一致。
- **WG-31840** 已解决：Override Output Bus 复选框代表父对象而不代表 Multi Editor 中的对象。
- **WG-31859** 已解决：N.1 声道配置（如 5.1 和 7.1）的 Convolution Reverb 缩混导致环绕声道和 LFE 无效。
- **WG-31870** （Xbox 360）移除了对 xapid.lib/xapi.lib 的依赖。
- **WG-31892** 已解决：当在 Multi Editor 内双击 RTPC 图标时随机性崩溃。
- **WG-31900** 已解决：工程载入失败，因为当通过 Multi Editor 取消链接时， Work Unit 文件包含无效属性类型。
- **WG-31908** 已解决：Audio File Importer 无法记忆最新设置。
- **WG-31909** 已解决：用制表符分割导入的方式创建 Dialogue Events 有时会导致路径属性设置为垃圾值。
- **WG-31910** 已解决：使用 Audio File Importer 导入 WAV 文件可能会将整个工程添加到 Perforce。
- **WG-31911** 已解决：在带有制表符分割导入的 Dialogue Event 中指派对象两次会导致不必要的警告。
- **WG-31920** 已解决：制表符分割导入不使用 Use Existing 选项，即使该选项可用。

# 其它更改

- **WG-31559** 在 Capture Log 中添加信息，以表示性能分析数据发送失败。
