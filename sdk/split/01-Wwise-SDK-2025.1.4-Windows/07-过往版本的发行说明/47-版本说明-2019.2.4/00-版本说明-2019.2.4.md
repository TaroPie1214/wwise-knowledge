# 平台 SDK 改进

- **Switch SDK**：已更新到 SDK 10.4.1。
- **PS4 SDK**: 已更新至 SDK 7.508.001.

# 新增功能

- **WG-49527** WwiseConsole：针对 SoundBank 生成添加了对 –no-wwise-dat 的支持。

# 行为改进

- **WG-49412** 已修复：(PS4) 在 SceAudio3d Object Panner 插件中，Spread 和 Focus 计算当中所用的距离与发声体和听者之间的实际距离不符。

# 漏洞修复

- **WG-48057** 已修复：在非立体声扬声器设置中，tvOS 将声道发送到了错误的扬声器。
- **WG-48340** 已修复：卸载 FilePackages 会导致线程不安全。
- **WG-49012** 已修复：在开始生成 SoundBank 时，Wwise 设计工具可能会出现卡顿。
- **WG-49094** 已修复：(WAAPI) 在将 [ak.wwise.core.object.create](ak_wwise_core_object_create.html) 和  `onNameConflict` 设为 `rename` 或 `merge` 的情况下创建 Work Unit 会发生错误。
- **WG-49111** 已修复：在 Project Explorer 处于未停靠状态时，Perforce 提交操作所用的键盘快捷方式 (Ctrl+Shift+Alt+C) 无法正常工作。
- **WG-49130** 已修复：在 RTPC 编辑器中更改 X 或 Y 编辑值后在曲线编辑器中单击时，无法更改编辑值所设定的位置。
- **WG-49319** 已修复：在复制整个层级的对象时可能会更改原始对象的媒体 ID。
- **WG-49320** 已修复：在同时共享同一 User-Defined Path 的声音数超过 255 时可能会发生崩溃。
- **WG-49371** 已修复：对于包含无效数据的音频源，在启用 bDebugOutOfRangeCheckEnabled 选项时，会导致系统错误地发送 "Wwise audio out of range: After calling GetBuffer on a source" 消息。
- **WG-49477** 已修复：Spatial Audio 衍射计算的不稳定会导致路径时不时地丢失。
- **WG-49498** 已修复：在 Source 或 Destination 处在嵌套 Work Unit 下时加载工程会移除 Music Transition。
- **WG-49522** 已修复：若 Spread 衰减曲线中出现 S-Curve 和 Inverted S-Curve 线段，则可能会产生偏负的 Spread 值，进而导致感知的声音方向发生反转。
- **WG-49550** 已修复：Wwise 会在保存工程时更新媒体 ID 文件 (<project-name>.mediaid)。
- **WG-49643** 已修复：在使用 MS Spatial 输出插件时，可能会意外取消 COM 初始化。
- **WG-49739** 已修复：在 Blueprint 中使用 AkGeometry 组件时出现位置和声学材质问题。

# 社区报告的漏洞修复

- **WG-49124** 已修复：WwiseCLI 会为 SoundBank 生成提供 HeaderFilePath 参数，而 WwiseConsole 不会。
