版本说明 2017.2.2

以下各节列举并描述 Wwise 版本 2017.2.1 和版本 2017.2.2 之间的变化。有关早期版本的信息，请参阅 [过往版本的发行说明](previousreleases.html) 章节。如果使用联机帮助，您还可以从目录列的顶部选择别的 Wwise 版本。

# 新功能

- **WG-19979** 现在支持在 Android 平台上应用振动效果。
- **WG-36165** 新增了 [Microsoft](namespace_microsoft.html) Spatial Sound Platform 输出插件，现在支持在 Windows 10 和 Xbox One 平台上实施空间音频渲染。

# 行为变化

- **WG-36488** 在内部设置内置参数时，声音引擎现在会忽略辅助发送听者（通过 `SetGameObjectAuxSendValues()` 设置），而只考虑直达听者（通过 `SetListeners()` 设置）。这样就避免了在使用 Spatial Audio 时根据 Room 游戏对象而非玩家听者获取内置 RTPC 值（如距离和声障）。

# 漏洞修复

- **WG-34317** 已修复：(PS4) 现在可选择不对音频包进行 ATRAC9 对齐检查 (`AkPlatformInitSettings.bStrictAtrac9Aligment`)。
- **WG-35934** 已修复：2D 声音的播放优先级受基于距离的优先级偏置影响。
- **WG-35964** 已修复：在播放后 Direct Play 时，Direct Play 没有声音。
- **WG-35981** 已修复：在卸载音频包时，音乐引擎出现罕见崩溃。
- **WG-36216** (Switch) 现在所有控制器模式均支持振动效果。
- **WG-36263** 已修复：(iOS, Mac) 在使用 Xcode 9.1 时会听到咔哒声。
- **WG-36330** 已修复：工具栏显示 Master-Mixer Hierarchy 可能包含的子对象。
- **WG-36348** 已修复：全局 RTPC 无法控制 Switch 类型的 Music Track 子音轨。
- **WG-36400** 已修复：对于包含 State Group 的 Switch Container，会播放指派给 *None* 状态的对象。
- **WG-36450** 已修复：在使用 *All Except* Action 时，动态序列发生死锁。
- **WG-36459** 已修复：在 Mac 上，Recorder 插件一直显示写入访问被拒绝。
- **WG-36486** 已修复：在播放的同一帧出现暂停且过渡导致需要停止时，无法停止已暂停的声音。
- **WG-36500** 已修复：在 Wwise LTX 中，Project Explorer 选项卡显示的 Contextual Help 有误。
- **WG-36553** 已修复：在电脑设为 English 以外的区域设置时，SoundSeed Impact 出现卡顿。
- **WG-36557** 已修复：在单声道声音以无声段落结束时，Vorbis 会读取过去的分配。
- **WG-36563** 已修复：在采用不同声道配置的设备之间切换时发生内存损坏。
- **WG-36641** 已修复：在精确到采样点的转码后播放部分 Vorbis 文件时，出现 *Invalid File Header* 错误。
- **WG-36643** 已修复：未按一致顺序将 State 属性写入 WWU 文件。
- **WG-36655** 已修复：在 RTPC 驱动不包含任何 Effect 的旁通 Effect 时发生崩溃。
- **WG-36691** 已修复：Reflect Effect Editor 出现内存泄漏，导致退出 Wwise 时发生崩溃。
- **WG-36710** 已修复：字符串格式化不再具有区域不变性。
- **WG-36745** 已修复：在降至音量阈值以下时，多点定位声音有时会突然达到极大音量。
- **WG-36759** 已修复：Schematic View 显示的总线定位信息有误。
- **WG-36766** 已修复：假如使用了 Unlink 功能并引用了 ShareSet，在重新加载工程时，混音器插件指派就会丢失。
- **WG-36816** 已修复：Control Surface 全局命令不使用当前选中项。
