# 通用工具

[Microsoft](namespace_microsoft.html) Visual Studio 的以下版本用于构建 Wwise 程序库 ：

- **[Microsoft](namespace_microsoft.html)® Visual Studio 2013:** VS2013 Update 5 (12.0.40629.00)
- **[Microsoft](namespace_microsoft.html)® Visual Studio 2015:** VS2015 Update 3 (14.0.25420.1)
- **[Microsoft](namespace_microsoft.html)® Visual Studio 2017:** VS2017 (15.9.12)

您会注意到，对于 Win32 和 x64，Wwise SDK 安装目录带有后缀“\_vc”+ *版本编号*（例如“\_vc140”或“\_vc150”）：

- **vc120**：包含依赖于 CRT DLL 版本 12.0.40629.00 （对应于 Visual Studio 2013 补丁包 8 版本 12.0.40629.00）的 Wwise Debug、Profile 和 Release 版本的库 。
- **vc140**：包含依赖于 CRT DLL 版本 14.0.25420.1（对应于 Visual Studio 2015 补丁包 2 版本 14.0.25420.1）的 Wwise Debug、Profile 和 Release 版本的库 。
- **vc150**：包含依赖于 CRT DLL 版本 15.0.27004.2010（对应于 Visual Studio 2017 版本 15.0.27004.2010）的 Wwise Debug、Profile 和 Release 版本的库 。

如果您使用 Visual Studio 的不同版本，或者不想让您的游戏依赖于 Visual Studio 2013 Service Pack 2 version 12.0.30501.00 的上述版本，您可以链接到 Debug (StaticCRT)、Profile (StaticCRT) 和 Release (StaticCRT) 文件夹下的库。
