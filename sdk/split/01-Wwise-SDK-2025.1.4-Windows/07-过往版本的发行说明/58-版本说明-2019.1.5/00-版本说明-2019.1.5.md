版本说明 2019.1.5

以下各节列举并描述 Wwise 版本 2019.1.4 和版本 2019.1.5 之间的变化。有关早期版本的信息，请参阅 [过往版本的发行说明](previousreleases.html) 章节。

- [其它更改](whatsnew_2019_1_5.html#misc_2019_1_5)

# 平台 SDK 的变化

- **Xbox One XDK**：已更新至 XDK 10.0.17134.5063 (2018年7月 qfe9)。
- **Switch SDK**：已更新到 SDK 8.3.1。
- **Stadia SDK**：已更新至 SDK 1.37。

# 新功能

- **WG-30060** 无论是主输出所用默认输出设备还是内置输出设备，现在都可利用 AK::GetPS4OutputHandle() 获取 sceAudioOut 句柄。
- **WG-44062** (NX) 新增了平台设置，现在可在插入耳机时自动切换声像摆位规则。
- **WG-44444** Stadia：现在支持语音聊天。

# 其它更改

- **WG-45147** 在 MAC 上添加了 CHM 文档。

# 漏洞修复

- **WG-19780** 已修复：在导入被锁定为只读模式的文件时发生共享冲突。
- **WG-29572** 已修复：在将采样率为 96 kHz 的 IR 文件导入到 Convolution Reverb 中时，生成的混响尾音长度只有预期的一半。
- **WG-42442** 已修复：在达到旁通频率时，Reflect 中的 Diffraction LPF 和 Diffraction HPF 导致产生可辨的咔哒声。
- **WG-42622** 已修复：在对具有混合阶 Ambisonics 声道配置的 Opus 文件进行转码时发生崩溃。
- **WG-42840** 已修复：Properties 字段被裁剪。
- **WG-42929** 已修复：Ambisonics 和 0.1 声道配置不支持 Loudness Meter。
- **WG-44010** 已修复：对于采用 Ambisonics 三阶总线和单声道冲激响应的 Wwise Convolution Reverb，使用了未初始化的内存。
- **WG-44016** 已修复：在远程连接情况下更改 Multi Editor 中的属性时并不总是更新游戏。
- **WG-44059** 已修复：部分视图（如 Voice Monitor、Mixing Desk 和 Music Segment Editor）中没有显示工具提示。
- **WG-44515** 已修复：在打开工程时，即便没有修改工程，有时也会在标题栏中显示星号（指示项目经过修改）。
- **WG-44785** 已修复：在针对 Blend Container 更新 RTPC 值时发生罕见崩溃。
- **WG-44843** 已修复：在 AkInitSettings::audioDeviceShareset 无效时无法改用 System Sink。
- **WG-44878** 已修复：在对 Vorbis 进行解码时，ARM CPU 出现罕见对齐异常。
- **WG-44912** 已修复：在将总线配置为父总线时，即便将父总线或音频设备设为 Ambisonics，3D Meter 选项也不可用。
- **WG-44930** 已修复：包含 Music Playlist 的 Music Switch Container 发生罕见崩溃。
- **WG-44984** 已修复：AK::SoundEngine::GetBufferStatusForPinnedEvent 不再返回大于 100% 的缓冲值。
- **WG-45003** 已修复：(Android) 在较老的设备上，由于本地引用表溢出，导致发生崩溃。
- **WG-45117** 已修复：在播放音乐段落期间暂停走带时，Voice Inspector 触发断言。
- **WG-45121** 已修复：在一并拖放多个 Music Event 提示时有可能将其放在所属段落之外。
- **WG-45133** 已修复：在 Windows 7 上加载包含 Motion 的工程时，有时会导致 Wwise 没有响应。
- **WG-45161** 已修复：在 Voice Inspector 中检视游戏定义的辅助发送时发生崩溃。
- **WG-45238** 已修复：在右键单击树视图控件时发生崩溃。
- **WG-45314** 已修复：在不兼容的父对象下尝试创建子对象时，WAAPI 返回了错误的对象类型名称。
- **WG-45317** 已修复：倘若多个音频包中存在重复的 Zero Latency 媒体，并且没有按照特定顺序卸载这些音频包，就会发生崩溃。
- **WG-45473** 已修复：Pulse Audio 可能会出现缓冲区溢出问题。
- **WG-45565** 已修复：在使用 PrepareEvent API 时内存不足，导致出现罕见内存泄漏。
- **WG-45568** 已修复：在出现 Opus 解码错误时发生罕见崩溃。
- **WG-45577** 已修复：在将 Wwise 连接到 Unity 游戏时可能会发生崩溃。
- **WG-45625** 已修复：(Android) 在有些设备上，默认初始化设置有可能会导致出现毛刺噪声，尤其是在 Wwise Motion 促使手机振动的情况下。**重要说明：**相较于之前的 2019.1 版本，新的默认设置会稍微增加高端设备上的音频延迟。若对新设置不满意，请务必参阅 [audiokinetic.com/android\_fastpath](https://www.audiokinetic.com/library/edge/?source=SDK&id=pg__android__fastpath.html)，来学习如何优化 Android 上的延迟设置。
- **WG-45643** 已修复：(Spatial Audio) 在移除的 Room 与包含对象的 Portal 相连时会发生崩溃。
- **WG-45681** 已修复：在使用流播放声音准备 Event 时过早地触发了 I/O 操作。
- **WG-45726** 已修复：在混合播放 Random Container、Sequence Container 和包含 3D 定位路径的 Blend Container 时触发失败断言。
