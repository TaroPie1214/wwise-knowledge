版本说明 2018.1.5

以下各节列举并描述 Wwise 版本 2018.1.4 和版本 2018.1.5 之间的变化。有关早期版本的信息，请参阅 [过往版本的发行说明](previousreleases.html) 章节。此版本为漏洞修复版本，因此对所有 2018.1.x 版本来说，工程、音频包和 API 都是兼容的，不会有任何变动。

|  |  |

# 平台 SDK 的变化

- **PS4 SDK**：已更新到 SDK 6.008.001。

# 性能变化

- **WG-39355** 针对部分插件（如 Wwise Expander 和 Wwise Compressor），将声音引擎性能提升了 ~7。
- **WG-40076** 大大减轻了执行性能分析时的 CPU 负荷。

# API 变化

- **WG-40062** 暴露了 `AkPlatformInitSettings` : `AkPlatformInitSettings::threadOutputMgr` 中缺失的线程属性，用于一个内部实用工具线程。
- **WG-40806** 将声音引擎线程的默认堆栈大小由 64 KB 增加到了 128 KB。堆栈大小仍可根据需要灵活地自定义。

# 漏洞修复

- **WG-32512** 已修复：在选定项包含无法静音的对象时，有时会无法清除 Mute 和 Solo 状态。
- **WG-40332** 已修复：对于采用匿名声道配置的输出设备，现在支持执行 SetSpeakerAngles。
- **WG-40520** 已修复：在播放期间更改输出声道配置和插件媒体设置时发生崩溃。
- **WG-40551** 已修复：解决了 Event Viewer 中窗口滚动不连贯的问题。
- **WG-40566** 已修复：在没有转码的情况下，假如播放的 WAV 文件超出值域，会出现咔哒噪声。
- **WG-40597** 已修复：在部分包含大量条目的视图（如 List View）中执行全选时，Wwise 挂起。
- **WG-40676** 已修复：MIDI Pitch Root 超级滑杆不会对音高控件的 MIDI 根音产生实际影响。
- **WG-40701** 已修复：(Switch) 在同一程序执行内关闭并重启声音引擎后，Wwise Motion 插件发生崩溃。
- **WG-40708** 已修复：在检查对象时，设计工具出现内存泄漏。
- **WG-40709** 已修复：在 Spatial Audio 中，有些情况下反射声不会绕过边缘发生衍射（因几何构造的配置而异）。
- **WG-40724** 已修复：在 5.1 声道设置下，Wwise Flanger 和 Wwise Tremolo 插件发生罕见崩溃。
- **WG-40736** 已修复：如果加载的性能分析会话中包含的对象已不在工程中，打开 Voice Monitor 会引发崩溃。
- **WG-40786** 已修复：在声音引擎加载初始化音频包之前，将 Wwise 连接到声音引擎时可能会引发崩溃。
- **WG-40835** 已修复：(Xbox One) 在启用多线程和使用精确到采样点的容器时，可能会发生崩溃。
- **WG-40859** 已修复：(Android) SoundSeed Grain 包络应用有误。
- **WG-40878** 已修复：(Linux) 在个别情况下，声音引擎初始化之后不久音频线程偶尔会挂起很长时间，进而导致音频卡顿。
- **WG-40915** 已修复：工程名称中第一个点号之后的字符被误删。
- **WG-40916** 已修复：(Opus) 因为嵌入了序列号，每次转码后的文件都不同。
- **WG-41037** (Android) `AK::SoundEngine::GetFastPathSettings` 现在无论在任何情况下都可以正确地报告最优设备设置。
- **WG-41038** 已修复：在通过 RTPC 变换正负后，SoundSeed Grain 的 Speed 插值出现错误。
