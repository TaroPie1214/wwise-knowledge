版本说明 2015.1.1

以下各节列举并描述 Wwise 版本 2015.1 和版本 2015.1.1 之间的变化

# 平台 SDK 更新

- Android：已更新至 NDK r10e
- Windows：已更新至 Visual Studio 2015 RTM 和 Windows SDK 10.0.10240.0
- Xbox One XDK 已更新至 2015 年 7 月

# 性能变化

- **WG-27809** 删除了未激活的限制器 RTPC 不必要的长时间初始化操作。
- **WG-27845** 如果效果器从未指定 RTPC，则删除了效果器对 RTPC 的搜索。
- **WG-27989** 添加了活跃/非活跃声音的运行时管理，以加速更新有许多映射的游戏参数。

# 漏洞修复

- **WG-27274** 已解决：在 Windows 扬声器禁用的情况下播放音频驱动的振动时崩溃。
- **WG-27662** 已解决：Wwise 工程未验证触发器和对白事件名称的唯一性。
- **WG-27818** 已解决：在 Vita、PS3 或 Xbox 360 上，发布版本无法正确播放后台音乐。
- **WG-27823** 已解决：Nuendo 连接：Mac Wwise Authoring 在禁用/重新启用 GAC 后随机崩溃。
- **WG-27906** 已解决：当 SoundBank 超过最大大小时， SoundBank 生成程序不报告 SoundBank 是否已经更新。
- **WG-27920** 已解决：Android：在旧版手机上初始化失败将导致崩溃。
- **WG-27931** 已解决：浮点舍入错误可能阻止使用锁定到缓存的数据。
- **WG-27936** 已解决：使用 Suspend(true) 有时会导致 Xbox One 上音频不同步。
- **WG-27988** 已解决：AkSinkWasapi 忽略设置变量“uNumRefillsInVoice”，而默认设置为 1。
- **WG-27992** 已解决：Nuendo 连接：从 Nuendo 导出后，Wwise 中发生间歇崩溃。
- **WG-27993** 已解决：对 Xbox One 使用错误的音频类别
- **WG-28009** 已解决：在 File Manager 中按 F1，对于某些效果器，audiokinetic.com 上未显示正确的帮助页面。
- **WG-28010** 已解决：当使用 SCE Audio3D 时，用户定义的定位不独立于在同一对象上播放的声音。
- **WG-28011** 已解决：iOS：当接受 Facetime 电话并使用 AAC 循环播放声音时，将丢失音频。
- **WG-28012** 已解决：Nuendo Connection：连接时未考虑连接参数变化。
- **WG-28017** 已解决：当播放流播放 ADPCM 并用内存用完时将出现断言。
- **WG-28022** 已解决：Nuendo Connection 不支持 UNC 路径作为导出素材共享路径。
- **WG-28033** 已解决：在性能分析期间，当从 InteractiveMusic 演示切换到 Music Sync Callback 演示时，IntegrationDemo 崩溃。
- **WG-28038** 已解决：当单击 Process 时，某些机器上的 Multi-Channel Creator 将死机。
- **WG-28039** 已解决：Multi-Channel Creator 输出预览列表过滤导致配置组合框中发生小故障。
- **WG-28041** 已解决：主总线上的音量升降比预期慢两倍，导致音量不连贯。
- **WG-28056** 已解决：（Wii U）罕见的竞争危害导致在启动时崩溃。
