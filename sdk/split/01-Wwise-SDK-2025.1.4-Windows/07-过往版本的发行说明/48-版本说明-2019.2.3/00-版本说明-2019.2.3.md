# 平台 SDK 改进

- **Stadia SDK**：已更新至 SDK 1.47。

# 新功能

- **WG-46202** 针对 Motion Source 添加了 "Haptics 2 channel" Actuator Configuration。这样可以为支持高保真振动数据的设备生成波形，并模拟使用非对称双电机振动的手柄的行为。
- **WG-49127** 新增了实验性的 WEM Opus 编解码器以提高寻址行为的效率。

# 行为变化

- **WG-49250** 在更改 SCE Audio3d Bed Output 的声道配置时，不再改变 SCE Audio3d Object Panner 所捕获的 Object 的音量。对于采用 Ambisonics 2-2 和 Ambisonics 3-3 声道配置实施混音的工程，可能需要分别将 Object 的音量调节约 +2db 或 +4db 以对此变动加以补偿。
- **WG-49252** 由 SCE Audio 3D Object Panner 以 Object 形式发送的多声道音频不再统一下混到单声道对象。现在每个声道都会作为单独的对象发送到 SceAudio3d。由于定位方式的改变以及下混行为的缺位，多声道对象可能需要调节音量来加以补偿。

# 性能改进

- **WG-48965** 提升了大工程中 Project Explorer 的刷新速度。
- **WG-49160** 大大加快了远程连接的操作速度。

# 其它更改

- **WG-48818** 现在改为分发 vc\_150 和 vc\_160 版的 Wwise 插件 dll。
- **WG-49009** 现在对 rpmalloc 符号进行了修饰，以免与其他库的使用发生冲突。

# 漏洞修复

- **WG-46049** 已修复：在加载工程的过程中无法显示模式窗口。
- **WG-46627** 已修复：在 Wwise 中实时编辑自定义属性时没有更新游戏中的值。
- **WG-47356** 已修复：在打开对象或切换视图时有时会弹出 Notes 字段。
- **WG-47646** 已修复：Capture Log 无法正确滚动到光标位置。
- **WG-48755** 已修复：在 Convolution 插件中使用自定义内存分配器时，可能会发生内存 stomp 问题。
- **WG-48766** 已修复：在 Perforce 服务器的名称中使用协议前缀会从配置中移除服务器设置。
- **WG-48771** 已修复：在第二次更改后没有立刻使用 Perforce 插件设置。
- **WG-48776** 已修复：在播放的声音具有自定义属性时触发断言。
- **WG-48799** 已修复：在游戏正在播放 Switch Container 时将 Wwise 连接到游戏会意外切换到另一 Switch。
- **WG-48822** 已修复：在循环流播放 Opus 声部时播放中止。
- **WG-48855** 已修复：在长时间未连接 Wwise 的情况下运行游戏会触发断言 (AkBehavioralCtx.cpp)。
- **WG-48907** 已修复：对于有些编解码器，SoundBank Editor 中错误地报告了 Decoded Size。
- **WG-48922** 已修复：(Mac Authoring) 在空闲状态占用大量 CPU 资源。
- **WG-48976** 已修复：在上一小版本中意外地移除了 iOS 中断处理程序保护代码。
- **WG-48999** 已修复：在 New Project 对话框中没有默认设为主机的 Documents 目录。
- **WG-49017** 已修复：在内存管理器初始化期间，`AkMemSettings` 中 `pfFalign` 回调的指针为 NULL 但却通过了验证。
- **WG-49023** 已修复：在 Unreal 中，StateGroup ID 可以为 0。
- **WG-49072** 已修复：(WAAPI) 在通过调用 [ak.wwise.core.object.create](ak_wwise_core_object_create.html) 提供注释时无法添加注释。
- **WG-49089** 已修复：在对包含奇数个声道的音频进行重新采样时可能会越界读取数组。
- **WG-49249** 已修复：在将 SCE Audio3d Bed Output 设为主 Sink 的情况下，在 Wwise 中执行某些交互会导致 Wwise 设计工具发生死锁。
- **WG-49264** 已修复：(Stadia) 在 `AkPlatformInitSettings::sampleType` 等于 `AK_INT` 时，5.1 Surround Audio 声道模式可能会发生崩溃。
- **WG-49303** 已修复：对于大于 16 字节的对齐操作，在执行分配时出现内存不足问题，导致 rpaligned\_alloc 发生崩溃。
- **WG-49393** 已修复：对于名称以零为开头并按编号区分的多个 Event，生成的头文件中只添加了一个条目。

# 社区报告的漏洞修复

- **WG-45240** 已修复：在 File Manager 中移除或移动文件时发生崩溃。
- **WG-45734** 已修复：在 Flat 模式下在 File Manager 中搜索源时发生崩溃。
- **WG-48584** 已修复：Loudness Normalization 对 External Source 不起作用。
- **WG-48625** 已修复：在通过将文件拖到对象上来将 .wav 文件导入到 Actor-Mixer 时发生崩溃。
- **WG-48937** 已修复：(Nintendo Switch) 在通过 Target Manager 进行连接时可能会发生崩溃。
