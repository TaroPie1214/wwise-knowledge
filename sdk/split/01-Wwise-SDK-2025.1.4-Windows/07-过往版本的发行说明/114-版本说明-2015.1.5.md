版本说明 2015.1.5

以下各节列举并描述 Wwise 版本 2015.1.4 和版本 2015.1.5 之间的变化。

# 平台 SDK 的变化

- Mac/iOS/tvOS：已更新至 Xcode 7.2 (Apple LLVM 7.0.2)
- tvOS：已更新至 tvOS 9.1
- Xbox One：已将 XDK 更新至 2015 年 11 月
- Xbox One：已移除对 ADK 的支持

# 新功能

- **WG-28856** 添加了 `-AuxBus` 关键词，从而使 SoundBank 定义文件支持在 SoundBank 中加入辅助总线。
- **WG-28593** 添加了将音频连通到 Xbox One 控制器耳机的选项。
- **WG-28942** 将 Wwise Meter 的 Attack Time 从 1 秒增加到 10 秒。
- **WG-29069** 添加了在 Windows 下选择主音频设备的可能性。请参阅 AkPlatformInitSettings.idAudioDevice 文档了解有关如何操作的信息。

# 性能变化

- **WG-28632** Auro Headphone 优化后现在比先前版本快了最多 1.6 倍。

# 其它更改

- **WG-28632** 为 PS4 和 Xbox One 添加了 Auro Headphone 和 Panner 插件。
- **WG-28946** 更新了 Impulse Response License 协议。

# 漏洞修复

- **WG-28472** 已解决：拔出耳机时 XAudio2 中崩溃。
- **WG-28589** 已解决：（Mac）在 IntegrationDemo 中检测到内存泄漏。
- **WG-28628** 已解决：Xbox 控制器与最新的 XInput DLL 不兼容。
- **WG-28637** 已解决：（iOS）内存泄漏（延迟的I/O线程）。
- **WG-28641** 已解决：Wwise 在命令行操作期间调用版本控制 GetFileStatus。
- **WG-28661** 已解决：有时候忽略 MIDI CC 消息。
- **WG-28662** 已解决：在设计工具中编辑时，由游戏参数驱动的切换开关不该不沿用。
- **WG-28672** 已解决：如果在连接游戏时应用RTPC，则勿按 Bypass effect 复选框值。
- **WG-28682** 已解决：（tvOS）当使用“Dolby Surround”选项初始化声音引擎时检测到 0 个声道。
- **WG-28732** 已解决：在查看游戏对象时，Attenuation Editor崩溃。
- **WG-28746** 已解决：在禁用听者数据时，Profiler崩溃。
- **WG-28750** 已解决：卷积混响：XML持续性不准确导致在工程重新加载时转码结果文件散列改变。
- **WG-28754** [已解决：在调用AK::Comm::Term()](namespace_a_k_1_1_comm_a94e307d2974b89cc4fbc8ab0fef20d2f.html#a94e307d2974b89cc4fbc8ab0fef20d2f)时发生罕见的崩溃。
- **WG-28755** 已解决：（PS4）在ATRAC9解码时，内存不足导致崩溃。
- **WG-28800** 已解决：在不完整的切换容器上准备数据时可能发生崩溃。
- **WG-28849** 已解决：File Packager 源缺少*Migrate6to7.cs*文件。
- **WG-28861** 已解决：无法检测到振动总线，也无法重命名。
- **WG-28873** [已解决：添加了用于获取通信错误的函数（AK::Comm::GetLastError()](namespace_a_k_1_1_comm_ab257079d30e10185e37ca7e146b386b2.html#ab257079d30e10185e37ca7e146b386b2)）。
- **WG-28896** 已解决：可选择预读媒体来播放非流播放声音，即使同一声音的所有媒体也可用，具体取决于加载的顺序。
- **WG-28912** 已解决：（PS4）在性能分析和断开网线时发生断言。
- **WG-28919** 已解决：在Suspend()后重新初始化声音引擎时发生崩溃。
- **WG-28923** 已解决：在调用WwiseCLI时，`CustomGlobalOpeningCmd` 和`CustomGlobalClosingCmd` 选项不起作用。
- **WG-28970** 已解决：（PS4）在ATRAC9解码期间，当从一个状态过渡到另一个状态时发生断言。
- **WG-28994** 已解决：当SoundBank线程被禁用（bUseSoundBankMgrThread = false）时，UnloadBank()/ClearBanks() 调用立即返回结果，即使媒体仍在使用中；这造成一种竞争状况。
- **WG-28996** 已解决：切换开关声轨过渡段截断淡入。
- **WG-29010** 已解决：（PS4）当ATRAC9流在循环点出现匮乏时可能发生崩溃。
- **WG-29048** 已解决：File Packager没有生成RSX文件包。
- **WG-29065** 已解决：无法修改 SoundBank 创建日志消息ID 10的严重性。
- **WG-29070** 已解决：（Xbox One）在XMA文件中搜索有时候可能因为硬件不响应而导致死机。
- **WG-29078** 已解决：在AkResampler.cpp中除数为0，导致极端音高值。
- **WG-29085** 已解决：File Packager中有时发生崩溃。
