版本说明 2019.1.8

以下各节列举并阐述了 2019.1.7 和 2019.1.8 版本之间针对 Wwise 所作的改进。有关早期版本的信息，请参阅 [过往版本的发行说明](previousreleases.html) 章节。

# 平台 SDK 的变化

- **Stadia SDK**：已更新至 SDK 1.43。
- **PS4 SDK**：已更新至 SDK 7.008.031。
- **Xbox One XDK**：已更新至 XDK 10.0.17134.5071（2018 年 7 月 qfe12）。
- **Switch SDK**：已更新至 SDK 9.4.1。

# API 变化

- **WG-47804** 2019.1.8 插件构建工具 (wp.py) 现在与 2019.2 版本同步。

# 漏洞修复

- **WG-30250** 已修复：CAkModulator::\_Trigger 发生罕见崩溃。
- **WG-36388** 已修复：现在清除了与蓝牙相关的 Android 代码中的 Java 异常。
- **WG-41384** 已修复：若在 State 变化过程中存在活跃的过渡，则卸载 SoundBank 可能会引发崩溃。
- **WG-44503** 已修复：在 macOS 上，Wwise 设计工具中没有正确显示有些 Notes 字段。
- **WG-46172** 已修复：在将多个选中的对象从 Sequence Container 拖放到 Playlist 后，对象的顺序发生了变化。
- **WG-46899** 已修复：工作线程池关联导致 [Microsoft](namespace_microsoft.html) Spatial Sound Platform 音频设备出现毛刺噪声。
- **WG-46985** 已修复：在 Opus 文件中执行寻址操作时有可能会出现卡顿。
- **WG-47113** 已修复：在启用 Voice Inspector Data 的情况下，在 Wwise 设计工具中执行性能分析时发生罕见崩溃。
- **WG-47141** 已修复：在使用 State 修改插件属性时发生罕见崩溃。
- **WG-47142** 已修复：在使用 3D Bus 并卸载全部 SoundBank 时发生罕见崩溃。
- **WG-47181** 已修复：(Stadia) 在使用动态库时，有些插件无法正常工作。
- **WG-47185** 已修复：在内存不足的情况下执行 RTPC 初始化可能会发生崩溃。
- **WG-47343** 已修复：(iOS) 在将应用程序切换回前台时，音频会话类别 AkAudioSessionCategorySoloAmbient 没有停止外部音乐。
- **WG-47391** 已修复：Capture Log 中没有报告在连接设计工具之前发生的关键错误。
- **WG-47484** 已修复：在播放包含无效 Music Transition 的 Music Container 时，Wwise 发生罕见崩溃。
- **WG-47534** 已修复：在内存不足的情况下播放外部音频源会发生内存泄漏。
