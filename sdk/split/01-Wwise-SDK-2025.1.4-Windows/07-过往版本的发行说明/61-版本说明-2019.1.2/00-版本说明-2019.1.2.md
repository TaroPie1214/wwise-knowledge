# 平台 SDK 的变化

- **Windows**：已将 Visual Studio 2017 更新至 15.9.12 版本。
- **Xbox One XDK**：已更新至 XDK 10.0.17134.5046 (2018年7月 qfe4)。
- **UWP\_vc150**：现在使用 Windows SDK 10.0.17763.0，同时增添了对 ARM64 的支持。
- **Magic Leap**：已将 SDK 更新至 0.21.0 版本。
- **Switch SDK**：已更新到 8.2.0。

# 新功能

- **WG-40496** WAAPI：在主机上运行多个 Sound Engine 实例时，现在可使用 ak.wwise.core.remote.connect 来指定要连接到哪个实例。
- **WG-42896** 现在 Android、iOS 和 MAC 上均可使用 iZotope 插件。
- **WG-43219** 插件定义文件中现在允许使用 Any 平台。

# 性能变化

- **WG-43686** 修复了连接到具有庞大的总线层级结构的游戏时 Master Mixer View 中存在的性能问题。

# API 变化

- **WG-42861** 从声音引擎和相关文档中移除了所有针对 AK::SoundEngine::AddSecondaryOutput() 的引用。

# 其它更改

- **WG-41746** 在 Voice Inspector View 的声部列表中添加了声部和音乐轨图标。

# 漏洞修复

- **WG-40466** 已修复：(Switch) 在设备断开时，Wwise Motion 插件发生崩溃。
- **WG-40897** 已修复：无法在 Multi Editor 中按照 Value 列进行排序。
- **WG-41506** 已修复：在 Project Explorer 中拖动矩形选择区域时无法停止自动滚动。
- **WG-41796** 已修复：在某些情况下，Opus 错误地将 1.1 声道音频源转码为立体声。
- **WG-42339** 已修复：在播放过程中通过 RTPC 禁用声音衰减时，衰减音量保持为当前值，而没有重置为 0dB。
- **WG-42340** 已修复：对于包含 CrankCaseAudio REV 数据的 Soundbank，生成的音频包二进制文件有时会不同。  
  已修复：提升了整体的稳定性。
- **WG-42436** 已修复：假如忘记注销全局回调，在反复重新初始化声音引擎时可能会出现错误。
- **WG-42488** 已修复：(UWP) 在移除 Motion 输出后，手柄振动仍会持续几秒。
- **WG-42510** 已修复：AkMidiCtx.cpp 中偶尔会出现 Debug Assert。
- **WG-42525** 已修复：(macOS) 在另一应用程序获取蓝牙音频输入时出现音频丢失。
- **WG-42769** 已修复：Voice Inspector 针对同一辅助总线对象报告了两个不同的辅助发送听者。
- **WG-42992** 已修复：(Magic Leap) Wwise 错误地将 Opus 编解码器和 Granular Synth 插件报告为不可用。
- **WG-43025** 已修复：假如未将输出设备配置为 48 kHz，Microsoft Spatial Sound Platform 输出插件会导致音频发出噼啪杂声。
- **WG-43064** 已修复：在实施 AAC 转码时，Wwise 卡死。
- **WG-43083** 已修复：对于设为 Continuous 的 Blend Container，假如所含声音活跃于同一 Blend Track 的多个区域，在区域之间也会将声音视为处于活跃状态。
- **WG-43113** 已修复：在加载包含无效的 Music Switch Container 或 Dialogue Event 参数的工程时发生崩溃。
- **WG-43146** 已修复：假如没有在正常速率下触发音频线程，Microsoft Spatial Sound Platform 输出插件会导致音频发出噼啪杂声。  
  已修复：来自 [Microsoft](namespace_microsoft.html) Spatial Sound Platform 输出插件的音频没有发送到正确的输出设备。
- **WG-43149** 已修复：在播放启用了触发速率和 3D 路径的 Random Container 时连接游戏会发生崩溃。
- **WG-43172** 已修复：在通过 `AK::SpatialAudio::RegisterListener()` 更改 Spatial Audio 中的听者后，没有更新现有房间并发送给新的听者。
- **WG-43230** 已修复：(PS4) SCE Audio 3D Object Panner 插件发生死锁。
- **WG-43238** 已修复：自定义衰减缺少所有者（Attenuation Editor 中的 Used By 字段）。
- **WG-43265** 已修复：在 Mac 上，Import 对话框会限制导入的文件数量。
- **WG-43271** 已修复：Crankcase REV 中出现除数为零的情况时会导致无限循环。
- **WG-43315** 已修复：在分区显示 Advanced Profiler 的情况下，更改光标位置后有时不会更新右侧或底部面板。
- **WG-43336** 已修复：在生成音频包时发生罕见崩溃。
- **WG-43350** 已修复：在连接 Wwise 后会在声音引擎中将已经渲染的效果器实例化。
- **WG-43368** 已修复：在引擎中每个缓冲区的采样点数不是 2 的幂时，Izotope Trash Box 效果器发生崩溃。
- **WG-43404** 已修复：在同时运行多个实例时，WwiseCLI 的性能可能会严重下降（启动时间延长）。
- **WG-43412** 已修复：假如有 State Group 在控制 Bypass Effect、Bypass All 或其中一项 Effect 属性，将不会渲染效果器。
- **WG-43426** 已修复：在打开 Multi-Editor 的情况下收起 Contents Editor 内选中的条目时会花费很长时间。
- **WG-43438** 已修复：在保存到 WWISEROOT% 以外的驱动器时，无法构建使用 wp.py 创建的插件。
- **WG-43440** 已修复：Interactive Music Switch Container 中 Path 的 None 部分被错误地理解为通配符 (\*)。重要说明：若要解决此问题，必须重新生成 SoundBank。
- **WG-43459** 已修复：在编辑包含 Post Event 动作的 Event 时，有时会发生崩溃。
- **WG-43480** 已修复：在连接具有庞大的总线层级结构的游戏时出现严重的延迟。
- **WG-43496** 已修复：在添加或移除输出设备时触发断言。
- **WG-43528** 已修复：没有针对效果器和源插件显示帮助信息。
- **WG-43545** 已修复：在使用不带 Motion 插件的 Wwise 加载包含 Motion 的 2019.1 工程时发生崩溃。
- **WG-43550** 已修复：在 State Group 指派设置发生变动时，部分 UI 元素没有更新。
- **WG-43555** 已修复：声音引擎出现罕见内存泄漏并触发断言。
- **WG-43586** 已修复：在 Wwise 之外执行文件操作时，有时不显示工程变更通知。
- **WG-43600** 已修复：在播放时无法将 State 应用到 Enable Attenuation。
- **WG-43683** 已修复：在播放 MIDI 音乐片段时触发罕见调试断言。
- **WG-43814** 已修复：在连接 Wwise 时错误地将游戏中的静态值应用于 Exclusive RTPC。
- **WG-43956** 已修复：在通过复制另一平台的设置添加新的平台时多次触发断言。
- **WG-43522** (Linux) 现在将 PulseAudio 设成了 Linux 上的默认 System Sink。
