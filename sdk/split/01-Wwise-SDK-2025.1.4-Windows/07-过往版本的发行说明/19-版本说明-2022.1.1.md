# 新增功能

- **WG-62553** 在 Project Explorer 中，现在会同时为 Pinned 视图保存滚动位置和对象的展开/折叠状态。

# API 改进

- **WG-62173** 在以下情形将原来的 –header-file-path 选项替换为了新的 –root-output-path 选项：
  - Waapi 操作 ak.wwise.cli.generateSoundbank
  - WwiseConsole 操作 generate-soundbank

# 行为改进

- **WG-62043** (Spatial Audio) 对不在 Room 之间正中位置的 Portal 的行为做了改进。

# 性能改进

- **WG-61463** 针对具有 Pause、Resume、Stop 和 Break 动作且其指向特定游戏对象或总线的 Event 做了处理性能上的提升。

# 漏洞修复

- **WG-54462** 已修复：在使用轮询频率较高的指针设备时，Wwise 中的窗口拖动操作出现明显的延迟。
- **WG-57142** 已修复：在使用有些字体大小设置时，Sequence Container 的播放列表中的元素无法正确地水平对齐。
- **WG-57552** 已修复：在 Reflect Effect Editor 中对镜像声源进行排序会导致坐标图中的光标标志消失。
- **WG-60993** 已修复：在 Control Surface Session 处于 Learning 模式的情况下选中当前对象时，Wwise 中可能会触发断言。
- **WG-61593** 已修复： [ak.wwise.core.audioSourcePeaks.getMinMaxPeaksInRegion](ak_wwise_core_audiosourcepeaks_getminmaxpeaksinregion.html) 忽略音频源上应用的声道覆盖设置。
- **WG-62060** 已修复：(Spatial Audio) 在移动与被禁用三角形几何构造关联的 Room 时没有更新 Room Extent。
- **WG-62152** 已修复：在单击 Help 按钮时，错误报告工具发生崩溃。
- **WG-62322** 已修复：(Spatial Audio) 在 Room 之间过渡时保留了不符合预期的源自 Portal 的衍射路径。
- **WG-62372** 已修复：(Spatial Audio) 在发声体穿过 Portal 时，源自发声体的路径有时会丢失。
- **WG-62378** 已修复：(Spatial Audio) 在从听者的角度来看第一个边缘被部分遮蔽的情况下，衍射路径有时会丢失。
- **WG-62387** 已修复：(Spatial Audio) 在听者处于最大距离之外静止不动的情况下移动 Distance Probe 时没有创建衍射路径。
- **WG-62388** 已修复：(Spatial Audio) 无法正确更新之前与 AkGeometry 关联的 Room。
- **WG-62399** 已修复：(Spatial Audio) 在听者和发声体处在同一 Room 且听者靠近偏离中心的 Portal 时，直达路径在听者后方错误地发生弯曲。
- **WG-62414** 已修复：(Spatial Audio) 在从 Room 移除所有 Portal 时，Game Object 3D Viewer 中可能会出现落单路径。
- **WG-62497** 已修复：在为 HFDamping 调用 `SetRTPCValue()` 时发出关于浮点值无效的警告。
- **WG-62526** 已修复：在播放音频的情况下将应用程序最小化/挂起时产生细微的毛刺噪声。
- **WG-62571** 已修复：在 Project Explorer 视图内更改当前 Selection Channel 时，有时无法正确恢复对象的展开/折叠状态。
- **WG-62584** 已修复：Profiler 中误报错误："Duplicated media has different length in bank."。
- **WG-62633** 已修复：在 Server build 上默认禁用了 Wwise Addressables。
- **WG-62699** 已修复：(Spatial Audio) 在使用具有复杂边缘拓扑结构的几何构造时发生崩溃。
- **WG-62763** 已修复：在将对象复制到 Sequence Container 的播放列表中时，Wwise 发生崩溃。
- **WG-62768** 已修复：将 Interactive Music 对象添加到了 Sequence Container 的播放列表。
- **WG-62794** 已修复：(Spatial Audio) 在已经有衍射路径时生成了不符合预期的源自 Portal 的直达路径。
- **WG-62836** 已修复：在使用 Paste Properties 将对象列表复制到一个或多个目标对象时可能会发生崩溃。
- **WG-62878** 已修复：(Spatial Audio) 在听者靠近 Portal 时，透射路径可能会发生弯曲。
- **WG-62886** 已修复：(Spatial Audio) 在听者或发声体处在 Portal 的边角且开口之间存在很大间隙时，衍射路径可能会丢失。
- **WG-62959** 已修复：在查看 HDR Bus 上的 Voice 时，有时不更新 Voice Inspector 中的影响因素视图。
- **WG-63113** 已修复：在删除 Mixing Desk 会话中使用的对象时可能会发生崩溃。
- **WG-63116** 已修复：(Spatial Audio) 在更新与 Portal 交叠的 Geometry Set 时发生死锁。

# 社区报告的漏洞修复

- **WG-54673** 已修复：在发声体为 3D 总线的情况下以编程方式添加发声体与听者关系时只会对通过所述总线发送的新的声音产生影响。
- **WG-59739** 已修复：若没有为声部的 Auxiliary Send 指定音量衰减曲线，但设置了声笼曲线或透射曲线，并将 3D Position 设为了 **Listener with Automation** 或 **Hold Emitter Position and Orientation**，会触发断言并将声部的 Auxiliary Send Volume 设为零。
- **WG-60853** 已修复：(Spatial Audio) 在音量较小的路径短于音量较大的路径且其应用了 Portal 声障时可能会选择前者而舍弃后者。
- **WG-61178** 已修复：在添加效果器后马上按下 Enter 键时，Wwise 可能会发生崩溃。
- **WG-61686** 已修复：External Editor 不适用于 Segment Editor 中的 Music Clip 和 Music MIDI Clip。
- **WG-61687** 已修复：WAQL originalFilePath 不适用于 MusicClip 和 MusicMidiClip。
- **WG-61709** 已修复：在加载 RTPC 中存在未处理的控制输入的工程时发生崩溃。
- **WG-61812** 已修复：(Spatial Audio) 在 Listener 和 Emitter 被打开的 Portal 和其他几何构造分隔时总是应用 Room 透射损失。
- **WG-61981** 已修复：在关闭声音引擎时，CAkTransitionManager::TermList 内出现内存 stomp 问题，进而导致发生崩溃。
- **WG-62011** 已修复：在 File Manager 中重命名文件夹时可能会发生崩溃。
- **WG-62057** 已修复：在从头开始重新播放 Vorbis 文件时显示错误的 Seek Table Absence 警告。
- **WG-62150** 已修复：在 RTPC 曲线上使用 S-Curve Interpolation 时，有时会提供指定范围以外的值。比如，曲线可能会生成负的 Spread 或 LPF/HPF 因数值，而非在 0 处截止。
- **WG-62177** 已修复：在 3D Audio 可用但 Passthrough 不可用的情况下将总线配置设为 **Same as passthrough mix** 时会将总线配置设为 Audio Objects 而非 Audio Device 的 Main Mix。
- **WG-62358** 已修复：在有些情况下，源自 `CAkVPL3dMixable::GetVolumes` 的堆栈溢出导致发生崩溃。
- **WG-62540** 已修复：在播放由设置了 Delay 和 Sound Instance Limit 的 Random Container 触发并设为 Continuous 的 Blend Container 时，Wwise 可能会发生崩溃。
- **WG-62672** 已修复：在累计 Voice Inspector 数据时，声音引擎出现内存重新分配错误。
- **WG-62703** 已修复：在生成包含某些 RTPC 的 SoundBank 时发生崩溃。
- **WG-62780** 已修复：在 Android 上，SetBasePath 为 OBB 文件中的所有位置返回 Ak\_PathNotFound。
- **WG-62791** 已修复：在打开较大的工程时，Perforce Submit 对话框会卡顿很长时间。
- **WG-62952** 已修复：在加载较大的工程时，在 Loading Layouts 阶段出现卡顿。
