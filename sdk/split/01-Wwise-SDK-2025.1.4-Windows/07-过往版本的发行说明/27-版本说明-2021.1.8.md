# 漏洞修复

- **WG-53467** 已修复：WwiseConsole 在 macOS 上消耗 100% 的 CPU 资源。
- **WG-54064** 已修复：在内存不足的情况下加载 SoundBank 时无法注册回调。
- **WG-58730** 已修复：Remote Connections 对话框并不总是列出本地设备上的所有可用实例。
- **WG-58825** 已修复：在过渡时长大于 0 时针对精确到采样点的容器报告了错误的 DurationMin/DurationMax。
- **WG-58947** 已修复：Contents Editor 中没有对 SoundSeed Grain 执行文件名显示更新。
- **WG-59021** 已修复：由于对在播声音的 Speaker Panning / 3D Spatialization Mix 进行了调节，导致在声音由 Main Mix 过渡到 Audio Object 时产生可辨的咔哒声。
- **WG-59071** 已修复：在加载新的 IR 文件后没有刷新 Wwise Convolution Reverb Channel Config 下拉列表。
- **WG-59244** 已修复：在 Master-Mixer Hierarchy 下重新加载 Work Unit 时没有评估自定义效果器的 Media ID。
- **WG-59273** 已修复：无法在 macOS Monterey 12.3 上启动 Wwise。
- **WG-59359** 已修复：有时会过早退出 WwiseConsole。
- **WG-59425** 已修复：Master Audio Bus 上的非原地对象处理器效果器向 Audio Device 发送了错误的数据。
- **WG-59473** 已修复：在加载采用更高版本的 Wwise SDK 构建的插件时，Wwise 设计工具可能会发生崩溃。
- **WG-59568** 已修复：在 Filter 模式下应用卷积处理时会将左声道复制到所有干声输出声道。
- **WG-59569** 已修复：WwiseConsole 中有时会忽略主循环的超时状态。

# 社区报告的漏洞修复

- **WG-57147** 已修复：在激活 Game Object 3D Viewer 时，`GameObject3DCtrl::TermScene` 发生崩溃。
- **WG-57941** 已修复：在由于 Playback Limit 而将声部发送到虚声部且声音选中 Hold Emitter Position and Orientation 选项时没有更新 Priority "offset by distance"。
- **WG-58626** 已修复：(Spatial Audio) 在发声体或听者慢慢穿过尚未出现在路径上的 Portal 时，Spatial Audio 所应用的 Spread 有时不连续。
- **WG-58706** 已修复：没有执行迁移步骤来从 iZotope Box Modeler 出厂素材中移除 boxmodel.wav 原始文件。
- **WG-58743** 已修复：在穿过 Portal 时，即便衍射值没有超出 100，长的衍射路径也有可能会被切断。
- **WG-58802** 已修复：`AkBufferPosInformation` 结构中存在未初始化的变量。
- **WG-58837** 已修复：在异步加载 Init.bnk 时有时无法加载 Audio Device 效果器。
- **WG-58943** 已修复：在 Windows 和 Xbox 上使用 3D Audio 时可能会发生崩溃并出现音频损坏。
- **WG-59019** 已修复：在 `AK::SoundEngine::Init()` 之后立即调用 `AK::SoundEngine::AddOutput()` 可能会发生崩溃。
- **WG-59259** 已修复：在内存不足的情况下设置带有过渡的 RTPC 值时发生崩溃。
- **WG-59382** 已修复：(Spatial Audio) 对于穿过多个 Portal 的直线路径，有时会出现非零衍射。
- **WG-59449** 已修复：WwiseAuthoringAPI SampleClient 的 AkAutoBahn 库中存在 codecvt 依赖项。
- **WG-59602** 已修复：在启动新的实例时，有些 SoundSeed Woosh Randomizer 选项无法正常工作。
- **WG-59751** 已修复：在使用特定的 Switch Container 时，SoundBank 生成期间发生死机。
