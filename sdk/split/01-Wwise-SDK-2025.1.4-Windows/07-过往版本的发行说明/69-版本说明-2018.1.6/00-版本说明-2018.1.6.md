# 版本说明 2018.1.6

|  |

| Wwise SDK 2025.1.4 - Windows |

版本说明 2018.1.6

以下各节列举并描述 Wwise 版本 2018.1.5 和版本 2018.1.6 之间的变化。有关早期版本的信息，请参阅 [过往版本的发行说明](previousreleases.html) 章节。此版本为漏洞修复版本，因此对所有 2018.1.x 版本来说，工程、音频包和 API 都是兼容的，不会有任何变动。

|  |  |
| --- | --- |
|  | **备注:** **插件开发人员：** 此 Wwise 版本支持通过 Wwise 2018.1 构建的插件。对于通过旧版 Wwise 构建的插件，开发人员必须使用兼容版本重新编译并重新打包。 |

# 性能变化

- **WG-41202** 改进了 Crankcase REV 源插件，将运行速度提升了 4 ~ 10 倍。
- **WG-41255** 改进了 Spatial Audio 反射点的计算方法，可大幅提升性能。

# 行为变化

- **WG-39890** 现在，Spatial Audio 的 `AK::SpatialAudio::SetEmitterAuxSendValues()` 会直接从 `AkAuxSendValue` 数组传给函数的参数获取 listenerID 字段。以前，Spatial Audio 会修改 listenerID 字段，并设为发声体当前所在 Room 的 Room ID。但是，这样的话灵活性会受到限制。假如需要沿用先前行为，现在必须显式传递 AkRoomID（即传给 `AK::SpatialAudio::SetGameObjectInRoom()` 的 Room ID），并将其设为 listenerID。
- **WG-40964** 现在在后台运行 Unity 游戏时可连接到游戏的构建版本。
- **WG-41738** SoundSeed Grain Source 选项卡中的瞬态起始点检测器灵敏度进行了优化，现在会首先选取较为明显的瞬态起始点。如需详细了解应用此改进后迁移期间会如何移动标记，请参阅 [迁移说明](whatsnew_2018_1_6_migration.html) 。

# 其它更改

- **WG-41288** 针对非混音总线改进了 Mixer Plug-in 选项卡中的消息功能。

# 漏洞修复

- **WG-38798** 已修复：无法使用键盘快捷键启用/禁用 Reflect 的 Mute 和 Solo 按钮。
- **WG-39601** 已修复：在按下 Shift-Play 播放 Sequence Container 时发生崩溃。
- **WG-39918** 已修复：假如在 SoundBank 生成期间执行 WAAPI 调用，有可能会发生崩溃。
- **WG-39944** 已修复：Opus 编码器将 5.1.2 声道音频源下混为 5.1。
- **WG-40168** 已修复：(Windows 7) XAudio2 错误地使用默认设备来播放指向其他音频设备的二路输出。
- **WG-40248** 已修复：在 Spatial Audio 中，假如只在开始播放时设置“发声体”游戏对象的位置，Reflect 插件的发送音量有时会收到错误的值，甚至有可能会听不见声音。
- **WG-40358** 已修复：假如 WAAPI 脚本正在运行并通过 `path` 返回选项侦听 `ak.wwise.core.preDeleted` 主题，在 Wwise 中删除 Music Segment 会引发崩溃。
- **WG-40367** 已修复：对于通过 Sequence Container 或 Random Container 播放的声音，假如启用 Transition 并设为 Trigger Rate，同时使用 3D Position Automation 提供的定位路径，在开始播放时总是采用新的路径。
- **WG-40392** 已修复：在性能分析期间按下 Follow Capture Time 时，Capture Log 不会滚动到最后一个条目。
- **WG-40550** 已修复：(Windows) 在声音引擎初始化之前，无法成功调用 `AK::GetDeviceIDFromName` 。
- **WG-40858** 已修复：(Android) 在每帧采样数设置初始化为 1008 时发生崩溃。
- **WG-40933** 已修复：在使用 ADPCM 文件时，假如 SoundSeed Grain 的发射速率太低且速度不断变化，有可能会发生崩溃。
- **WG-41024** 已修复：在将高通或低通滤波器应用于非单声道声音时，假如播放该声音的 Audio Bus 指定有 Mixer Plug-in，音频会产生音高很高的静态噪音或者突然静音。
- **WG-41029** 已修复：假如声音的输出总线静音但辅助发送可闻，在初始化时声音会被错误地归为虚声部。
- **WG-41048** 已修复：在 SoundBank Editor 中选中对象后，右键单击并选择 Show in Multi Editor 时不能显示正确的对象。
- **WG-41057** 已修复：(WAAPI) 为不支持的浮点值返回了无效的 JSON。
- **WG-41072** 已修复：在所连目标引用的 Wwise Object 已从工程中删除时，Voice Monitor 和 Advanced Profiler 中的 Mute 和 Solo 按钮无响应或显示有误。
- **WG-41108** 已修复：(iOS) 在暂停并恢复 iOS 应用后，无法将该应用识别为 Remote Connection 目标。
- **WG-41179** 已修复：在转码后采样率和输入采样率不同时，SoundSeed Grain 无法正确选取粒子。
- **WG-41244** 已修复：在复制粘贴 SoundSeed Grain 实例时发生崩溃。
- **WG-41260** 已修复：在 Transport Control 中按下 Shift-Play 播放声音时无法正确响应 RTPC 变化。
- **WG-41306** 已修复：在播放音高异常高的声音时，可能会导致声音引擎触发无限循环，进而导致软件挂起。
- **WG-41336** 已修复：在 State 发生变化并影响到 Output Bus LPF/HPF 时，非混音总线无法正常过渡。
- **WG-41371** 已修复：在 SoundSeed Grain 中无法对 Ambisonics 音频源实施 3D Spatialization。
- **WG-41402** 已修复：在 Crankcase REV 中导入模型时发生崩溃。
- **WG-41424** 已修复：在 List View 和 SoundBank Manager 中执行某些操作时，Wwise 设计工具出现内存泄漏。
- **WG-41494** 已修复：在 Contents Editor 中单击新的对象时发生崩溃。
- **WG-41497** 已修复：存在内存损坏问题，在查看新的 Modulator RTPC 对象时可能会导致 Wwise 设计工具发生崩溃。
- **WG-41688** 已修复： `PostMIDIOnEvent` 一直锁定游戏对象。
- **WG-41747** 已修复：在将 Wwise 连接到使用 SceAudo3dMixer 插件的游戏时，游戏端发生崩溃。
