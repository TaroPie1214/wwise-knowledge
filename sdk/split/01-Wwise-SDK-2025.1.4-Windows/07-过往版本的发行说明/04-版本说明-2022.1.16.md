# 新增功能

- **WG-68107** (wp.py) 现在允许在构建后暴露参数并传给 `SignTool.exe` 来在 Windows 上对动态库进行签名。
- **WG-69592** 添加了通过 wp.py 对特定工具集执行 premake 操作的选项。

# 漏洞修复

- **WG-73491** 已修复：在使用浮点输入源重新采样时内存不足导致发生崩溃。
- **WG-73617** 已修复：在对 Vorbis 音频源进行解码时内存不足导致内存泄漏。

# 社区报告的漏洞修复

- **WG-63881** 已修复：在更改音乐对象的输出总线时发生崩溃。
- **WG-69402** 已修复：相对于 WwiseConsole.exe 而言，有些使用 Wwise.exe 生成的 `.bnk` 和 `.wem` 文件存在二进制差异。WwiseConsole.exe 生成的文件保持不变，Wwise.exe 生成的文件则略有不同。
- **WG-71833** 已修复：(Mastering Suite) 多声道音频对象导致增益估算出错。
- **WG-72796** 已修复：(macOS) EULA 消息框有时不显示任何内容。
- **WG-73257** 已修复：在使用 `AK::SoundEngine::SetSpeakerAngles` 时，ComputeVBAP3D 发生崩溃。
- **WG-73692** 已修复：在媒体处于在用状态时添加它的多个副本会导致 `AkMediaEntry::Release` 发生崩溃或触发断言。
- **WG-73708** 已修复：在使用 PCM 编解码器时，`Bypass_I16_2ChanAVX2` 发生崩溃。
- **WG-73849** 已修复：在更改 Project Settings 或生成 SoundBank 后 [ak.wwise.core.getProjectInfo](ak_wwise_core_getprojectinfo.html) 没有提供正确的 SoundBank 路径。
