版本说明 2018.1.9

以下各节列举并阐述了 2018.1.8 和 2018.1.9 版本之间针对 Wwise 所作的改进。有关早期版本的信息，请参阅 [过往版本的发行说明](previousreleases.html) 章节。此版本为漏洞修复版本，因此对所有 2018.1.x 版本来说，工程、音频包和 API 都是兼容的，不会有任何变动。

|  |  |
| --- | --- |
|  | **备注:** **插件开发人员：** 此 Wwise 版本支持通过 Wwise 2018.1 构建的插件。对于通过旧版 Wwise 构建的插件，开发人员必须使用兼容版本重新编译并重新打包。 |

# 平台 SDK 改进

- **Lumin SDK**：已更新至 SDK 0.22.0。
- **PS4 SDK**：已更新至 SDK to 6.508.031。
- **Stadia SDK**：添加了 SDK 1.34。
- **Switch SDK**：已更新至 SDK to 8.3.0。
- **Xbox One XDK**：已更新至 XDK 10.0.17134.5049 (2018年7月 QEF5)。

# 新增功能

- **WG-43522** (Linux) 现在将 PulseAudio 设成了 Linux 上的默认 System Sink。
- **WG-44662** 添加了对 Stadia 的初步支持。

# 性能改进

- **WG-43686** 已修复：在连接到具有庞大的总线层级结构的游戏时，Master-Mixer 视图中存在性能问题。
- **WG-44270** 修复了 Switch 和老的 ARM CPU 上 ADPCM 性能方面存在的小问题。

# 行为改进

- **WG-44336** 行为变化：在实施第 WG-14711 项修复后，我们废除了原有行为。之前，小节和拍子通知有时会在段落结尾附近发送一条多余的通知。现在修复了这一问题，**因此必须将游戏原本用来处理这一状况的变通措施全部还原**。

# 漏洞修复

- **WG-14711** 已修复：在采用特定 BPM 值时，音乐同步回调会在小节结尾触发一个多余的非法拍子。
- **WG-37311** 已修复：在通过游戏线程调用 [AK::SoundEngine::RegisterBusMeteringCallback](namespace_a_k_1_1_sound_engine_a4935dbc8c375c82c03475a1dbde37a51.html#a4935dbc8c375c82c03475a1dbde37a51) 时可能会发生崩溃。
- **WG-42340** 已修复：对于包含 CrankCaseAudio REV 数据的 Soundbank，生成的音频包二进制文件有时会不同。 已修复：提升了整体的稳定性。
- **WG-42488** 已修复：(UWP) 在移除 Motion 输出后，手柄振动仍会持续几秒。
- **WG-43113** 已修复：在加载包含无效的 Music Switch Container 或 Dialogue Event 参数的工程时发生崩溃。
- **WG-43336** 已修复：在生成音频包时发生罕见崩溃。
- **WG-43438** 已修复：在保存到 WWISEROOT% 以外的驱动器时，无法构建使用 wp.py 创建的插件。
- **WG-43480** 已修复：在连接具有庞大的总线层级结构的游戏时出现严重的延迟。
- **WG-43586** 已修复：在 Wwise 之外执行文件操作时，有时不显示工程变更通知。
- **WG-43814** 已修复：在连接 Wwise 时错误地将游戏中的静态值应用于 Exclusive RTPC。
- **WG-43832** 已修复：对于 5.1 和 7.1 声道配置，在 Android 上初始化声音引擎时发生崩溃。
- **WG-44023** 已修复：部分 Sound Engine 解决方案中没有针对 Windows 定义 AK\_ENABLE\_ASSERTS。
- **WG-44057** 现在为自定义命令文件创建了不同的版本。
- **WG-44110** 已修复：(Linux) 在初始化 Pulse Audio Sink 插件时可能会应用不理想的缓冲属性。
- **WG-44160** 已修复：在右键单击 Post Event Action 的 Target 字段时，Wwise 出现卡顿/发生崩溃。
- **WG-44194** 已修复：若在选择 Remove Path(s) 上下文菜单选项时没有选中任何元素，则 Dynamic Dialogue 编辑器会发生崩溃。
- **WG-44234** 已修复：在针对 Sound SFX 对象执行撤消操作时，nullptr 访问引发罕见崩溃。
- **WG-44267** 已修复：(Android) 若采用了不支持的扬声器设置，则初始化声音引擎时会发生崩溃。
- **WG-44280** 已修复：在迁移超大 Work Unit 文件（文件中包含的 XML 节点对象超过 10,000,000 个）时有可能会导致工程不完整。
- **WG-44541** 已修复：在同时播放多个声音实例的情况下，无法写入转码后的文件。
- **WG-44544** 已修复：在打开新的 Multi Editor 窗口时有时无法正确更新或禁用属性。
- **WG-44566** 已修复：在更改主输出设备或音频硬件暂时中断时，NSW 发生崩溃。
- **WG-44700** 已修复：在为多声道文件应用 EQ 时，音频可能会出现毛刺噪声。
