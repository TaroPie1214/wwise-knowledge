版本说明 2017.1.2

以下各节列举并描述 Wwise 版本 2017.1.1 与版本 2017.1.2 之间的变化。有关早期版本的信息，请参阅 [过往版本的发行说明](previousreleases.html) 章节。如果使用联机帮助，您还可以从目录列的顶部选择别的 Wwise 版本。

# 漏洞修复

- **WG-33811** 已解决：在一些动态通路情境下，静音或 solo 辅助总线时无法按预期工作。
- **WG-34217** 已解决：输出总线参数 RTPC 的相关问题。
- **WG-34289** 已解决：带有混音子总线的混音器插件可能导致生成的 Init.bnk 不同，即使工程中并没有变更。
- **WG-34317** 已解决：（PS4）即使包中不包含 ATRAC9，依然进行 ATRAC9 校正。
- **WG-34396** 已解决：Tab Delimited Import（制表符分割导入）和 WAAPI 导入会生成重复条目。
- **WG-34501** 已解决：带有大 State Group 时，Soundbank 生成中有错误。
- **WG-34504** 已解决：在使用精确到采样点的过渡，并且媒体在多个包中被复制时，卸载包会导致罕见崩溃。
- **WG-34509** 已解决：（Spatial Audio） `SetEmitterPosition` 破坏虚拟位置数据，可能导致崩溃。
- **WG-34512** 已解决：循环点反向时导致 Invalid File Header（无效文件头）错误。
- **WG-34540** 已解决：（WAAPI）当请求超过 4,096 字节时，Wwise 可能崩溃。
- **WG-34542** 已解决：（PS4）如果有两个输出设备在活动中，那么关机会导致崩溃。
- **WG-34545** 已解决：（WAAPI）当使用 `ak.wwise.core.audio.import` 时，Switch Container 指派、Event 创建和 Dialogue Event 创建无法进行。
- **WG-34551** 已解决：在工程加载时，与总线上的 States 实时混音不起作用。
- **WG-34577** 已解决：（Switch）在 `AkIsValidThread` 中缺少初始化线程状态验证。
- **WG-34593** 已解决：当在工程中使用 HDR 转码音频文件时发生罕见 Wwise Authoring 卡死。
- **WG-34597** 已解决：当声音在播放时，纹理参数不更新。
- **WG-34603** 已解决：当导入并播放一个使用 IBM Text-To-Speech 生成的 WAV 文件时崩溃。
- **WG-34607** 已解决：当在 Event Viewer 窗格中按下 Ctrl+X 时崩溃。
- **WG-34608** 已解决：在保存 Music Track 预设值时崩溃。
- **WG-34612** 已解决：（Spatial Audio）在计算反射路径时，同平面的三角形可能彼此阻挡。
- **WG-34617** 已解决：“在 External Editor（外部编辑器）上打开”在 macOS 上无法正常工作。
- **WG-34634** 已解决：撤销然后重做 Dialogue Event 创建后发生罕见的撤销堆栈损坏。
- **WG-34637** 已解决：Music Editor 标尺文本在高 DPI 显示器上被截短。
- **WG-34641** 已解决：使用了特定语言源代码排除时，无法迁移旧的 Wwise 工程。
- **WG-34643** 已解决：当 `AK::InitSettings::bUseSoundBankMgrThread` 被设置为 `false` 时，`UnloadBank` 有时会返回错误码。
- **WG-34647** 已解决： CopyStreamedFiles：在零延迟媒体中使用 `-includedFilesInBanks` 选项时断言。
- **WG-34688** 已解决：`TrimBegin` 和 `TrimEnd` 选项返回同样的订阅 ID。
- **WG-34708** 已解决：直达声和非常近的墙上产生的一阶反射声间的延迟无缘无故过长。
- **WG-34716** 已解决：Acoustic Texture Editor 不应有 RTPC 选项卡。
- **WG-34727** 已解决：当 Solo 反射时发生罕见崩溃。
- **WG-34774** 已解决：总线音高的 RTPC 被应用在互动音乐上。
- **WG-34793** 已解决：Wwise Reflect 的延迟线更新不正确。
