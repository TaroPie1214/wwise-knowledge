版本说明 2016.1.1

以下各节列举并描述 Wwise 版本 2016.1 和版本 2016.1.1 之间的变化

# API 改进

在 Windows上，要调用 `AK::GetDeviceID，AK::GetDeviceIDFromName，以及AK::GetWindowsDeviceName 这几个调用，必须要在 CoInitialize （微软的` COM API） 为调用线程所调用之后。

# 平台 SDK 的变化

- **WG-30238** Linux： iZotope 插件现已可用。
- **WG-30383** PS4：SDK 更新到 3.500。
- **WG-30384** Xbox One：XDK 更新到2016年3月。

# 漏洞修复

- **WG-27354** 已解决：当旁通一个位置错误的 Effect（效果器），如插入在一个0.1 （LFE）声源上的 TimeStrech时，做非重要断言。
- **WG-28848** 已解决：转码后文件内的 RIFF 提示点和标签依然存在，且时间点不正确，即使是被修剪工具处理过之后。
- **WG-29122** 已解决：在 AkSinkDirectSound.cpp 的输出调试字符串中没有回车。
- **WG-29313** 已解决： 当运行 Profile\_EnableAsserts 配置而又不使用自定义断言钩子时，发生崩溃。
- **WG-29480** （iOS，tvOS，Mac，Android）启用Crankcase Audio REV 插件的编译和打包功能。
- **WG-29805** 已解决：（iOS, Mac） AAC 编码解码器被无任何用处地限制在最多2声道，现已支持7.1。
- **WG-29836** 已解决：当在新机器上首次转码音频文件时偶尔出现文件访问失败。
- **WG-29849** 已解决：用于3D定位的环绕扬声器角度错误，配置为有高度声道但无后置扬声器（即 5.1.2、Auro 9.1、Auro 10.1，以及 Auro 11.1）。
- **WG-29925** 已解决：（iOS） 游戏在连接蓝牙耳机后卡死。
- **WG-30026** 已解决：（Windows 7）XAudio2.7 无法在 Win 7 上使用，导致崩溃或无声音。
- **WG-30035** 将所有 POSIX 平台的 Default Stack Size （默认堆栈大小）增加到 64 KB。
- **WG-30111** 已解决：（iOS） `AudioSessionCategoryAmbient` 在 Unity 中不工作。
- **WG-30146** 已解决：当生成 SoundBank 时，如果文件不存在并且定义文件 Init.bnk 没有新的更改，则 Init.txt 不会重新生成。
- **WG-30149** 已解决：`Non-cachable` （不可缓存的）声音属性在 **All Properties** 清单中正确的子群内不显示。
- **WG-30186** 已解决：第三方 Motion 插件导致载入崩溃。
- **WG-30196** 已解决：在 AkResamplerSIMD.cpp 中的 Div by 0（被0除）非法运算。
- **WG-30211** （Android）将默认 `platformInitSettings.uNumRefillsInVoice` 更改为适应蓝牙（6个缓冲区而不是4个）。
- **WG-30232** 已解决：在载入 Init bank 之前连接到 Wwise 会引起 sink 插件故障。
- **WG-30269** 已解决：（PS4）当音高在一阵内大幅度降低时导致 AT9 包源匮乏。
- **WG-30280** 已解决：(Android) 在启动时，`CAkSinkOpenSL` 内发生罕见的崩溃。
- **WG-30285** 已解决：（iOS） Unity 广告导致音频无法恢复。
- **WG-30296** 已解决：当转换带有错误数据块大小的 WAV 文件的采样率时，Wwise 会挂起。
- **WG-30312** 已解决：（Windows）当调用 `GetWwiseXAudio2Interface` 并且 XAudio 未初始化时崩溃。
- **WG-30319** 已解决：在 AkSink 插件代码示例中缺失源文件。
- **WG-30321** 已解决：当以某些角度在 5.1.2 总线上对3D声音进行声像摆位时崩溃。
- **WG-30327** 已解决：Sink 插件使用不同的 ShareSets 载入。
- **WG-30332** 已解决：当子线取代输出总线时，一些 Blend Container 和 Switch Container 参数在回放时会被忽略。
- **WG-30333** 已解决：（iOS） AAC 声音接管用户音乐（只是 AudioSession SoloAmbient 这个类别）。
- **WG-30338** 已解决：当在尾部过程中旁通位置错误的总线 Effect 时崩溃。
- **WG-30371** 已解决：（Mac）设计工具：User Defined Position Editor 3D （用户定义位置编辑器3D）无法打开。
- **WG-30385** 已解决：当关闭工程时 Perforce 偏好未保存。
- **WG-30391** 已解决：当用调制器暂停/恢复声音时偶尔会触发断言。
- **WG-30399** 已解决：实时更新 RTPC 曲线之后，关联属性上的 RTPCs 被破坏。
- **WG-30401** 已解决：迁移项目时可能崩溃。
- **WG-30492** 已解决：Error Report（错误报告）： SplitWords 在 std::locale 构造函数中 raise Exception（抛异常）。
