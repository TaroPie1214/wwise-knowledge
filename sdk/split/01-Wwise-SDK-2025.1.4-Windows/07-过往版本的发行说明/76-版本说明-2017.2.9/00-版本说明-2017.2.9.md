版本说明 2017.2.9

以下各节列举并描述 Wwise 版本 2017.2.8 和版本 2017.2.9 之间的变化。有关早期版本的信息，请参阅 [过往版本的发行说明](previousreleases.html) 章节。如果使用联机帮助，您还可以从目录列的顶部选择别的 Wwise 版本。此版本为漏洞修复版本，因此对所有 2017.2.x 版本来说，工程、音频包和 API 都是兼容的，不会有任何变动。

# 平台 SDK 的变化

- **Lumin SDK**：已更新至 SDK 0.18.0。
- **PS4 SDK**：已更新到 SDK 6.008.001。
- **Switch SDK**：已更新至 SDK 6.4.0。

# 漏洞修复

- **WG-32443** 已修复：(PS4) 在运行 SCE Bed 时调用 suspend 会引发崩溃。
- **WG-32512** 已修复：在选定项包含无法静音的对象时，有时会无法清除 Mute 和 Solo 状态。
- **WG-36264** 已修复：Music Segment 上下文菜单发生崩溃。
- **WG-36410** 已修复：有些固定宽度的列在后续会话中会变窄。
- **WG-39857** 已修复：UI 出现故障，导致 Wwise 在应用部分插件（包括 McDSP Limiter 和 Futzbox）时发生随机崩溃。
- **WG-39871** 已修复：在处理某些耗时的进程时关闭 Wwise 两次会引发崩溃。
- **WG-39917** 已修复：(PS4) PS4 控制器上的 Motion 效果减弱。
- **WG-40036** 已修复：在较响的声音启用了辅助发送时，干声和湿声路径中的 HDR 衰减不均衡。
- **WG-40097** 已修复：（Windows、Xbox One）即便指定了别的设备，Microsoft Spatial 输出也会继续选择默认输出设备。
- **WG-40152** 已修复：无法将 Wwise 连接到不同子网中的主机。
- **WG-40211** 已修复：在包含大量 Event 的工程中浏览目标时，Wwise 发生崩溃。
- **WG-40330** 已修复：(PS4) 扬声器配置始终为 7.1。
- **WG-40724** 已修复：在 5.1 声道设置下，Wwise Flanger 和 Wwise Tremolo 插件发生罕见崩溃。
- **WG-40878** 已修复：(Linux) 在个别情况下，声音引擎初始化之后不久音频线程偶尔会挂起很长时间，进而导致音频卡顿。
- **WG-40915** 已修复：工程名称中第一个点号之后的字符被误删。
- **WG-41037** (Android) `AK::SoundEngine::GetFastPathSettings` 现在无论在任何情况下都可以正确地报告最优设备设置。
