# 行为改进

- **WG-57727** 已修复：在采用 Ambisonics 或 Objects 配置时，Expander 中显示的结果不准确。

# 性能改进

- **WG-43880** 在大部分情况下将 Wwise Compressor 和 Wwise Expander 效果器的吞吐量性能提升了 2 倍之多。
- **WG-57501** 提升了 Reflect 的运行时性能。

# 其他改进

- **WG-57071** 针对 WAAPI 示例和主题更新了 npm 依赖项库 (package.json)。
- **WG-57526** Wwise 现在支持 Windows 11。
- **WG-57641** 针对内存溢出添加了调试工具。

# 漏洞修复

- **WG-56196** 已修复：在 Wwise 关闭时，Advanced Profiler 视图发生崩溃。
- **WG-57257** 已修复：在音频设备列表发生变化时，Windows 发生崩溃。
- **WG-57385** 已修复：在将损坏的 .wav 文件用于 Impacter 插件时发生崩溃。
- **WG-57483** 已修复：在清理 SoundBank 后针对暂停的 Interactive Music 对象调用 `StopAll` 操作时发生崩溃。
- **WG-57532** 已修复：在使用 Profiler 和 Voice Inspector 时发生崩溃。
- **WG-57575** 已修复：在针对使用 Convolution Reverb 的工程生成 SoundBank 时，会忽略取消链接的 HW Accel 值，而将链接的值用于所有平台。
- **WG-57702** 已修复：在采用 Ambisonics 配置时，Compressor 中的增益估算不准确。
- **WG-57739** 已修复：对于大于 5.1 的配置，Loudness Meter 不使用标准 BS.1770-4 声道加权。
- **WG-57851** 已修复：(Spatial Audio) 在两个 Portal 位置靠近并连接两个以上 Room 时，两者可能会相互干扰，造成不符合预期的辅助发送、衍射和透射结果。
- **WG-58004** 已修复：在移除 Windows 输出声音设备时发生崩溃。

# 社区报告的漏洞修复

- **WG-42355** 已修复：在播放过程中没有立即应用对 Speaker Panning / 3D Spatialization Mix 参数所作的更改。
- **WG-54925** 已修复：在 Windows 上通过 Wwise 调用通信设备会导致其他应用程序被闪避。
- **WG-57194** 已修复：由于在主机上安装了 CRT，导致 SoundBanks.xml 数据中的 MaxAttenuation 和 MaxDuration 信息出现舍入差异。
- **WG-57418** 已修复：在取消链接 System Audio Device Editor 中的效果器后重新打开工程时显示警告消息。
- **WG-57461** 已修复：在当前输出音频设备与新的 ShareSet 相同且与多个 Audio Bus 相连时，通过 Audio Device ShareSet 调用 ReplaceOutput 可能会导致音频丢失。
- **WG-57520** 已修复：“管理插件媒体”的示例代码无法编译。
- **WG-57714** 已修复：在对 Spatial Audio 实施性能分析时发生崩溃或出现内存 stomp 问题。
- **WG-57768** 已修复：在反复生成 SoundBank 时，SoundBanksInfo.xml 中 ExternalSources 的顺序不一致。
- **WG-57828** 已修复：在不同电脑上生成 SoundBank 时，SoundBanks.xml 元数据中所列的 SoundBank 生成时间有所不同。
- **WG-57884** 已修复：在将 Wwise 设计工具连接到配置有 Profile\_EnableAsserts 的 Sound Engine 时出现错误。
- **WG-57897** 已修复：在调用 `AK::SoundEngine::Suspend` 时没有应用淡出。
- **WG-57931** 已修复：在连接 Wwise Profiler 时，SoundEngine 崩溃或挂起。
