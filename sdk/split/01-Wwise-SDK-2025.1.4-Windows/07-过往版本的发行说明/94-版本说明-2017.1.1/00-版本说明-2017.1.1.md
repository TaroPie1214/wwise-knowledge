版本说明 2017.1.1

以下各节列举并描述Wwise版本2017.1与版本2017.1.1之间的变化。有关早期版本的信息，请参阅 [过往版本的发行说明](previousreleases.html) 章节。如果使用联机帮助，您还可以从目录列的顶部选择别的 Wwise 版本。

# 行为变化

- **WG-33961** 现在可以在 Wwise 2017.1.1 和更高版本中读取 2016.1 之前的外部源和流播放 WEM 文件。

# 平台 SDK 的变化

- **Xbox One XDK:** 已更新至 XDK to 170301 (2017年3月 QFE 1)

  |  |  |
  | --- | --- |
  |  | **备注:** 从 XDK 的这一版本开始，不再支持 [Microsoft](namespace_microsoft.html) Visual Studio 2012，并且开始支持 [Microsoft](namespace_microsoft.html) Visual Studio 2017。 |

# 新增功能

- **WG-34177** 为 WAAPI 添加了 Python 3.6 的支持。所有的 Python 代码示例已用最新的 Autobahn 版本（17.8.1）进行了测试，该版本使用 Python 3.6 的内置 *asyncio* 库和 Python 3.7的 *trollius* 库。

# 漏洞修复

- **WG-33074** Emitter-Listener（发声体-听者）选项卡的“Default（默认）” Game Object 不再能够添加到 Watches 选项卡。
- **WG-33554** 已解决：启用定位选项不再在 Mixing Desk 中可用。
- **WG-33565** 已解决：即使总线已启用 2D，在 3D Game Object 查看器中依然显示衰减。
- **WG-33794** 已解决：在调用 WAAPI 时退出 Wwise 有时会导致即使关闭主窗口之后 Wwise.exe 进程依然在运行。
- **WG-33845** 已解决： 当将选项变更为一个混合值时，在 Multi Editor 中的 Channel Configuration Override 组合框可能不更新。
- **WG-33956** 已解决： 带有精确到采样点的过渡的容器如果有子容器，且子容器带有 Trigger Rage 过渡，则可能导致崩溃。
- **WG-34020** 已解决：在生成包含音乐对象的 SoundBank 时， Wwise 可能会花很多时间在 SoundBank 生成过程中的‘收集 SoundBank 信息'阶段。
- **WG-34023** 已解决：在重新注册听者之后，游戏中没有声音。
- **WG-34024** 已解决：当子对象覆盖到其父级 Blend Container 的同一条输出总线时，Bus Voice（总线声部）属性错误地被应用两次。
- **WG-34028** 已解决：（PS4）在浸泡测试过程中（超过 12 小时）发生罕见死锁。
- **WG-34097** 已解决：（Mac）如果工程是直接通过 Launcher 打开的，那么将 Factory Assets 导入到 Integration Demo 会导致卡死。
- **WG-34103** 已解决：当在立体声声部上使用 1.1（Center + LFE）冲激响应时，在 Convolution Reverb （卷积混响）中发生崩溃。
- **WG-34119** 已解决：当调用 0 发送的 Spatial Audio 辅助发送 API 时，API 性能分析器会报告无效数据。
- **WG-34159** 已解决：如果 Windows 用户的键盘布局默认不是英语，Wwise 会删除英语键盘布局。
- **WG-34206** 已解决：声学纹理预设值不应打包在包（bank）中。
- **WG-34207** 已解决：Sink 插件出故障时不恢复到默认的 Sink，导致静音。
- **WG-34208** 已解决：调用 `SetDefaultListeners()` 时不能正确地更新正在播放的声音；声音变得听不见。
- **WG-34210** 已解决：当连通单声道到 LFE 总线并且 2D 定位启用声像摆位器时，发生内存损坏。
- **WG-34214** 已解决：使用 `RegisterBusVolumeCallback()` 注册回调后，不会对其进行调用。
- **WG-34221** 已解决：当销毁类型为 `CAkConnectedListeners` 的静态对象时，在关机时发生崩溃。
- **WG-34226** 已解决：Harmonizer Effect 在一定条件下可能在音频信号中生成非法数值。
- **WG-34243** 已解决：（Switch）在 Opus 解码开始时，出现性能问题。
- **WG-34273** 已解决：（Switch）接收端不遵守 `AkInitSettings::settingsMainOutput::channelConfig`。
- **WG-34284** 已解决：（Android）在特定硬件（mobiistar, Nova, Play Spark）初始化时出现被零除。
- **WG-34344** 已解决：在 Profiler 中整理 Loaded Media 栏时崩溃。
- **WG-34349** 已解决：（PS4）在创建硬件实例失败后发生断言。
- **WG-34350** 已解决：如果使用 `AK::IAkGlobalPluginContext::ComputeWeightedAmbisonicsDecodingFromSampledSphere` 时具有非常大的点数，则可能导致堆栈溢出。
- **WG-34354** 已解决：（PS4）在 PS4 文档- `libSceAjm_stub_weak` 中缺失依赖。
- **WG-34376** 已解决：当编辑 Wwise 数值时，在 Multi Editor 中发生崩溃。
- **WG-34378** 已解决：Game Object Profiler 中的空内存访问崩溃报告。
