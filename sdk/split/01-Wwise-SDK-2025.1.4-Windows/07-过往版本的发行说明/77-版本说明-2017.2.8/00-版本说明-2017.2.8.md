版本说明 2017.2.8

以下各节列举并描述 Wwise 版本 2017.2.7 和版本 2017.2.8 之间的变化。有关早期版本的信息，请参阅 [过往版本的发行说明](previousreleases.html) 章节。如果使用联机帮助，您还可以从目录列的顶部选择别的 Wwise 版本。此版本为漏洞修复版本，因此对所有 2017.2.x 版本来说，工程、音频包和 API 都是兼容的，不会有任何变动。

# 平台 SDK 的变化

- **Magic Leap** 已更新至 SDK 0.16.0.
- **Mac**：最低目标 macOS 版本现在为 10.9 Mavericks。
- **Xbox One XDK**: 已更新至 XDK 10.1.17133.2020 (2018年4月)。
- **PS4 SDK**: 已更新至 SDK 5.508.041.

# 新功能

- **WG-37837** (WAAPI) 现在允许在导入时指定是否要将导入的文件自动添加到版本控制系统中。
- **WG-39152** (WAAPI) 为对象查询添加了返回选项 `audioSource:language` 。

# 性能变化

- **WG-39085** (Xbox One) 对于非 Debug 配置，移除了 ShapeXma 函数中的验证代码。

# 漏洞修复

- **WG-31497** 已修复：在配有多个网络适配器的机器上运行时，Wwise Remote Connect 对话框不显示全部应用程序。
- **WG-38280** 已修复：在加载包含链接的属性的 Preset 后，取消链接的属性将保持“取消链接”状态。
- **WG-38734** 已修复：在通过 SoundFrame 播放空的 Event 时发生崩溃。
- **WG-38849** 已修复：在播放父级音乐段落时，假如删除子级音乐片段，就会发生崩溃。
- **WG-39078** 已修复：(iOS) 在中断 Sink 时触发罕见断言。
- **WG-39089** 已修复：在音频包未全部同步时， `CAkActiveParent` 发生罕见崩溃。
- **WG-39096** 已修复：在 Capture Log Filter 中启用 **Include parents and busses** 时会导致崩溃。
- **WG-39098** 已修复：在针对 Blend Container 的子对象设置“不沿用 User-Defined Auxiliary Send”时，假如未定义子对象的辅助发送，设置会不起作用。
- **WG-39131** 已修复：在按名称监视游戏对象时，假如被监视对象的名称为另一游戏对象的子字符串，则后者可能会同时显示在 3D Game Object Viewer 中。
- **WG-39168** 已修复：在删除 Switch Container 中的多个声音时，假如 Source Editor 中正在显示容器中的某个声音，就可能会导致崩溃。
- **WG-39229** 已修复：在 Dialogue Event Editor 和 Music Switch Container Association Editor 中，无法重新排列 Switch Group 和 State Group 的顺序。
- **WG-39249** 已修复：`GetPanningRule`、`GetSpeakerAngles`、`GetSpeakerConfiguration` 和 `GetPS4OutputHandle` 不等声音引擎启动即返回默认值。
- **WG-39279** 已修复：在根目录导入文件时，Tab Delimited Import 出现卡顿。
- **WG-39286** 已修复：(Android) 在 Android P 上启动游戏时发生罕见崩溃。
- **WG-39312** 已修复：在将音量偏置传给 `SetListenerSpatialization` 和使用 Ambisonics 时， `CAkListener::ComputeSpeakerMatrix3D` 发生无限循环。
- **WG-39407** 已修复：在生成文件（如 SoundBank 和 WEM）时，无法通过重解析点查找这些 Wwise 文件。
- **WG-39441** 已修复：(Windows) Xbox One 控制器的振动效果减弱。
- **WG-39467** 已修复：在内存不足时，MS Spatial Sink 插件有可能发生崩溃。
- **WG-39474** 已修复：(Windows) 在有些硬件上将 Windows Playback Device Settings 设为高分辨频率 (192 kHz) 时，假如在 Windows 上使用 WASAPI 音频 Sink，有可能导致音频故障。
- **WG-39498** 已修复：在 Wwise 设计工具中切换工程时，XAudio2 发生崩溃。
- **WG-39509** 已修复：在填写 `AkInitSetting.settingMainOutput.audioDeviceShareset` 时，不会将主 Audio Bus 切换至指定设备。
- **WG-39515** 已修复：Reflect 的 Delay Line 管理发生罕见崩溃。
- **WG-39523** 已修复：Preset 未加载 Initial Delay。
- **WG-39529** 已修复：在输出设备没有准备就绪时启动会引发罕见崩溃。
- **WG-39541** 已修复：在有多个游戏对象播放同一 Switch Container 时，针对其中一个游戏对象调用 `StopPlayingID()` 会清除其他游戏对象的 Switch 历史记录。
- **WG-39544** 已修复：在指定了过渡时间的情况下， `SetRPTCValueByPlayingID` 函数调用忽略 Playing ID。
- **WG-39624** 已修复：在为总线选择新的混音器插件后不更新 Wwise Effect Editor。
- **WG-39678** 已修复：AkRecorder 文件路径较长时会被截断。
- **WG-39745** 已修复：在 44,100 Hz 采样率的设备上增大或减小 LPF 参数时发生故障。
- **WG-39783** 已修复：`GetSpeakerAngles()` 出现计时问题。假如在 `LoadBank("Init.bnk")` 之后很快调用，有可能会失败。
- **WG-39846** 已修复：在将 Wwise Profiler 连接至游戏时，`AkInitSettings` 中的 Audio Device 设置会被重置。
