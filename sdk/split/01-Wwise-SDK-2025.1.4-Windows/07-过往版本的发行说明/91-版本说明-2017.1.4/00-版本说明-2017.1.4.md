版本说明 2017.1.4

以下各节列举并描述Wwise版本2017.1.3与版本2017.1.4之间的变化。有关早期版本的信息，请参阅 [过往版本的发行说明](previousreleases.html) 章节。如果使用联机帮助，您还可以从目录列的顶部选择别的 Wwise 版本。

# 平台 SDK 的变化

- **Switch SDK:** 已更新至 SDK 3.5.1.

# 行为变化

- **WG-35338** Import Conflict Manager 现在允许多重选择，并支持针对错误列表内选中的所有行统一更改解决操作。

# 新增功能

- **WG-35367** 现在可在多个 Wwise 窗口中同时打开同一工程，而不会有警告提示。
- **WG-35417** 现在支持将 Control Surface 绑定应用于 Mixing Desk 中的 State。

# 漏洞修复

- **WG-28063** 已修复：在值达到 0 时出现 HPF clicking 噪声。
- **WG-34674** 已修复：在采用单声道配置（0.1 单声道除外）时，McDSP ML1 读取出现乱码。
- **WG-34756** 已修复：在终止声音引擎时可能会死机 `CAkAudioMgr::RemoveAllPreallocAndReferences。`
- **WG-35032** 已修复：若在替换原始 WAV 文件时其正显示在 Source Editor 或 Music Editor 中，则会出现罕见崩溃。
- **WG-35067** 已修复：在播放期间编辑音频源时可能会出现崩溃。
- **WG-35109** 已修复：若在启动 Wwise 时开着筛选器并连接至游戏，则无需单击 Filter... 按钮 Capture Log 便会加载信息。
- **WG-35118** 已修复：`CAkRTPCMgr::SetBuiltInParamValue()` 出现罕见崩溃。
- **WG-35245** 已修复：(Android) 在 Android 上，Unity 无法正常使用 Crankcase REV。
- **WG-35253** 已修复：SoundBank 中未打包 MIDI Target 层级结构。
- **WG-35257** 已修复：在修改特定 State 属性后退出工程并重新打开时，SoundBank 出现差异。
- **WG-35317** 已修复：若在成功迁移后包含被排除的音频源（已删语言），则会导致 Work Unit 损坏。
- **WG-35323** 已修复：在 Motion FX 对象上添加 State 时出现崩溃。
- **WG-35338** 已修复：在 Tab Delimited Import 过程中出现冲突时，Set all to **Cancel** 按钮显示为灰色。
- **WG-35359** 已修复：在阈值以下开始播放 MIDI 循环驱动的声音时出现崩溃。
- **WG-35390** 已修复：在连接性能分析器或所加载音频包包含已在播放的内容时，会白白重新安排互动音乐。
- **WG-35413** 已修复：User defined Aux Bus 复制有误。
- **WG-35430** 已修复：(Xbox One) 在挂起时，IAudioClient2 对象泄漏。
- **WG-35500** 已修复：若手动更改了 RTPC，则会忽略 Wwise Meter 的 RTPC 值。
- **WG-35531** 已修复：在加载从 ProTools 导出的 Ambisonics WAV 文件时出现错误。
- **WG-35590** 已修复：在 PS4 5.008.041 或更高版本上运行 Wwise Parametric EQ 时出现问题。
- **WG-35721** 已修复：在内存不足时，Vorbis 解码器出现崩溃。
- **WG-35728** 已修复：在使用 `CalcOcclusionAndVirtualPositions()` 访问来自游戏线程的 Spatial Audio 数据时死机。
