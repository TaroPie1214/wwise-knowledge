# 平台 SDK 的变化

- **Magic Leap** 已更新至 SDK 0.16.0.

# 新增功能

- **WG-35285** (WAAPI) AkAutobahn Client 现在支持断开后触发的回调。C++ SampleClient 新增了用于展示如何监控 Wwise 状态（连接、断开、工程更改）的例子。

# 性能变化

- **WG-39015** 优化了 Wwise 设计工具创建的 GDI 对象数量。
- **WG-39060** (PS4) 提升了 Wwise Profiler 计时器集合的性能。

# 其它更改

- **WG-38947** (WAAPI) 现在从 AkAutobahn 登录时会统一添加 AkAutobahn 前缀。

# 漏洞修复

- **WG-29891** 已修复：在取消链接和切换平台时，混音器插件属性丢失。
- **WG-38040** 已修复：衰减声锥与 Portal 衍射之间的联系中断。
- **WG-38442** 已修复：在某些旋转条件下，二阶 Ambisonics 播放流 U 声道的音量电平极高。
- **WG-38685** 已修复：在运行 WwiseCLI.exe 时，假如命令行中未提供 `-ConvertExternalSources` 可选平台参数和 `-ExternalSourcesOutput` 参数，就会显示无效错误消息。
- **WG-38687** 已修复：在重命名当前 Event 后，Event Editor 中显示的 Event ID 不随之改变。
- **WG-38715** 已修复：(PS4) 在调用 `AK::SoundEngine::Init` 后，`AK::GetPS4OutputHandle` 不能立即凑效。
- **WG-38778** 已修复：文件头中的块对齐和每样本声道数/位数不一致，导致 Wwise 设计工具和 Convolution Reverb 无法读取部分 WAV 文件。
- **WG-38818** 已修复：有可能将音频文件导入到已卸载的 Work Unit 中。
- **WG-38849** 已修复：在播放父级音乐段落时，假如删除子级音乐片段，就会发生崩溃。
- **WG-38884** 已修复：在将音频设备恢复为 System 或 No\_Output 时，Advanced Profiler 的 Voices Graph 中显示的名称不随之改变。
- **WG-39013** 已修复：在切换桌面时，线程可能会发生泄漏。
- **WG-39018** 已修复：HiDPI 监视器上 Soundcaster 走带的标题栏存在显示问题。
- **WG-39020** 已修复：(iOS) 在 CPU 资源匮乏的情况下，假如更改输出设备，有可能会发生崩溃。
- **WG-39063** 已修复：(PS4) 在启用并行执行时，无法正常进行 ATRAC-9 编码。
- **WG-39074** 已修复：在声音启用了 Listener Relative Routing 和 3D Position Automation 的情况下，发生间歇崩溃。
- **WG-39089** 已修复：在音频包未全部同步时， `CAkActiveParent` 发生罕见崩溃。
- **WG-39092** 已修复：(WAAPI) 在使用 vc140 或 vc150 进行编译时，AkAutobahn 中的错误消息内容出现损坏。
- **WG-39110** 已修复：(Xbox One) 在启用并行执行时，无法正常进行 XMA 编码。
- **WG-39131** 已修复：在按名称监视游戏对象时，假如被监视对象的名称为另一游戏对象的子字符串，则后者可能会同时显示在 3D Game Object Viewer 中。
- **WG-39149** 已修复：在 HiDPI 显示器下使用 Perforce Source Control 插件时，发生崩溃。
- **WG-39149** 已修复：在长时间打开 Wwise 时，Property Help 视图发生崩溃。
- **WG-39175** 已修复：在 Event 包含未设置目标的 Action 时，执行 Event 键盘快捷键命令会引发崩溃。
- **WG-39179** 已修复：在单击 Frame All 按钮时，Game Object 3D Viewer 发生崩溃。
