版本说明 2019.1.7

以下各节列举并阐述了 2019.1.6 和 2019.1.7 版本之间针对 Wwise 所作的改进。有关早期版本的信息，请参阅 [过往版本的发行说明](previousreleases.html) 章节。

- [API 改进](whatsnew_2019_1_7.html#api_changes_2019_1_7)

- [漏洞修复](whatsnew_2019_1_7.html#bugs_2019_1_7)

# 平台 SDK 改进

- **Stadia SDK**: Updated to SDK 1.40.
- **Lumin SDK**: Updated SDK to 0.23.0.
- **PS4 SDK**: Updated SDK to 7.008.001.

# API 改进

- **WG-45935** 添加了相应的选项以便在音频处理中检测无效的非数字 (NaN) 浮点值。该选项可通过 [AkInitSettings::bDebugOutOfRangeCheckEnabled](struct_ak_init_settings_a0ffc62cdbb07d0c8ebb2440fd177625b.html#a0ffc62cdbb07d0c8ebb2440fd177625b "Debug setting: Enable checks for out-of-range (and NAN) floats in the processing code....") 来启用。

# 性能改进

- **WG-46857** 修复了以下性能问题：倘若工程中同时加载了大量 Switch Container，在同一帧内变换多个 Switch 时，有时会出现 CPU 用量峰值。

# 漏洞修复

- **WG-42845** 已修复：在修改 IR Volume 后，Convolution Reverb 没有立即作出响应。
- **WG-44503** 已修复：在 macOS 上，有些 Notes 字段无法在 Wwise 设计工具中正确显示。
- **WG-45963** 已修复：在针对非听者游戏对象调用 SetListenerSpatialization 时没有将该游戏对象注册为听者。
- **WG-46262** 已修复：在外部编辑器中编辑 .wav 文件时可能会发生崩溃。
- **WG-46378** 已修复：在对 API 调用 Sink 初始化消息进行格式化时，有时会导致系统发生崩溃。
- **WG-46386** 已修复：WAAPI 现在已与 NodeJS V12.14.1 LTS 兼容。
- **WG-46396** 已修复：在针对经过音高变换的声音使用 ADPCM 编解码器时发生罕见内存泄漏。
- **WG-46494** 已修复：在执行 Move 操作期间自动重命名 Work Unit 可能会导致其丢失 Add 标记。
- **WG-46558** 已修复：在“耳机声像摆位”模式下实施 3D 空间化处理时可能会将非数字 (NaN) 值用作音频时长。
- **WG-46664** 已修复：(Android) 在运行 Android x86 模拟器时可能会发生崩溃。
- **WG-46713** 已修复：(Switch) 在默认音频设备的声道配置不是立体声或 5.1 时没有声音且可能发生崩溃。
- **WG-46756** 已修复：在启用 Voice Inspector Data 的情况下连接 Wwise Profiler 时，声音引擎可能会发生崩溃。
- **WG-46859** 已修复：在为工程添加平台时，自定义（非共享集）插件可能会出现 ID 冲突。
- **WG-46995** 已修复：Lumin 平台的声音引擎数据包缺少某些文件。
