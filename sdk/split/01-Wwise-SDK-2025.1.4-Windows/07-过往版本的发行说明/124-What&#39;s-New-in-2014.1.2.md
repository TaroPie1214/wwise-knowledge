What's New in 2014.1.2

2014.1.2 属于补丁包。以下各节列举并描述 Wwise 版本 2014.1.1 和版本 2014.1.2 之间的变化。

# 平台 SDK 的变化

- Linux：将构建环境更改为 Steam Runtime 和 GCC 4.8.1
- Xbox One XDK 更新至 2014 年 11 月
- PS4 SDK 更新至 2.000
- PlayStation®Vita SDK 更新至 3.300

# 性能改进

- **WG-26402** 当工程包含大量工作单元时，提高加载速度。

# 漏洞修复

- **WG-26160** 已解决：在创建 RTPC 过渡时发生崩溃，游戏对象在下一个过渡更新前毁坏。
- **WG-26288** 已解决：在发布版本中，OutputDebugMsg 仍然输出内容。
- **WG-26293** 已解决： SoundBank 定义文件中包含效果器可能导致 SoundBank 中包含同名的预置。
- **WG-26293** 已解决：在加载包含预置和相似名称的非预置时错误地报告短 ID 冲突。
- **WG-26316** 已解决：实例限制功能导致在使用 HDR 和虚声部的场景中 CAkListener::ComputeSpeakerMatrix3D() 发生无限循环。
- **WG-26328** 已解决：（iOS）在播放使用 Vorbis 编码 SoundBank 的事件时，ARM64 设备发生崩溃。
- **WG-26338** 已解决：在播放有不同总线父覆盖的多个节点时，更新可能丢失的参数。
- **WG-26341** 已解决：在性能分析器中，使用的命令队列总是超过 100%
- **WG-26346** 已解决：使用无序完成请求的方式加载大文件时流管理器的延迟设备崩溃（2014.1 中的回归）。
- **WG-26347** 已解决：MIDI 片段音符力度当用 MIDI 目标源上的 RTPC 控制时不更新。
- **WG-26353** 已解决：在使用 FM 和 PWM 播放 Sawtooth 时，Synth One: ASSERT (AkSynthOneSimd.inl)。
- **WG-26373** 已解决：内存分配故障导致 ProcessModulators() 崩溃。
- **WG-26376** 已解决：CPU 利用率低时，PS4 输出接收端中存在竞争危害。
- **WG-26382** 已解决：在设计工具中播放时更改卷积混响“类型”导致崩溃。
- **WG-26385** 已解决：（Android）Wwise 在后台时占用 CPU。
- **WG-26386** 已解决：在多人游戏中断开控制器时振动崩溃。
- **WG-26388** 已解决：卸载初始化 SoundBank 时没有移除内置参数绑定，导致重新加载初始化 SoundBank 时出现断言。
- **WG-26397** 已解决：（Windows）设置 AkAPI\_DirectSound 为的 AkInitSettings::eMainOutputType 不工作。
- **WG-26405** 已解决：（VitaSW）ATRAC9 流播放文件不是总能释放流缓冲，导致长源文件发生问题。
- **WG-26411** 已解决：只有在终止声部时，在 Release 版本的库中缺少 Ak\_EndOfEvent 回调。多个声音同时启动，一个声音在启动时被终止，此时可能发生崩溃。
- **WG-26413** 已解决：（Xbox One ADK）安装程序中缺少某些工程和属性表。
- **WG-26413** 已解决：（Xbox One ADK）未（按照 Xbox 要求）启用缓冲区安全检查。
- **WG-26422** 已解决：在使用连接到设计工具的设备实时播放时，MIDI 参数（包括音符力度）对 RTPC 无任何影响。
- **WG-26424** 已解决：（Linux）使用 2 个以上声道播放 48 kHz 音频媒体时产生噪声。
- **WG-26428** 已解决：同时播放并暂停多个声音时出现“Inconsistent source status”错误消息，并且声部被终止。
