版本说明 2016.2.3

以下各节列举并描述Wwise版本2016.2.2与版本2016.2.3之间的变化。

# 新平台

- **WG-30253** （Switch）Wwise 现已支持 Nintendo Switch 主机及其 Opus 声音编码。

# 新功能

- **WG-32545** （Windows）在 Windows 下为 Xbox One 扳机键增加了 Motion（振动）支持。现在已支持 PC 端 UWP（Universal Windows Platform）应用上的 Xbox One Motion Triggers。需要 Windows 10 SDK 10.0.14393 进行构建。

# 平台 SDK 的变化

- **Switch SDK** SDK 版本为 0.12.13。

# 漏洞修复

- **WG-30417** 已解决：SoundBank 生成过于缓慢。
- **WG-31200** 已解决：变更 States 后，Mixing Desk 不显示关联的 State Group 参数值。
- **WG-32274** 已解决：当没有设置 AkPlatformInitSettings.pJavaVM 时导致崩溃。
- **WG-32551** 已解决：当 Multi Editor 视图打开时，多重选择变慢。
- **WG-32597** 已解决：在某些制表符分割导入的情况下在根目录创建的对象。
- **WG-32606** 已解决：在制表符分割导入中将一个对象重复两遍会创建出很多的备份。
- **WG-32617** 已解决：振动音量偏置影响到 HDR 窗口。
- **WG-32776** 已解决：当多重选择时焦点改变。
- **WG-32800** 已解决：FilePackager Sources 缺失
- **WG-32856** 已解决：在 iZotope Hybrid Reverb's Front Stereo 或 Rear Stereo 中设置 Wet 或 Dry Levels 不起作用。
- **WG-32867** 已解决：用连续模式中播放 Blend Containers 时出现音频故障。
- **WG-32934** 已解决：当导入并本地化语音时，Wwise Authoring 崩溃。
