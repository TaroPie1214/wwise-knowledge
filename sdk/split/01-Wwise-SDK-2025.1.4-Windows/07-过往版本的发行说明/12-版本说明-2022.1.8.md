# 新增功能

- **WG-58168** Wwise 工程中现在包含 Perforce 和 Git 版本控制系统的默认 ignore 文件（即 `.p4ignore` 和 `.gitignore`）。

# 性能改进

- **WG-66994** 现在会异步获取 Work Unit 的 Source Control 状态图标。这样可以加快工程加载和部分版本控制操作的速度。而且，Refresh Icons 命令执行得也会更快。
- **WG-67188** 对 3D Audio Bed Mixer 效果器插件的设置进行了调节。现在将 **System-Eligible Audio Object Limit** 的默认值和最大值分别设为了 64 和 512。这样可以大幅降低效果器在实例化时分配的内存。

# 漏洞修复

- **WG-58703** 已修复：在无法保存 Source Control 设置时，日志中没有显示错误消息。
- **WG-60363** 已修复：(Mac Authoring) 在路径包含 Unicode 字符时，无法导入音频文件。
- **WG-66153** 已修复：有时会因为越界计算而无法采用 **Detect from Transients** 自动生成标记。
- **WG-67156** 已修复：在 MIDI 环境下应用于 Voice Volume 以外的属性时，Modulator 更新值可能会出现错误。
- **WG-67216** 已修复：即便没有收到任何 Audio Object，3D Audio Bed Mixer 效果器插件依然会输出 Main Mix 和 Passthrough Mix。
- **WG-67230** 已修复：在加载 Source Control 插件后，Wwise Console 无法正确处理 Ctrl+C 操作。
- **WG-67267** 已修复：有时不会初始化 XMLErrorTranslator 的 s\_reader。

# 社区报告的漏洞修复

- **WG-53851** 已修复：在使用 Voice Inspector 的情况下重新启动 Profiler 会话会引发崩溃。
- **WG-60282** 已修复：“无授权”插件错误没有指向基础平台。
- **WG-65890** 已修复：无法通过 RTPC 和 State Group 控制 Audio Device ShareSet。
- **WG-66295** 已修复：(Perforce) Pending Source Control Operations 对话框中缺少部分图标。
- **WG-66496** 已修复：在生成 SoundBank 的过程中，若有转码结果文件但无原始文件，便不会将转码结果文件包含在 SoundBank 中。
- **WG-66614** 已修复：Query API 函数 `GetListenerSpatialization()` 错误地获取听者索引而非 AkGameObjectID。
- **WG-66845** 已修复：在退出性能分析模式时发生崩溃。
- **WG-66974** 已修复：在实施性能分析时移除 RTPC 曲线会导致 Wwise 发生崩溃。
- **WG-66979** 已修复：缺少用来重新构建 AkImpacter 动态库的示例工程。
- **WG-67005** 已修复：在通过 WAAPI Import 导入本地化 WAV 文件时没有顾及给定对象路径。
- **WG-67076** 已修复：在 Music Switch Container 中的 Music Segment 之间过渡时播放 Stinger 可能会引发崩溃。
- **WG-67270** 已修复：对于之前没有生成的 User-defined SoundBank，SoundBank 元数据文件 SoundBanksInfo.(xml|json) 缺少 Hash 字段。
- **WG-67279** 已修复：由于使用的 AK\_RESTRICT 宏不一致，导致发生链接器错误。
- **WG-67362** 已修复：在连接 Wwise Browser 的情况下选中一个 Project Explorer 会更新所有 Project Explorer。
- **WG-67344** 已修复：在启用和禁用 Background Processing 用户首选项的情况下进行转码时，WEM 文件差异会导致元数据出现差异。对此，可通过在重新生成前删除所有 `.akd` 文件来加以解决。
- **WG-67377** 已修复：在调用 `AK::SoundEngine::CancelCallbackCookie` 时发生死锁。
- **WG-67432** 已修复：在工程设置中将 **Voice LPF and HPF Behavior** 设为 **Use highest value** 时，没有正确应用 RTPC 对 LPF 和 HPF 的更新。
