# 漏洞修复

- **WG-54462** 已修复：在使用轮询频率较高的指针设备时，Wwise 中的窗口拖动操作出现明显的延迟。
- **WG-62763** 已修复：在将对象复制到 Sequence Container 的播放列表中时，Wwise 发生崩溃。
- **WG-62768** 已修复：有可能会将不兼容的对象类型（如 Interactive Music 对象）添加到 Sequence Container 的播放列表。
- **WG-62836** 已修复：在使用 Paste Properties 将对象列表复制到一个或多个目标对象时可能会发生崩溃。
- **WG-62878** 已修复：(Spatial Audio) 在听者靠近 Portal 时，透射路径可能会发生弯曲。
- **WG-62886** 已修复：(Spatial Audio) 在听者或发声体处在 Portal 的边角且开口之间存在很大间隙时，衍射路径可能会丢失。
- **WG-62959** 已修复：在查看 HDR Bus 上的 Voice 时，有时不更新 Voice Inspector 中的影响因素视图。
- **WG-63330** 已修复：在执行 WG-61727 修复后，Wwise 2021.1.11 和 2022.1.1 中出现不符合预期的行为变化。在特定情形下，具有目标过渡且淡变偏置值为正的 Music Switch Container 会受到影响。
- **WG-63495** 已修复：在重新加载一个或多个 Work Unit 后没有清理 Wwise Profiler 视图，并可能因而引发崩溃。
- **WG-63506** 已修复：由于将 g\_pTransitionManager 设为 null，导致在 `CAkPBI::Term` 中终止声音引擎的过程中触发断言。
- **WG-63519** 已修复：在没有打开任何工程的情况下，WAAPI 函数调用迟缓。
- **WG-63769** 已修复：在重新加载的 Work Unit 包含 Wwise 中当前检视的 Music Switch Container 使用的 State Group 或 Switch Group 时发生崩溃。

# 社区报告的漏洞修复

- **WG-58892** 已修复：无法将 Spread 和 Focus 不为零的单声道声音精确摆位到听者后方。
- **WG-60199** 已修复：在使用将字符串作为参数的插件（如 Recorder 插件）时进行连接可能会引发崩溃。
- **WG-60972** 已修复：(Spatial Audio) 在 Portal 之间移动时出现声像摆位问题。
- **WG-61812** 已修复：(Spatial Audio) 在 Listener 和 Emitter 被打开的 Portal 和其他几何构造分隔时总是应用 Room 透射损失。
- **WG-61870** 已修复：在调用的 WAAPI 函数 [ak.wwise.core.object.get](ak_wwise_core_object_get.html) 包含仅由 GUID 组成而无双引号和美元符号的 WAQL 查询时发生崩溃。
- **WG-61907** 已修复：在几何构造包含无效的三角形时发生崩溃。
- **WG-62393** 已修复：Stereo Delay 效果器插件没有将干声增益应用到高度声道。
- **WG-62540** 已修复：在播放由设置了 Delay 和 Sound Instance Limit 的 Random Container 触发并设为 Continuous 的 Blend Container 时，Wwise 可能会发生崩溃。
- **WG-62672** 已修复：在累计 Voice Inspector 数据时，声音引擎出现内存重新分配错误。
- **WG-62780** 已修复：在 Android 上，SetBasePath 为 OBB 文件中的所有位置返回 Ak\_PathNotFound。
- **WG-62791** 已修复：在打开较大的工程时，Perforce Submit 对话框会卡顿很长时间。
- **WG-63034** 已修复：在 Mastering Suite 初始化失败时报告了一般性错误。现在会在遇到相应情形时报告声道配置无效。
- **WG-63088** 已修复：在调用 `AK::SoundEngine::PostMIDIOnEvent()` 时无法释放部分分配的内存。
- **WG-63270** 已修复：在 Reflect 调用内部 Wwise 声音引擎函数时，若 Wwise 音频源发生改变，可能会导致兼容性问题。这些内部函数调用被替换为了对公共 API 的调用。
- **WG-63379** 已修复：在加载工程时显示了 Motion 插件通知。
- **WG-63545** 已修复：在以下情况下调用 UnloadBank 时发生死锁：
  - 在调用 UnloadBank 的过程中以过渡方式停止声音。
  - 在调用 UnloadBank 的同一帧暂停以上所述声音。
- **WG-63749** 已修复：在调用 `AK::SoundEngine::ExecuteActionOnEvent` 时，会针对所有活跃的 MIDI 媒体项执行操作（从 `AK::SoundEngine::PostMIDIOnEvent` 开始），而不管 Event 动作的目标是什么。
