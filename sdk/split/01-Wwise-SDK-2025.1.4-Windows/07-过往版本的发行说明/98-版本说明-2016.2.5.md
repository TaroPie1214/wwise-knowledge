版本说明 2016.2.5

以下各节列举并描述 Wwise 版本 2016.2.4 和版本 2016.2.5 之间的变化。

# 行为变化

- **WG-33961** 现在可以在 Wwise 2016.1.1 和更高版本中读取 2016.1 之前的外部源和流播放 WEM 文件。

# 平台 SDK 的变化

- **Xbox One XDK**：已更新至 XDK to 170301 (2017年3月 QFE 1)。

# 漏洞修复

- **WG-32252** 和 **WG-34024** 已解决：当子对象覆盖到其父级 Blend Container 的同一条输出总线时，Bus Voice（总线声部）属性错误地被应用两次。
- **WG-32484** 已修复：在远程连接时， `CAkPBI::VirtualPositionUpdate()` 发生崩溃。
- **WG-32987** 已修复：(Switch) 在播放到某些文件的结尾时，Opus 解码器出现故障。

  |  |  |
  | --- | --- |
  |  | **备注:** 为了让修复生效，必须手动清除 .cache 文件夹和所生成的 SoundBank（音频包）。 |
- **WG-33393** 已修复：在子对象不沿用父级 Blend Container（混合容器）时，错误地将 Blend Container 子对象属性值添加到了总线值中。
- **WG-33405** 已解决：（Android）在 Android 平台上没有设置 `jNativeActivity` 时，平台可能发生崩溃。
- **WG-33413** 已修复：*Source of Override* 查询中断。
- **WG-33448** 已修复：间歇性地显示本不该显示的 *Inconsistent source status* 错误。
- **WG-33603** 已修复：(iOS) 当调用 [`Suspend`](namespace_a_k_1_1_sound_engine_afc66d31bf5c6425d841ca3510e2d0539.html#afc66d31bf5c6425d841ca3510e2d0539) 时初始化崩溃。
- **WG-33816** 已解决：（Android）被 0 除导致的崩溃。
- **WG-33956** 已修复：若 Random Container（随机容器）或 Sequence Container（序列容器）采用精确到采样点的过渡，而其子容器采用精确到触发率的过渡，则可能会导致 Wwise 崩溃。
- **WG-34020** 已解决：在生成包含音乐对象的 SoundBank 时， Wwise 可能会花很多时间在 SoundBank 生成过程中的‘收集 SoundBank 信息'阶段。
- **WG-34028** 已解决：（PS4）在浸泡测试过程中（超过 12 小时）发生罕见死锁。
- **WG-34207** 已解决：Sink 插件出故障时不恢复到默认的 Sink，导致静音。
- **WG-34226** 添加了用于调试音频缓冲区中无效样本的工具（默认关闭）。
- **WG-34243** 已解决：（Switch）在 Opus 解码开始时，出现性能问题。
- **WG-35118** 已修复：CAkRTPCMgr::SetBuiltInParamValue() 出现罕见崩溃。
