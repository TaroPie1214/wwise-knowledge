版本说明 2015.1.2

以下各节列举并描述 Wwise 版本 2015.1.1 和版本 2015.1.2 之间的变化。

# 新功能

- 在设计工具的 User Preferences 的 Audio System 组合框中现在可选择 WASAPI。
- 可结合 PrepareEvent API 使用的松散媒体文件现在可添加到文件包中。要支持松散媒体文件，使用 File Packager 先前版本创建的文件包项目必须通过 File Packager 应用程序迁移。在加载 SoundBanksInfo.xml 或迁移文件包项目前，请确保已使用 Wwise 2015.1 或更新版本生成 SoundBanksInfo.xml。在 File Packager 中尝试加载 SoundBanksInfo.xml 的旧版本将失败，并导致错误消息。

# 性能变化

- **WG-28130** 现在 WwiseCLI 在生成 SoundBank 时按需加载插件 DLL。
- **WG-28210** 在没有发生游戏更新的每个音频帧处，稍微降低了每游戏对象的 CPU 开销。

# 漏洞修复

- **WG-27694** 已解决：如果干声信号路径低于音量阈值，则不会对辅助发送信号进行滤波。
- **WG-27811** 已解决：（Linux）在连接到 Wwise Authoring 的情况下加载/卸载 SoundBank 时发生崩溃。
- **WG-28036** 已解决：当加载无平台的工程时发生崩溃。
- **WG-28087** 已解决：如果调制器锁到其它调制器，则当关闭声音引擎时发生崩溃。
- **WG-28113** 已解决：（XBox360）在 Xbox 360 上，使用代码编译 Vorbis 库发生故障。
- **WG-28123** 已解决：（Android）在 Java Garbage Collector 传递和应用程序切换后发生崩溃。
- **WG-28133** 已解决：角色混音器音量没有改变振动音量。
- **WG-28148** 已解决：当虚声部行为设置为终止声部时，在声部启动前，交叉淡变过渡将终止声部。
- **WG-28173** 已解决：将 dummy sink 当作主要输出使用将导致死锁。
- **WG-28180** 已解决：当将文件转换为不支持的转码格式时，Wwise Authoring 中发生罕见的崩溃。
- **WG-28186** 已解决：用户首选项：输出缓冲计数未持久保存。
- **WG-28203** 已解决：如果在总线已开始播放后注册扬声器矩阵回调，则总线不会调用该回调。
- **WG-28228** 已解决：WwiseCLI 使用 GUI 消息框报告音频文件缓存错误。
- **WG-28252** 已解决：启动时同时使用循环、精确到采样点和虚声部，造成 Vorbis 崩溃。
- **WG-28261** 已解决：随机/未初始化字节编写于 ATRAC9 文件头填充中。
- **WG-28271** 已解决：如果某个结构已被弃用，则某些媒体项目会被错误地报告为受到 SoundBanksInfo.xml 中的 SoundBank 引用。
- **WG-28280** 已解决：当 SoundBank 卸载后，带有多个 SoundBank 中所含归一化数据的流播放声音可能崩溃。
- **WG-28285** 已解决：如果更改状态组的默认或自定义过渡时间，则不会生成初始化 SoundBank ，并且会报告为已经更新。
