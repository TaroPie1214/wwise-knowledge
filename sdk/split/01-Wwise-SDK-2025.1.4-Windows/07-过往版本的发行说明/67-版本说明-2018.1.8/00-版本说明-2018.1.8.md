版本说明 2018.1.8

以下各节列举并描述 Wwise 版本 2018.1.7 和版本 2018.1.8 之间的变化。有关早期版本的信息，请参阅 [过往版本的发行说明](previousreleases.html) 章节。此版本为漏洞修复版本，因此对所有 2018.1.x 版本来说，工程、音频包和 API 都是兼容的，不会有任何变动。

|  |  |
| --- | --- |
|  | **备注:** **插件开发人员：** 此 Wwise 版本支持通过 Wwise 2018.1 构建的插件。对于通过旧版 Wwise 构建的插件，开发人员必须使用兼容版本重新编译并重新打包。 |

# 平台 SDK 的变化

- **Linux**：已将工具链更新至 Clang 3.8。
- **Lumin**：已更新至 SDK 0.19。
- **Mac, iOS, tvOS**：已更新至 Xcode 10.1。
- **Switch SDK**：已更新到 SDK 7.4。
- **Windows**：已将 Visual Studio 2017 更新至 15.9.12 版本。
- **Xbox One XDK**：已更新至 XDK 10.0.17134.5046 (2018年7月 qfe4)。

# 漏洞修复

- **WG-40466** 已修复：(Switch) 在设备断开时，Wwise Motion 插件发生崩溃。
- **WG-40521** 已修复：在播放 ASIO 源时断开 ASIO 设备会引发崩溃。
- **WG-41214** 已修复：在关闭 Wwise 设计工具时，Transport Control 发生崩溃。
- **WG-41796** 已修复：在某些情况下，Opus 错误地将 1.1 声道音频源转码为立体声。
- **WG-42525** 已修复：(macOS) 在另一应用程序获取蓝牙音频输入时出现音频丢失。
- **WG-42716** 已修复：在播放精确到采样点的层级结构时，假如其子容器采用精确到触发率的过渡，可能会发生崩溃。
- **WG-42807** 已修复：Advanced Profiler 中的 Voices Graph 太宽。
- **WG-42813** 已修复：在 Transport 或 SoundCaster 中移动 RTPC 光标时发生崩溃。
- **WG-42858** 已修复：在有些 Spatial Audio 错误消息中，游戏对象 ID 被截断成 32 位。
- **WG-42860** 已修复：转码会话连续中断会生成大量 Opus 音频转码临时缓存文件。
- **WG-42866** 已修复：对于同时包含 2D 和 3D 衰减声音的 Event，SoundbanksInfo.xml 错误地报告 Max Attenuation Distance。
- **WG-42908** 已修复：在 Release 版本中，假如 Random Container 采用精确到触发率的过渡，在使用 3D Automation 时可能会发生崩溃。
- **WG-43025** 已修复：假如未将输出设备配置为 48 kHz，Microsoft Spatial Sound Platform 输出插件会导致音频发出噼啪杂声。
- **WG-43083** 已修复：对于设为 Continuous 的 Blend Container，假如所含声音活跃于同一 Blend Track 的多个区域，在区域之间也会将声音视为处于活跃状态。
- **WG-43146** 已修复：假如没有在正常速率下触发音频线程，Microsoft Spatial Sound Platform 输出插件会导致音频发出噼啪杂声。   
  已修复：来自 [Microsoft](namespace_microsoft.html) Spatial Sound Platform 输出插件的音频没有发送到正确的输出设备。
- **WG-43149** 已修复：在播放启用了触发速率和 3D 路径的 Random Container 时连接游戏会发生崩溃。
- **WG-43155** 已修复：(Linux) DSP 插件注册问题。
- **WG-43230** 已修复：(PS4) SCE Audio 3D Object Panner 插件发生死锁。
- **WG-43238** 已修复：自定义衰减缺少所有者（Attenuation Editor 中的 Used By 字段）。
- **WG-43271** 已修复：Crankcase REV 中出现除数为零的情况时会导致无限循环。
- **WG-43350** 已修复：在 Wwise 处于连接状态时，已渲染的效果器在声音引擎中生成实例。
- **WG-43412** 已修复：假如有 State Group 在控制 Bypass Effect、Bypass All 或其中一项 Effect 属性，将不会渲染效果器。
- **WG-43440** 已修复：Interactive Music Switch Container 中 Path 的 None 部分被错误地理解为通配符 (\*)。

  |  |  |
  | --- | --- |
  |  | **备注:** 重要说明：若要解决此问题，必须重新生成 SoundBank。 |
- **WG-43459** 已修复：在编辑包含 Post Event 动作的 Event 时，有时会发生崩溃。
- **WG-43496** 已修复：在添加或移除输出设备时触发断言。
- **WG-43550** 已修复：在 State Group 指派设置发生变动时，部分 UI 元素没有更新。
