版本说明 2017.2.6

以下各节列举并描述 Wwise 版本 2017.2.5 和版本 2017.2.6 之间的变化。有关早期版本的信息，请参阅 [过往版本的发行说明](previousreleases.html) 章节。如果使用联机帮助，您还可以从目录列的顶部选择别的 Wwise 版本。此版本为漏洞修复版本，因此对所有 2017.2.x 版本来说，工程、音频包和 API 都是兼容的，不会有任何变动。

# API 变化

- 从现在开始，会检测并拒绝所有无效的游戏对象位置。

# 行为变化

- **WG-38293** 在插件 ID 无效时将报告错误，并拒绝加载插件。

# 漏洞修复

- **WG-36351** 已修复：SpatialAudioTutorial Unity 场景的 `AkRoomPortal` 组件 Baker 窗口引发错误。
- **WG-36402** 已修复：在导入制表符分割的文件时，相对对象路径开头可能包含反斜杠。
- **WG-36981** 已修复：(WAAPI) `ak.wwise.core.object.propertyChanged` 的发布模式有误。
- **WG-37950** 已修复：在听者通过门户从某一房间慢慢进入另一房间时，假如还有其他与后者关联的门户，从多点定位到单点定位的切换就会突然停止。
- **WG-38029** 已修复：在听者通过门户慢慢进入外部空间时，没有正确更新声音传播属性。
- **WG-38098** 已修复：在打开工程并通知 Wwise 重新生成音频包时，报告 ID 存在冲突。
- **WG-38103** 已修复：在按下 Obstruction/Occlusion 设置中的 Reset 按钮时会重置为非默认值。
- **WG-38170** 已修复：AK::SoundEngine::SetScalingFactor 对散布和聚焦都不起作用。
- **WG-38188** 已修复：(WAAPI) 现在支持查询以数字开头的属性名称，如 3DPositionSource。
- **WG-38202** 已修复：在重新加载工程时，不会自动重新启用 Original Audio Files 分组框中的 Override location for current user 选项。
- **WG-38215** 已修复：若在使用 PostMIDIOnEvent 时卸载音频包，则会出现故障。
- **WG-38237** 已修复：在更改音频块大小后，输出总线滤波器可能会变得不稳定。
- **WG-38245** 已修复：SoundBank 生成速度缓慢。
- **WG-38311** 已修复：若音频包包含 MIDI 或 Convolution Reverb 数据，则应用 LoadAndDecode 模式时会出现加载错误。
- **WG-38369** 已修复：在传递无效的游戏对象位置或朝向时，可能会导致 3D 定位发生崩溃。另外，在解决因内存不足而无法注册听者的问题后，也可能会导致 3D 定位发生崩溃。
- **WG-38391** 已修复：在通过门户从某一房间慢慢进入另一空间时，经由门户衍射的发声体的虚声源位置不连续。
- **WG-38405** 已修复：(Android 8.1) 在更新至 Android 8.1 后，AssetManager 出现崩溃。
- **WG-38475** 已修复：在将 [AkRoomParams::RoomGameObj\_AuxSendLevelToSelf](struct_ak_room_params_a07f3e05103b351f449673dfff361aa71.html#a07f3e05103b351f449673dfff361aa71) 设为大于 0 的值时触发断言。
