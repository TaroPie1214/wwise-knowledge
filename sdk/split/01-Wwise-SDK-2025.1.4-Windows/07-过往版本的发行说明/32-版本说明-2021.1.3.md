# 新增功能

- **WG-51853** 在 Profiler Settings 中添加了 **Inactive Game Syncs** 选项，以便捕获所有 Game Sync 并显示在 Game Sync Monitor 中，而无论其是否与播放的声音关联。有关相关改进的详细信息，请参阅以下“行为改进”章节中的 **WG-54345** 和 **WG-54349**。
- **WG-53434** Profiler Filter：添加了新的菜单选项以避免在没有任何性能分析器筛选器处于活跃状态时显示性能分析数据。
- **WG-53504** (wp.py) 为基架增添了 Visual Studio Code 集成支持。

# 行为改进

- **WG-54345** 在实施性能分析时，不再同步工程内对象没有使用的 Game Parameter。
- **WG-54349** 不过对于此小版本，无法将 Wwise 设计工具连接到之前的小版本声音引擎。
- **WG-54770** 改进了 Game Sync Monitor 视图的用户体验。现在允许以简单列表或树视图形式查看 Game Sync。
- **WG-54968** 现在只会在撞击的开头应用 Impacter 中的 RTPC。比如，在循环之后或在声音对象多次触发 Impacter 实例的情况下。
- **WG-55367** SounbanksInfo.xml 现在包含 Switch Group、State Group 及子级 Switch 和 State 的全局唯一标识符 (GUID)。

# 性能改进

- **WG-55096** 提升了 Voice Monitor 在使用筛选器时的性能。

# 其他改进

- **WG-53048** 明确了 Audio Device Editor 中两项 Main Mix Configuration 属性的名称。

# 漏洞修复

- **WG-54122** 已修复：设计工具中发出图像丢失的误报。
- **WG-54484** 已修复：对 Mixing Desk 推子在移动鼠标时的外观实施了改进。
- **WG-54514** 已修复：(Spatial Audio) 在未调用 `AK::SpatialAudio::RemoveRoom()` 的情况下关闭声音引擎会触发断言。
- **WG-54649** 已修复：`AK::StreamMgr::CAkStmTask::GetFileOffset` 中的大文件可能会出现乘法溢出。
- **WG-54772** 已修复：Game Sync Monitor 针对 Game Sync 显示错误的标签。
- **WG-54808** 已修复：(Mac Authoring) 只有先运行 Wwise.app 才能使用启动脚本。
- **WG-54898** 已修复：Performance Monitor 的上下文菜单中缺少 *Save All Counters to File...* 条目。现在已修复并重命名为 \*Save Performance Counters\*。
- **WG-54924** 已修复：在 Play In Editor 期间编辑 AkGeometryComponent 属性时发生崩溃。
- **WG-54996** 已修复：在撤消后没有更新 Steering Panner 的 Z 坐标。
- **WG-55153** 已修复：(Spatial Audio) 在发声体和听者之间的直线路径与 Portal 相交时，即便 Portal 不是发声体和听者之间路径的一部分，也不会应用透射损失。
- **WG-55155** 已修复：对于某些自定义波形文件，Krotos Igniter Live Synth 会发生崩溃。
- **WG-55179** 已修复：在设计工具插件的 `GetBankParameter` 函数中使用 `DataWriter::WriteString` 时总是写入空白字符串。
- **WG-55183** 已修复：在以较小的 Seek Table Granularity 播放长的 Vorbis 声音时出现源匮乏错误。
- **WG-55206** 已修复：CAkAudioMgr::ReserveQueue 偶尔会导致无限循环。
- **WG-55219** 已修复：在使用多点定位并将位置数设为大于 255 的值时出现问题。
- **WG-55234** 已修复：在不必要的情况下刷新 Convolution Reverb Effect Editor 数据。
- **WG-55238** 已修复：在最初添加时，Convolution Reverb Effect Editor 有时不使用冲激文件。
- **WG-55314** 已修复：(WAQL) 在使用 referenceTo 关键字时可能会发生崩溃。
- **WG-55364** 已修复：带有 iZotope Trashbox Modeler 插件的 Wwise 工程在工程加载后被设为未同步状态。
- **WG-55371** 已修复：精确到采样点的 Sequence Container 包含带有效果器的子容器，导致在播放时发生崩溃。
- **WG-55417** 已修复：Impacter 插件包含对 C++ STL 的运行时依赖项。
- **WG-55458** 已修复：Voice Graph 和 Voice Inspector 中没有显示 Audio Device 的 Device ID 编号。
- **WG-55622** 已修复：在使用 Blend Container 时发生罕见崩溃。
