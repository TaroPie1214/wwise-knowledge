# 重要的迁移说明2019.1

|  |
| --- |
| Wwise SDK 2025.1.4 - Windows |

重要的迁移说明2019.1

由于部分 [2019.1 新增功能](whatsnew_2019_1_new_features.html) 的原因，您在迁移到 Wwise 2019.1 时需要注意以下事项。

# ADPCM 编解码器不再有 Encoding Mode 属性

在改用 Platinum Games 提供的 ADPCM 编解码器后，不再支持编码模式。本次迁移将移除 Wwise 工程文件 ADPCM Plugin SharedPropertyList 以及 Work Unit 文件中有关此属性的所有实例。

# WAAPI 日志条目格式

[ak.wwise.core.log.get](ak_wwise_core_log_get.html) 和 [ak.wwise.core.log.itemAdded](ak_wwise_core_log_itemadded.html) 现在会返回 `string` 而非 `integer` 类型的 `messageId`。 [ak.wwise.core.audio.import](ak_wwise_core_audio_import.html) 和 [ak.wwise.core.audio.importTabDelimited](ak_wwise_core_audio_importtabdelimited.html) 现在会在调用失败时返回错误日志条目列表。这些条目可通过细节对象的 `log` 属性以数组形式进行检索。

# 先前的容器历史记录不再有效

之前通过 GetContainerHistory 函数保存的容器历史记录与 Wwise 2019.1 不兼容，因此必须弃用。其他方面，GetContainerHistory 和 SetContainerHistory 函数跟以前没有差别。

# 移除了 Wwise.exe "-CallFromWrapperApp" 参数

之前可通过将 "-CallFromWrapperApp" 参数传递给 `Wwise.exe` 来启用命令行命令。现在移除了该参数，并将所有命令行功能转移到了 `WwiseCLI.exe。`
