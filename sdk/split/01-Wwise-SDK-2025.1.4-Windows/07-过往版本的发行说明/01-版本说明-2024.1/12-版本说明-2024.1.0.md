# 新增功能

- **WG-46778** (WAAPI) 为 WAQL 添加了新的访问器以便访问音频文件信息。
- **WG-54197** 现在会在 Performance Monitor 面板中列出游戏引擎集成提供给 Wwise 但并不由 Wwise 直接管理（比如通过 [AK::SoundEngine::LoadBankMemoryView](namespace_a_k_1_1_sound_engine_a7e8069027779f518a2049dffbb8ecbcb.html#a7e8069027779f518a2049dffbb8ecbcb) 和 [AK::SoundEngine::SetMedia](namespace_a_k_1_1_sound_engine_ae93bba30404755a4d2e592c3bfbf340c.html#ae93bba30404755a4d2e592c3bfbf340c) 管理）的媒体和 SoundBank 的内存用量。另外，由 [AK::SoundEngine::SetMedia](namespace_a_k_1_1_sound_engine_ae93bba30404755a4d2e592c3bfbf340c.html#ae93bba30404755a4d2e592c3bfbf340c) 提供的媒体项现在会将 SoundBank Name 显示为 From Integration/SetMedia 而非 Prepared。
- **WG-59905** 在 Game Object 3D Viewer 中添加了用来显示 Portal 正面的箭头。
- **WG-61638** (Spatial Audio) 为 Acoustics Engine 新增了 Parameter Smoothing 实验功能。在传给声音引擎并实施渲染之前，Parameter Smoothing 会对声学引擎输出的衍射路径参数应用指数移动平均线 (Exponential Moving Average) 平滑处理函数。这些参数包括衍射、透射、声笼和声障以及虚拟定位和增益。通过平滑处理中对其他 Spatial Audio 参数做相对保守的设置以节省 CPU 资源（参见 AkSpatialAudioInitSettings::fSmoothingConstantMs）。
- **WG-64733** Dark 主题现在颜色更深，样式选择更一致，控件也更集中。
- **WG-64979** (WAAPI) 添加了 `ak.wwise.core.profiler.getMeters`、`ak.wwise.core.profiler.registerMeter` 和 `ak.wwise.core.profiler.unregisterMeter` 以便从 Profiler 检索电平表数据。
- **WG-67903** 在 List View 中新增了 Duration Min 和 Duration Max 列。
- **WG-68180** Attenuation Editor 中现在会列出应用了衰减的对象及其 Distance Scaling 和 Diffraction Warping 属性。
- **WG-68263** Attenuation Editor 现在会显示扭曲后的曲线。
- **WG-68690** 更新了 Dark 主题，添加了新的颜色。
- **WG-68997** (Spatial Audio) 在采用 Simplified Reflect 工作流程时，会将 Portal Occlusion 应用于早期反射。
- **WG-69032** 对于 7.1.4 声音和总线，Balance-Fade 声像摆位器现在支持设置 Z 轴。
- **WG-69048** (Spatial Audio) 添加了对几何透射损失做加法和乘法处理的选项。
- **WG-69269** 重写了 Wwise 的默认内存分配器系统，以优化游戏的内存使用、配置和性能分析。现在使用 `AkMemoryArena` 作为默认内存分配器。它产生的内存用量和碎片与大多数通用内存分配器（如 TLSF）相当，同时还允许通过按需预留内存来实现资源的动态扩展。跟之前的默认内存分配器 (rpmalloc) 相比，在很多情况下都可将内存用量降低约 10。跟将 `AkMemSettings::uVMSpanCount` 设为 `AkSpanCount_Small` 时相比，预留内存的水印也略有改善；跟将 `AkMemSettings::uVMSpanCount` 设为 `AkSpanCount_Huge` 时相比，则有显著改善；同时，很少需要调用外部内存分配挂钩。另外，还将之前用于提供内存的回调由 `AkMemSettings::pfAllocVM` 和 `AkMemSettings::pfFreeVM` 更新为了 `AkMemoryArenaSettings::fnMemAllocSpan` 和 `AkMemoryArenaSettings::fnMemFreeSpan`。它们跟之前回调的功能差不多，不过实现起来更加简单。因为其对所提供内存的对齐方式并没有严格的要求，而且只需分配和释放底层部分的内存，无需考虑单独映射物理和虚拟内存。比如，在 Windows 和 POSIX 平台上，`AkMemoryArenaSettings::fnMemAllocSpan` 和 `AkMemoryArenaSettings::fnMemFreeSpan` 的默认实现现在可直接调用 `std::malloc` 和 `std::free` 来获取和释放所需内存（不需要调用 `mmap` 和 `munmap`）。各项内存分配和释放操作的现有回调仍然可用并保持不变。不过，建议最好试着改用新的默认内存分配器，因为现在有了更多的性能分析信息和配置。
- **WG-69272** 在 Advanced Profiler 中添加了新的视图以便直观地了解 AkMemoryArena 在声音引擎运行期间的状态和效率。该视图包含各个活跃 AkMemoryArena 的整体统计数据。比如，所用内存总量、总计预留内存和最大可用空间。另外，其还包含分配给 AkMemoryArena 使用的内存的详细信息。其中包括内存碎片状态的详细信息以及每次调用 `AkMemoryArenaSettings::fnMemAllocSpan` 回调时提供的 UserData 值。
- **WG-69432** (Spatial Audio) 为几何构造实例添加了标记以免被 Portal 移除。
- **WG-69711** 添加了新的 Secondary Editor 以便显示各条总线的总线状态。
- **WG-69861** (Spatial Audio) 添加了一项实验性功能以均衡反射计算的 CPU 用量。`AkSpatialAudioInitSettings::uMaxReflectionPaths` 负责设置最多允许存在多少条经过验证的反射路径。借助启发式计算，系统会对所有潜在路径进行排序，然后确定哪些路径最为突出。在针对所有发声体全局查找 `AkSpatialAudioInitSettings::uMaxReflectionPaths` 后，将不再做任何路径计算。
- **WG-69902** 向 Wwise Runtime API 添加了 `AK::SoundEngine::PrepareBus` 函数。借助此函数，可管理总线上的效果器插件的媒体加载和卸载，而无需将媒体打包到 User-defined SoundBank 中。
- **WG-70275** (WAAPI) 添加了相应的 WAAPI 函数以便编辑 Blend Track 和 Blend Track 指派。这些函数带有 `ak.wwise.core.blendContainer` 前缀。同时，为 `ak.wwise.core.object.get` 函数添加了 WAAPI `blendTracks` 访问器。
- **WG-70355** (WAAPI) 在 WAQL 中添加了 `panner` 访问器以便通过定位来检索声像摆位器对象。
- **WG-70700** 在各个声音的定位参数中添加了新的扭曲值。这些扭曲值可突显或弱化声音所用 Attenuation ShareSet 的声障、声笼、衍射和透射曲线产生的效果。
- **WG-70758** 现在可导入 RF64 或 BW64 格式的 WAV 文件。
- **WG-70877** 为 `ak_wwise_ui_commands_execute` 新增了 `OpenRecycled` 命令。该命令可直接打开对象，而不创建新的窗口选项卡。
- **WG-70879** (Spatial Audio) 现在允许将声音在边缘附近的衍射角度设到 180 度以上。同时，添加了新的初始化设置。通过 `AkSpatialAudioInitSettings::fMaxDiffractionAngleDegrees`，可指定可能的最大衍射角度。若超过此角度，则不计算传播路径并且不可闻。
- **WG-70988** 为 Balance-Fade 视图暴露了 Z 坐标。
- **WG-71394** 替换了四频段滤波器，加快了素材滤波速度。现在改用单个双通道搁架滤波器。跟之前的素材滤波相比，CPU 用量降低了四倍，不过精度稍微低一点。
- **WG-71455** 现在可通过 Contents Editor 中的 **Sources** 按钮执行 Import Audio Files 操作。
- **WG-71582** 在 Positioning 选项卡中新增了 Distance Scaling % 属性。该属性可通过 RTPC 按比例调节 Wwise 设计工具中的最大衰减距离。另外，还对 Attenuation Editor 的界面实施了优化，以包含应用了衰减的对象及其 Distance Scaling % 值。在选中对象并在 Attenuation Editor 中更改 Distance Scaling % 时，可在坐标图中预览做过调节的曲线。除此之外，我们还对 Attenuation Editor 界面做了一些小的改进。比如，对曲线列表进行了归类，并对 UI 代码实施了优化。在 Reflect 中，对于应用了衰减和 Distance Scaling 的镜像声源，会相应地更新其相对于听者的距离。在 Reflect - Effect Editor 中，会在镜像声源列表的 Distance 列和坐标图中的标记位置反映所做的更改。
- **WG-71584** 新增了独立于 Obstruction 和 Occlusion 的 Diffraction 和 Transmission Loss 曲线。
- **WG-71591** 在 Wwise Reflect 中添加了新的高架滤波器，可通过自定义的距离和衍射曲线加以控制，从而以更平滑的频率衰减对这些现象进行建模。
- **WG-71657** 暴露了 `SetMaxReflectionPaths` 函数，以便设置最多可有多少条反射路径。
- **WG-71836** 添加了对 ADM WAV 文件导入的实验性支持。
- **WG-71839** 为 Switch Container 添加了新的编辑器以便将子对象与 Switch 关联。
- **WG-71956** (WAAPI) 新增了 `ak.wwise.core.audio.convert` 和 `ak.wwise.core.audio.setConversionPlugin` 函数。
- **WG-72088** 从 `AK::IAkLowLevelIOHook` 接口中移除了 `BatchCancel()`。
- **WG-72342** PCM 编解码器现在支持 24 位音频。新的 Max Bit Depth 选项允许选择转码为 16 位（默认）或保留为 24 位精度。
- **WG-72569** 现在可将 RTPC 关联到 System Output Setting 的 Mix Behavior 参数。

# API 改进

- **WG-66144** `AK::Query::GetRTPCValue` 不再阻塞。
- **WG-68749** 移除了已弃用的 `UnsetMedia` 函数。请改用 `TryUnsetMedia`。
- **WG-68958** 添加了 Bookmark Allocator 以便声音引擎和插件基于预先分配的内存池创建短时内存分配。在 Advanced Profiler 中监测内存时，Bookmark Allocator 分配具有单独的 MemID。现在推出了新的 `PluginServiceType_TempAlloc` 插件服务，以便于插件利用 Temp Alloc 和 Bookmark Allocator 内存分配系统。同时，将与 Temp Alloc 和 Bookmark Allocator 相关的公共函数和结构从 `AkMemoryMgr.h` 移到了新的头文件 `AkTempAllocDefs.h`。为此，可能需要更新有些文件包含项。
- **WG-69350** 移除了原有的 Wwise Authoring Plug-in API。如需详细了解如何将插件迁移到 21.1 版本中最新引入的 API，请参阅 [Migrating a pre-2021.1 Authoring Plug-in](plugin_21_1_migration.html) 章节。
- **WG-69426** 移除了 `–no-wwise-dat` 命令行参数。不再生成 `wwise.dat` 文件。
- **WG-69446** 将 `AkAudioObject` 和 `AkAudioObjects` 从 `AkCommonDefs.h` 移到了 `AkAudioObject.h`。对于使用这些类型的插件，可能需要更新包含的头文件。同时从 `AkCommonDefs.h` 中移除了 `IAkPluginMemAlloc.h`、`AkArray.h` 和 `AkString.h` 的其他间接头文件包含项。这些头文件包含项可能也要更新。
- **WG-70383** 添加了新版本的 `AK::Wwise::Plugin::PropertySet` 服务。其包含 `AK::Wwise::Plugin::V2::PropertySet::OpenBraces`、`AK::Wwise::Plugin::V2::PropertySet::CloseBraces` 和 `AK::Wwise::Plugin::V2::PropertySet::GetOpenedBraces`。不过并未移除版本 1 中的函数，所以 API 与先前版本完全兼容。虽然未对 [ak\_wwise\_plugin\_host\_object\_store\_v1](structak__wwise__plugin__host__object__store__v1.html) 做任何更改，但更新了 `AK::Wwise::Plugin::ObjectStore` C++ 封装类以同时兼容两个版本。
- **WG-72279** 将用于保存 Wwise 设计工具布局的 XML 文件更新为了新的格式。
- **WG-72574** (Spatial Audio) 移除了已弃用的 `AkGeometryInstanceParams::RoomID`。
- **WG-72735** 移除了用来针对默认内存分配器禁用设备内存堆的配置操作 `AkMemSettings::bEnableSeparateDeviceHeap`。现在可通过将 `AkMemoryArenaSettings::fnMemAllocSpan` 和 `AkMemoryArenaSettings::fnMemFreeSpan` 设为 `nullptr` 来禁用 Device `AkMemoryArena`。
- **WG-72738** 移除了用来针对默认内存分配器做全局内存限制的配置操作 `AkMemSettings::uMemAllocationSizeLimit`。现在每个 AkMemoryArena 都有自己的预留限值 `AkMemoryArenaSettings::uMemReservedLimit`。
- **WG-72875** 移除了 `AK_PERF_RECORDING` 宏。
- **WG-73895** 新的 `AkInitSettings::bOfflineRendering` 成员允许在初始化时实施离线渲染。
- **WG-74114** (Spatial Audio) 添加了新的 API 函数 `AK::SpatialAudio::SetMaxDiffractionPaths`。
- **WG-74230** 将 `SetMaximumReflectionPaths` 重命名为了 `SetMaxGlobalReflectionPaths`。同时，将 `uMaxReflectionPaths` 重命名为了 `uMaxGlobalReflectionPaths`。
- **WG-74496** 向 `IAkMixerPluginContext` 接口添加了 `RegisterAnonymousConfig` 和 `UnregisterAnonymousConfig` 方法，以便在 `ComputePositioning` 中自定义扬声器配置。
- **WG-74721** (Spatial Audio) 将 `AkRoomParams::RoomPriority` 由 AkUInt32 改为了 AkReal32。

# 行为改进

- **WG-61023** `PrepareEvent()` 现在可为 Actor-Mixer Hierarchy 和 Interactive Music Hierarchy 中的对象预备插件媒体。
- **WG-62679** 对 Game Object 3D Viewer 中的摄像机导航控件做了改进。透视摄像机现在支持使用 WSAD 或箭头键实施第一人称导航。为此，弃用了第一人称摄像机以及听者摄像机。
- **WG-66102** 无论是将 Ambisonics 音频源摆位到多声道目标，还是将多声道声源摆位到 Ambisonics 目标，都会对响度做大幅的修正，以便在各种混音场景下获得更加一致的结果。比如，跟将一阶 Ambisonics 音频源摆位到 7.1.4 环绕声总线相比，将五阶 Ambisonics 信号摆位到同一 7.1.4 环绕声总线的音量原本会低 9 dB。现在两种场景下的响度应该差不多。有的时候，可能需要对整体混音做一些修正。不过，大多是为了抵消为解决以前的错误而引入的修正。在实施这一改进的过程中，还修改了将 Ambisonics 混音为立体声输出的方式，以便与将 Ambisonics 摆位到其他声道配置的方式协调一致。
- **WG-68207** 现在在循环播放 ADPCM 文件时可精确到采样点。
- **WG-69416** 现在会在声音引擎检测到部分连续播放产生堆叠时显示 "Number of simultaneous active playback critically high" 错误消息。
- **WG-71672** 在没有连接到远程声音引擎时，现在只有声音对象能直接播放。
- **WG-72072** 在基于对象的配置下，若设为 `AK_EnableBusMeter_KPower`，现在整条总线只会调用一次总线电平测量回调。若将 `AkMeteringFlags` 设为别的值，则每个 Audio Object 都会调用一次。
- **WG-72312** 现在支持将媒体转码为 48 kHz 以上的采样率。另外，将 Max Sample Rate 的设置由 None 改为了新的默认设置 48000。在使用 PCM 和 ADPCM 编解码器时，将 Max Sample Rate 设为 None 的话会保留较高的采样率。
- **WG-72341** 为 Reflect 新增了淡变时间参数，以便在动态添加镜像声源时淡入。同样，在移除镜像声源时会淡出。
- **WG-72540** 将 Auro 9.1 声道配置重命名为了 5.1.4。

# 性能改进

- **WG-61061** 对以下效果器和音频源进行了更新以便使用 Bookmark Allocator：AK Convolution Reverb、Flanger、Guitar Distortion、Harmonizer、Pitch Shifter、RoomVerb、Time Stretch、Stereo Delay、SoundSeed Air Wind、SoundSeed Air Woosh 和 Synth One。这样可以避免这些插件在每帧执行的时候都产生很多临时分配。
- **WG-63811** 在没有源自当前播放声部的输入连接的情况下实施总线处理（比如处理混响效果器的尾音）时，`AK BusTask` 作业在开始执行前不再依赖于 `CAkLEngine::PostRunVpls` 作业。这样在完成 `AK SwVoice` 和 `AK HwVoice` 作业后会继续运行作业辅助函数。如此一来，便可缩短声音引擎时钟周期的总体挂钟时间。
- **WG-67233**将 Gain 效果器由原地处理效果器转换为了原地对象处理器。也就是说，现在只会每条 Audio Objects 总线实例化一次，而不会每个 Audio Object 都实例化一次。另外，现在的处理尽可能只影响 Audio Object 的累积增益，而不是直接将增益应用于对象信号。在将效果器用于 Audio Objects 总线时，这样可以节省一定的内存并大幅提升其性能。
- **WG-70681** 现在会对 WAQL 编译进行缓存。这样在多次执行同样的查询时速度会更快。
- **WG-71033** 提升了 List View 在显示行数和列数较多的查询时的性能。
- **WG-71688** 对反射路径的计算做了优化。在启用路径排序功能时，会对 n 条最优潜在路径进行验证。

# 其他改进

- **WG-63289** 对 Capture Log 中 "Failed Creating Source" 错误的日志记录做了优化。
- **WG-71495** 将 Visual Studio 2022 版本升级到了 17.8.8。
- **WG-72863** 将 `AkPlacementNew` 从 `AkObject.h` 移到了新的文件 `AkPlacementNew.h`。对于使用这些类型的插件，可能需要更新包含的头文件。

- **WG-58722** 已修复：在打开 Multi Editor 的情况下删除很多对象时可能会出现卡顿。
- **WG-59490** 已修复：有可能通过 Project Logs Settings 配置 InvalidEventAction 的严重程度。
- **WG-61985** 已修复：Game Object 3D Viewer 中没有正确显示 Room 名称。
- **WG-64157** 已修复：没有将使用 **New SoundBank** 按钮新建的 SoundBank 放在所选 Work Unit 下。
- **WG-67207** 已修复：在播放 Blend Container 的下级对象时，会将容器的 Blend Track 应用于所播放的声音。
- **WG-68184** 已修复：有时没有将对象设为 Override parent 也会允许修改、复制粘贴或移除其应用的效果器。
- **WG-68401** 已修复：Switch Container 包含不再适用且无法通过 Wwise 配置的属性。
- **WG-68535** 已修复：(Mac Authoring) 在拖动分隔条以至超出最小布局大小时，无法正确裁剪 Game Object Profiler 的 3D 视图。
- **WG-68629** 已修复：在 Unity 中显示声音引擎错误时发生崩溃。
- **WG-68658** 已修复：有时不会在停止播放时发送 `ak.wwise.core.transport.stateChanged` WAAPI 通知。
- **WG-68681** 已修复：在显示模式对话框时有时允许使用浮动视图。
- **WG-68686** 已修复：在多个 Wwise Recorder 插件写入到同一文件时可能会发生崩溃。
