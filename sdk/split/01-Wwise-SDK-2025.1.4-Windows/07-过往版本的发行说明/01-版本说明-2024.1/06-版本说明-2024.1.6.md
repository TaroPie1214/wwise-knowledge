# 新增功能

- **WG-77699** 现在允许使用授权密钥文件名而非授权密钥文本来在命令行中为 [generate-soundbank](ak_wwise_cli_generatesoundbank.html) 指定授权密钥。
- **WG-77994** 现在会在 Wwise 菜单栏中显示工程授权状态。
- **WG-78377** (wp.py) 为 premake 操作新增了 `–signtool-path` 选项，以提供自定义 `SignTool.exe` 可执行文件的完整路径。藉此，可指定跟用于在 Windows 上构建插件的 Windows SDK 提供的可执行文件不同的可执行文件。

# 行为改进

- **WG-77768** 现在会在 Property Editor 中进行搜索时考虑分组名称。

# 漏洞修复

- **WG-77087** 已修复：(Spatial Audio) 在发声体上动态设置多个位置时发生崩溃。
- **WG-77504** 已修复：(Spatial Audio) 在启用 Load Balancing 时既没清理也未执行剩余 Portal 任务。
- **WG-77653** 已修复：在通过列表触发对象检查时可能会发生崩溃。
- **WG-77914** 已修复：Wwise 设计工具中的有些字符串转换会使得线程不安全，进而导致 Impacter 音频文件转码时不时出现故障。
- **WG-78008** 已修复：在修改 Work Unit 文件后重新加载工程可能会引发崩溃。
- **WG-78013** 已修复：无法仅通过加载了媒体的声音来选择 Random Container。必须做到这一点才能提供对 Level-of-Detail 的支持。
- **WG-78252** 已修复：Live Media Transfer 触发的媒体分配缺少 AkMemType\_Media。
- **WG-78335** 已修复：在多线程场景下从 WAV 文件读取 XML 数据时可能会发生崩溃。

# 社区报告的漏洞修复

- **WG-75579** 已修复：(Spatial Audio) 对于包含很多 Portal 且其应用有声笼或声障的复杂地图，Room 遍历算法有时会卡顿好几秒。
- **WG-76642** 已修复：在 Property Editor 中选中 **Override Parent** 时发生崩溃。
- **WG-77327** 已修复：(WAAPI) 无法在 Attenuation 上设置 ConeUse 属性。
- **WG-77386** 已修复：在声部变为实声部后的第一帧紧接着又变为虚声部时可能会发生崩溃。
- **WG-77405** 已修复：(Spatial Audio) `AkAcousticPortal::CalcIntersectPoint()` 触发断言。
- **WG-77591** 已修复：在单击另一对象后，Object Tab 中有时显示错误的编辑器。
- **WG-77692** 已修复：在播放的容器使用了时长很短（20 ms 以内）的 Trigger rate 过渡时，Live Media Transfer 发生崩溃。
- **WG-77775** 已修复：(Spatial Audio) 在 Portal 的衰减小于相对其中心点距离的衰减时，会将该 Portal 后方的声音砍掉并为其指派 100% 的透射损失。
- **WG-77910** 已修复：在计算扬声器矩阵以将 Ambisonics 音频源摆位到 Ambisonics 目标时，`CAkSpeakerPan::GetSpeakerVolumes` 可能会发生崩溃。
- **WG-77911** 已修复：在多 3D 对象配置下使用五阶 Ambisonics 音频源时出现堆栈溢出。
- **WG-77930** 已修复：在 Audio Bus 同时应用有启用了 **Process LFE** 的 Parametric EQ 效果器和别的用来处理尾音的效果器时，音频管线中可能会生成 INF 或 NaN 值。
- **WG-77943** 已修复：在将很多对象拖到 Mixing Desk 中时，用户界面会卡顿很长时间。
- **WG-78011** 已修复：在处理带发声体声道掩码的音频对象时，`CAkSrcLpHpFilter::ExecuteSrcLpHpFilters` 发生崩溃。
- **WG-78058** 已修复：在使用 Mixing Desk 时重新加载 Work Unit 可能会引发崩溃。
- **WG-78266** 已修复：在启用浮点异常时，AkConvolutionReverb 发生崩溃。
- **WG-78271** 已修复：在删除效果器时可能会发生崩溃。
- **WG-78341** 已修复：在执行包含 "Source/Channels (Original)" 或 "Source/Channels (Converted)" 的查询时，不会返回与 Ambisonics 或匿名音频源相关的结果。
- **WG-78359** 已修复：在执行 Live Media Transfer 操作时，声音引擎可能会发生死锁。
- **WG-78440** 已修复：对于有些采用 48 kHz 以外的采样率并将循环点放在文件末尾附近或在 Original 文件中指定循环点的 ATRAC9 媒体，在实施转码时出现错误。
- **WG-78443** 已修复：在效果器实例持续执行 24 小时以上的处理后，AK Convolution 插件的性能严重下降并出现处理错误。

# 文档改进

- **WG-75508** 在[使用 Object Tab 和 Object Tab Group](https://www.audiokinetic.com/library/edge/?source=Help&id=working_with_object_tabs) 和[了解 Object Tab Group 的标题栏](https://www.audiokinetic.com/library/edge/?source=Help&id=understanding_object_tab_title_bar)中添加了有关 **Open New Tab on Double-Click** 选项的信息并对 Change Selection Channel 菜单中的选项做了详细的说明。
- **WG-78118** 在[使用 HDR](https://www.audiokinetic.com/library/edge/?source=Help&id=using_hdr) 中明确了如何启用 HDR 以及相关注意事项对子总线来说意味着什么。
