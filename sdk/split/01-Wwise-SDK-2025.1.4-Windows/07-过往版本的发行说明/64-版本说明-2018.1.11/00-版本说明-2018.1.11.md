版本说明 2018.1.11

以下各节列举并阐述了 2018.1.10 和 2018.1.11 版本之间针对 Wwise 所作的改进。有关早期版本的信息，请参阅 [过往版本的发行说明](previousreleases.html) 章节。此版本为漏洞修复版本，因此对所有 2018.1.x 版本来说，工程、音频包和 API 都是兼容的，不会有任何变动。

# 平台 SDK 改进

- **Stadia SDK**: Updated to SDK 1.40.
- **Switch SDK**：已更新到 SDK 9.3.1。
- **PS4 SDK**: Updated SDK to 7.008.001.

# 新增功能

- **WG-45918** Stadia：添加了对语音聊天的支持。

# 其他改进

- **WG-45147** 在 MAC 上添加了 CHM 文档。

# 漏洞修复

- **WG-45121** 已修复：现在可通过同时拖放多个 Music Event 提示来将其放在所属段落之外。
- **WG-45681** 已修复：在使用流播放声音准备 Event 时过早地触发了 I/O 操作。
- **WG-45726** 已修复：在混合播放 Random Container、Sequence Container 和包含 3D 定位路径的 Blend Container 时触发失败断言。
- **WG-46378** 已修复：在对 API 调用 Sink 初始化消息进行格式化时，有时会发生崩溃。
- **WG-46396** 已修复：在针对经过音高变换的声音使用 ADPCM 编解码器时发生罕见内存泄漏。
- **WG-46558** 已修复：在“耳机声像摆位”模式下实施 3D 空间化处理时可能会将非数字 (NaN) 值用作音频时长。
- **WG-46664** 已修复：(Android) 在运行 Android x86 模拟器时可能会发生崩溃。
- **WG-46713** 已修复：(Switch) 在默认音频设备的声道配置不是立体声或 5.1 时没有声音且可能发生崩溃。
- **WG-46976** 已修复：损坏的 SoundBank 可能会导致 Profile 和 Release 配置故障。
