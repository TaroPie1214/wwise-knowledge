版本说明 2019.1.4

以下各节列举并描述 Wwise 版本 2019.1.3 和版本 2019.1.4 之间的变化。有关早期版本的信息，请参阅 [过往版本的发行说明](previousreleases.html) 章节。

- [新功能](whatsnew_2019_1_4.html#feature_changes_2019_1_4)

# 新功能

- **WG-43747** (Stadia) 添加了对 Motion 的支持。
- **WG-44670** (Stadia) 添加了对 Google Resonance 的支持。

# 漏洞修复

- **WG-41040** 已修复：在删除 Game Sync Monitor 中固定的 RTPC 时可能会发生崩溃。
- **WG-44006** 已修复：在运行多个 WwiseCLI 并发实例时出现 .cache 共享问题。
- **WG-44193** 已修复：在按照 Value 列标题进行排序时，Batch Rename 视图发生崩溃。
- **WG-44476** 已修复：在使用 [AK::SoundEngine::SetListenerSpatialization](namespace_a_k_1_1_sound_engine_a3e3e9e5335df01fca37f9b3ed518e2bd.html#a3e3e9e5335df01fca37f9b3ed518e2bd) 时，3D Spatialization 设为 None 的声音会淡出。
- **WG-44498** 已修复：在关闭工程时可能会发生崩溃。
- **WG-44509** 已修复：在 Transport Control 中播放 Interactive Music 段落时，设计工具发生罕见崩溃。
- **WG-44647** 已修复：若启用了多核渲染，则在精确到采样帧的过渡中播完声音时可能会发生崩溃。
- **WG-44698** 已修复：在 Continuous 模式下使用 Blend Container 播放 Transition Type 设为 Trigger Rate 的连续容器时可能会发生崩溃。
- **WG-44784** 已修复：在 Property Editor 的 Effects 选项卡中对效果器进行重新排序后出现各种问题。
- **WG-44798** 已修复：在一开始为虚声部的声部首次变为实声部时，出现音量峰值。
- **WG-44811** 已修复：(Spatial Audio) 若使用 Portal 连通两个并非完全平齐的 Room，则有可能会在短时间内将对象指派给错误的 Room，从而导致声音出现不连贯问题。
- **WG-44834** 已修复：若在针对声音引擎初始化期间生成的主输出设备调用 RemoveOutput 后调用 AddOutput 来创建新的输出设备，则音频会含混不清或发出噼啪噪声。
- **WG-44913** 已修复：(Spatial Audio) 若在调用 SetRoom/SetPortal 时场景中没有要处理的几何构造，则会出现 CPU 用量峰值。
- **WG-44918** 已修复：(WAAPI) 在使用 Override parent 时，访问 UserAuxSends 属性时无法返回正确的值。
- **WG-44938** 已修复：FilePackager 为 Lumin 和 Stadia 生成无效的文件包。
- **WG-44947** 已修复：WAV 文件中的标记文本有时会错误地显示为 base64 编码格式。
- **WG-45046** 已修复：(Android) 基于 Kirin 980 芯片组的华为手机上出现音频卡顿。
