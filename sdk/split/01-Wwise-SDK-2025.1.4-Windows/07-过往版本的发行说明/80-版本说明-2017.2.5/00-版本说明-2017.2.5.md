版本说明 2017.2.5

以下各节列举并描述 Wwise 版本 2017.2.4 和版本 2017.2.5 之间的变化。有关早期版本的信息，请参阅 [过往版本的发行说明](previousreleases.html) 章节。如果使用联机帮助，您还可以从目录列的顶部选择别的 Wwise 版本。此版本为漏洞修复版本，因此对所有 2017.2.x 版本来说，工程、音频包和 API 都是兼容的，不会有任何变动。

# 平台 SDK 的变化

- **Switch SDK:** SDK 已更新至 4.5.0.
- 添加了对 Magic Leap (Lumin OS) 平台的实验性支持。

# 漏洞修复

- **WG-37393** 已修复：在双击 Randomizer 图标以便查看属性时，有时无法弹出 Randomizer 视图。
- **WG-37564** 已修复：在向 RoomVerb 效果器应用奇数数量的采样点时可能会出现挂断/故障情况。
- **WG-37587** 已修复：部分 WAAPI 示例不再适用。
- **WG-37609** 已修复：(PS4) AkLock 使用未初始化的内存。
- **WG-37617** 已修复：Random Container 的 Trigger Rate Duration 会忽略 State。
- **WG-37733** 已修复：在性能分析处于非活动状态时不会更新 Game Object 静音/独奏按钮。
- **WG-37746** 已修复：Convolution Reverb 和非二次幂音频缓冲区帧计数存在问题。
- **WG-37768** 已修复：(PS4) 在执行 SCE Audio3d 回归分析时，对象位置无法根据听者朝向调整。
- **WG-37785** 已修复：(PS4) 在添加第二个设备时，可能存在竞争条件和崩溃情况。
- **WG-37819** 已修复：若在播放 MIDI 内容的过程中卸载引用该内容的音频包，则可能会出现崩溃。
- **WG-37895** 已修复：根据崩溃报告随机修复 Property Editor。
- **WG-37942** 已修复：无法导入文件或文件夹以进行本地化。
- **WG-37943** 已修复：在修改总线层级结构后连接时， `CAkBus::GetDeviceShareset` 发生崩溃。
- **WG-37958** 已修复：(iOS) 若在已连接蓝牙设备的情况下启动声音引擎，则可能会出现卡顿。
- **WG-37991** 已修复：总线无法发送至自身的其他实例（影响 Wwise Spatial Audio Room 和 Portal）。
- **WG-38110** 已修复：为通往同一房间的多个门户指派了错误的湿衍射值，导致“房间”游戏对象的声障值也不正确。
