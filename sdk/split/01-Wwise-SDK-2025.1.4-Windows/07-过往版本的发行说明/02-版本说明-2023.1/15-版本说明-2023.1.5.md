- **WG-72404** (Spatial Audio) 通过移除冗余路径对湿声路径衍射计算做了优化。

# 漏洞修复

- **WG-70816** 已修复：(Spatial Audio) 在 Game Object 3D Viewer 中显示 Portal 信息时，无法正确显示括号中所示的 Room ID。
- **WG-71908** 已修复：在重新打开工程时，Switch 的对象颜色丢失。
- **WG-72028** 已修复：在部分平台上使用有些自定义内存分配挂钩时，因为使用了带有标记的指针而导致 AkBusCallbackMgr 发生崩溃。
- **WG-72104** 已修复：在内存不足的情况下无法正确清理 Prepared Game Sync。
- **WG-72280** 已修复：在 `StreamMgr` 出现争用问题时可能会发生崩溃。
- **WG-72319** 已修复：(WwiseConsole) 无法解析相对工程路径。
- **WG-72362** 已修复：(Spatial Audio) 远处的 Portal 可能会对 Reverb Zone 过渡产生干扰。
- **WG-72772** 已修复：在播放 FuMa Ambisonics 文件时，Wwise Wave Viewer 发生崩溃。

# 社区报告的漏洞修复

- **WG-67917** 已修复：在处理的任务过多时出现死锁。
- **WG-71752** 已修复：(Spatial Audio) 在 Wwise 2023.1 中，由于衍射点的计算方式不同，衍射路径查找算法会占用较多 CPU 资源。在 2023.1 中，路径被限制在角落处。这样在发声体或听者进入或走出受影响区域时会产生平滑的淡入或淡出。虽然音频品质提高了，但 CPU 用量也增加了。现在只有在衍射点与边缘末端最多相距一个边缘长度时才会将路径限制在角落处。藉此，可将部分场景下的 CPU 用量降低 15-20，同时还能确保平滑的音频输出。
- **WG-71821** 已修复：在选择使用 MIDI 文件的音乐对象时，Wwise 出现卡顿。
- **WG-71951** 已修复：在启用负荷均衡的情况下，Room 和 Portal 发生崩溃。
- **WG-71971** 已修复：(Spatial Audio) 在以 Distance Probe 的形式进入 Portal 时，房间底噪发生跳变。
- **WG-72060** 已修复：Windows 平台上出现争用问题，导致在终止 Sound Engine 的过程中 Windows 回调发生崩溃。
- **WG-72086** 已修复：在加载工程时无法正确同步对象 3D 定位自动化属性。
- **WG-72095** 已修复：在 Attack Time 或 Release Time 过短时，Mastering Suite 的 Multiband Compressor 会生成杂音。
- **WG-72100** 已修复：对 I/O 挂钩 `Close` 函数的调用发生延迟。这会导致文件保持打开状态的时间过长。无需更改 I/O 挂钩代码。
- **WG-72101** 已修复：在使用 Job Manager 同时执行多项 Spatial Audio 任务时发生崩溃。
- **WG-72381** 已修复：在使用 Opus 或 Vorbis 的情况下调节 Pitch 时，5.1 声音存在 Right 和 Center 声道互换的情况。
- **WG-72578** 已修复：在 `AK::HashTable::RemoveIf` 存在逻辑错误时，可能会多次访问哈希表中的条目。

# 文档改进

- **WG-68715** 对离线渲染相关的文档内容做了更新以展示如何在捕获渲染的音频前设置新的输出设备声道配置。
- **WG-71448** 明确了诸多场景下 Convolution Reverb 输入信号的来源。
