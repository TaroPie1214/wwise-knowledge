# 新增功能

- **WG-70303** 为 Wwise Console 的 `generate-soundbank` 动作添加了 `use-user-overrides` 选项。此选项允许在通过 Wwise Console 生成 SoundBank 时使用基于用户的 Override 设置（如自定义 SoundBank 路径）。跟之前一样，在通过 Wwise Console 生成 SoundBank 时默认只使用常用的 Project Settings。
- **WG-70475** 现在支持启用环境变量。类似于在 Project Settings 中定义 Original Audio Files、Cached Audio Files 和 External Sources Output Paths 的文件夹路径的 Soundbank Generation Step Commands。同时添加了新的对话框以便于定义这些路径。

# 其他改进

- **WG-70494** 现在支持通过 Visual Studio 2022 来构建 Level 3 Source Code。

# 漏洞修复

- **WG-55590** 已修复：Capture Log 中的 Spatial Audio API 调用信息不充分。
- **WG-65549** 已修复：在接连打开并关闭工程时出现残留 Directory Watcher 线程。
- **WG-67996** 已修复：在导入多条音轨或导入短的音频文件时，没有自动调节 Music Segment 的 Exit Cue 的位置。
- **WG-68199** 已修复：在更改子级 Sequence Container 的播放列表时，没有更新 Sequence Container 的播放列表。
- **WG-68243** 已修复：有时不显示 Error Reporter 中输入的客户信息（名称、电子邮件、公司）。
- **WG-68268** 已修复：(Spatial Audio) Room Transmission Loss 被错误地应用于穿过两个 Room 之间间隙的听者。
- **WG-69203** 已修复：(Mac Authoring) 在编辑文本字段时没有更新文本。
- **WG-70415** 已修复：(Spatial Audio) 在通过 Portal 进行过渡时，源自 Reverb Zone 过渡区的路径有时无法淡入或淡出。
- **WG-70481** 已修复：无法在多个视图之间正确同步 Mastering Suite Master Volume 模块的 **Link Channels** 按钮。
- **WG-70505** 已修复：在所复制的列表行包含重新排序的列时可能会发生崩溃。
- **WG-70521** 已修复：在使用长的文本字符串筛选 Capture Log 时可能会发生崩溃或死机。
- **WG-70548** 已修复：(Spatial Audio) 倘若 Reverb Zone 过渡区与 Portal 过渡区重叠，在听者离开 Reverb Zone 并通过 Portal 进行过渡时，Reverb Zone 会淡出然后又突然切回。
- **WG-70566** 已修复：(Spatial Audio) `SearchStochasticPath()` 断言无效。
- **WG-70618** 已修复：在更改 Original Audio Files 和 Cached Audio Files 路径配置后没有自动重新加载工程。
- **WG-70646** 已修复：(Spatial Audio) 在游戏对象离开处在两个 Room 之间孔隙位置的 Portal 时触发断言。
- **WG-70672** 已修复：(Spatial Audio) 在 Portal 后方的相邻 Room 中播放房间底噪时，没有在 Reverb Zone 过渡区计算 Room 透射损失。
- **WG-70717** 已修复：在 Performance Monitor Settings 中，单击字段时不会选中其中的所有文本。
- **WG-70770** 已修复：在基于 ARM 的 CPU 上以 Debug 模式运行时，AkRingBuffer 断言失败。
- **WG-70812** 已修复：在调整子对象位置并回归原位后，无法在 Wwise 设计工具中正常播放新建的 Random Container。
- **WG-70874** 已修复：在退出 `WaveViewer.exe` 时发生崩溃。
- **WG-70989** 已修复：在播放过程中可能无法反映对 Wwise 设计工具中的总线层级结构所做的更改。
- **WG-71069** 已修复：(Spatial Audio) 在 Reverb Zone 环绕包含听者并通过 Portal 连通的 Room 时，为 Reverb Zone 计算的 Spread 和 Diffraction 值不正确。
- **WG-71089** 已修复：没有在 Wwise 设计工具中立即应用对 Stinger 或 Transition 所做的更改。
- **WG-71096** 已修复：在使用“浏览”按钮选择路径后，User SoundBank Settings 的 External Sources 选项卡中的 **OK** 按钮不可用。

# 社区报告的漏洞修复

- **WG-63424** 已修复：在使用滑杆控件时可能会发生崩溃。
- **WG-68493** 已修复：在加载缺少 Convolution Reverb WAV 文件的工程时，Project Load Log 条目包含重复的 IRConvolutionStop 属性值。
- **WG-69622** 已修复：User SoundBank Settings 中覆盖的 SoundBank 路径指向同一平台。
- **WG-69663** 已修复：在与文本字段进行交互时可能会发生崩溃。
- **WG-70292** 已修复：无法将 RTPC 关联到 [AK](namespace_a_k.html "Definition of data structures for AkAudioObject") Channel Router Metadata 的 Channel 参数。
- **WG-70515** 已修复：在尝试访问 AudioFileSource 的状态属性时，WAAPI 发生崩溃。
- **WG-70518** 已修复：(Spatial Audio) 在单独调用时，SetReverbZone API 无法设置未同步标记。这可能会导致 Reverb Zone 的 Spread 和 Diffraction 值不正确。
- **WG-70586** 已修复：在连续执行 Harmonizer 插件几分钟后死机。
- **WG-70643** 已修复：在重新加载 Work Unit 时，Music Switch 条目获取了新的 ID。
- **WG-70692** 已修复：在启用  `AK_EnableGetSourceStreamBuffering` 的情况下渲染虚声部时发生崩溃。
- **WG-70728** 已修复：在将 WAV 文件数据存储到缓存中时，Wwise 设计工具发生崩溃。
- **WG-70738** 已修复：在处理音量较大的输入或在高比率下运行时，Compressor 和 Expander 效果器插件抛出浮点异常或给出不确定的结果。
- **WG-70838** 已修复：淡入、淡出和定位自动化的过渡时间不准确。过渡时长最多会跟过渡长度相差 6。
- **WG-71083** 已修复：Loudness Normalization 在 Interactive Music Hierarchy 中无法正常运行。
- **WG-71095** 已修复：Pin/UnpinEventInStreamCache 调用失衡会导致在终止声音引擎时发生死锁。

# 文档改进

- **WG-69244** 在 [Voice Inspector](https://www.audiokinetic.com/library/edge/?source=Help&id=voice_inspector) 中添加了对 Normalization 驱动因素的描述。
- **WG-69566** 在[管理外部音频编辑器](https://www.audiokinetic.com/library/edge/?source=Help&id=user_preferences#selecting_external_audio_editors)中添加了有关 External Editor Start Mode 选项的信息。
- **WG-70375** 对 IAkLowLevelIOHook 文档进行了改进。
- **WG-71118** 对整个 Wwise Help 的内容做了调整，减少了页面数量并增加了每页的信息。比如，[使用预设](https://www.audiokinetic.com/library/edge/?source=Help&id=using_presets)章节现在只有一页而不是六页。
