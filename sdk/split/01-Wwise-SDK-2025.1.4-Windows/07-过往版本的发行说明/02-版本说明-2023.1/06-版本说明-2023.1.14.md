# 新增功能

- **WG-77699** 现在允许使用授权密钥文件名而非授权密钥文本来在命令行中为 [generate-soundbank](ak_wwise_cli_generatesoundbank.html) 指定授权密钥。
- **WG-77994** 现在会在 Wwise 菜单栏中显示工程授权状态。
- **WG-78377** (wp.py) 为 premake 操作新增了 `–signtool-path` 选项，以提供自定义 `SignTool.exe` 可执行文件的完整路径。藉此，可指定跟用于在 Windows 上构建插件的 Windows SDK 提供的可执行文件不同的可执行文件。

# 漏洞修复

- **WG-77914** 已修复：Wwise 设计工具中的有些字符串转换会使得线程不安全，进而导致 Impacter 音频文件转码时不时出现故障。

# 社区报告的漏洞修复

- **WG-75579** 已修复：(Spatial Audio) 对于包含很多 Portal 且其应用有声笼或声障的复杂地图，Room 遍历算法有时会卡顿好几秒。
- **WG-77327** 已修复：(WAAPI) 无法在 Attenuation 上设置 ConeUse 属性。
- **WG-77386** 已修复：在声部变为实声部后的第一帧紧接着又变为虚声部时可能会发生崩溃。
- **WG-77775** 已修复：(Spatial Audio) 在 Portal 的衰减小于相对其中心点距离的衰减时，会将该 Portal 后方的声音砍掉并为其指派 100% 的透射损失。
- **WG-77910** 已修复：在计算扬声器矩阵以将 Ambisonics 音频源摆位到 Ambisonics 目标时，`CAkSpeakerPan::GetSpeakerVolumes` 可能会发生崩溃。
- **WG-77930** 已修复：在 Audio Bus 同时应用有启用了 **Process LFE** 的 Parametric EQ 效果器和别的用来处理尾音的效果器时，音频管线中可能会生成 INF 或 NaN 值。
- **WG-78011** 已修复：在处理带发声体声道掩码的音频对象时，`CAkSrcLpHpFilter::ExecuteSrcLpHpFilters` 发生崩溃。
- **WG-78058** 已修复：在使用 Mixing Desk 时重新加载 Work Unit 可能会引发崩溃。
- **WG-78341** 已修复：在执行包含 "Source/Channels (Original)" 或 "Source/Channels (Converted)" 的查询时，不会返回与 Ambisonics 或匿名音频源相关的结果。
- **WG-78443** 已修复：在效果器实例持续执行 24 小时以上的处理后，AK Convolution 插件的性能严重下降并出现处理错误。

# 文档改进
