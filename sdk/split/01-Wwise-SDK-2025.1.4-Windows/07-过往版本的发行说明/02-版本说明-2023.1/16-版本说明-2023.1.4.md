# 新增功能

- **WG-68770** 现在支持使用事先定义的 `AK_NULL_PLATFORM` 宏来编译某些公用 SDK 头文件。这样在平台上尝试编译应用程序而其声音引擎不受 Wwise 支持或必要的文件暂不可用时可以大大减少编译器错误。

# 行为改进

- **WG-69971** 为了支持将来的补丁，我们对 Wwise 设计工具用来分析运行时声音引擎的通信协议架构做了修改。不过，这会破坏小的补丁版本之间通信的向后兼容性。也就是说，Wwise 2022.1.13 之前的设计工具将无法连接到运行 Wwise 2022.1.13 及之后声音引擎的游戏。不过，反过来是可以的。比如，将 22.1.13 设计工具连接到运行 22.1.xx 声音引擎的游戏。同样的逻辑也适用于 23.1.4 版本。

注意，SoundBank 内容的架构并没有做修改，所以在这些版本之间还是兼容的。

- **WG-71327** 从 Advanced Profiler 的 **Busses** 选项卡中移除了 **Mixer** 列。

# 性能改进

- **WG-70930** (Spatial Audio) 对 Spatial Audio Emitter 的更新进行了优化。

# 其他改进

# 漏洞修复

- **WG-68844** 已修复：在关闭 Wwise 时可能会发生崩溃。
- **WG-69988** 已修复：控制器命令 Pass MIDI Note 不起作用。
- **WG-70365** 已修复：在更改在播对象上的 State 属性时发生崩溃。
- **WG-70636** 已修复：在执行重命名操作后没有应用 Effect ShareSet。
- **WG-70637** 已修复：在自定义测量对象弹出列表中的列时发生崩溃。
- **WG-70662** 已修复：在选中 Cue Name 字段时删除 Stinger 行和撤消对其所做的操作会引发崩溃。
- **WG-70670** 已修复：在显示 Checkout 的 Perforce 对话框时，按下了 ESC 键却没有撤消相应操作。
- **WG-70723** 已修复：在无法解析对象 ShortId 时，Voice Inspector 发生崩溃。
- **WG-70726** 已修复：在使用 Capture Log 时可能会发生崩溃。
- **WG-71046** 已修复：在切换视图时没有保存浮动视图的 Z 排序。
- **WG-71091** 已修复：(Spatial Audio) 在 Reverb Zone 过渡与父级 Room 中的 Portal 重叠时会跳转到 Room Aux Send。
- **WG-71142** 已修复：音频源的响度测量为后置和高度声道应用了 +1.5 dB 的增强，但实际上只应为旁置声道应用这样的增强。
- **WG-71255** 已修复：在删除效果器时可能会发生崩溃。
- **WG-71621** 已修复：在 Metadata Editor 处于打开状态时将自定义 Metadata 转换为 ShareSet 可能会引发崩溃。
- **WG-71778** 已修复：Convolution Reverb IR 统计数据窗口包含无效的值。
- **WG-71843** 已修复：在卸载的 Work Unit 包含具有多个 Path 条目的 Music Switch Container 时发生崩溃。

# 社区报告的漏洞修复

- **WG-68118** 已修复：Crankcase Audio REV 插件的 Loop 编辑器不遵从 Windows 鼠标辅助设置。
- **WG-69217** 已修复：Game Parameter 值 [-10000, 10000] 的取值范围限制太严格了。现已将取值范围改为 [-1000000000, 1000000000]。
- **WG-69228** 已修复：(Spatial Audio) 在没有衍射路径时，Diffraction 内置参数会将其 RTPC 值设为 0 而非 100。
- **WG-69515** 已修复：在 Crankcase Audio REV 插件中，Mix Sensitivity 选项的默认值不正确。
- **WG-70306** 已修复：在请求获取 Perforce 文件状态时发生崩溃。
- **WG-70581** 已修复：在使用实验性的 Live Media Transfer 时重新连接 Profiler 会引发崩溃。
- **WG-70748** 已修复：在 SoundBanks Settings 的 SoundBanks Paths 分区双击 **Platform** 列中的单元格时可能会发生崩溃。
- **WG-70952** 已修复：在 SoundCaster 会话对象上设置颜色后无法加载工程。而且，没有保存混音会话的颜色。
- **WG-71088** 已修复：(Spatial Audio) 在单个 Room 连有很多 Portal 时性能出现下降。
- **WG-71127** 已修复：无法加载数据块大小不是 1 的文件包中存储的 SoundBank。
- **WG-71135** 已修复：在音轨被加载到 Multi-Editor 中的情况下修改 Music Track 的 Switch 或 State 会引发崩溃。
- **WG-71263** 已修复：衍射射线分配被错误地标记为 `AkMemID_Profiler`（不过在 Release 版本中需要这样标记）。
- **WG-71279** 已修复：在将 Music Track 转换为 Sub-Track 时可能会发生崩溃。
- **WG-71369** 已修复：在高比率下运行时，Compressor 和 Expander 效果器插件抛出浮点异常。
- **WG-71418** 已修复：(Spatial Audio) 在禁用虚声源定位时，有时会将无效的声音位置传给声音引擎。
- **WG-71610** 已修复：在将 Wwise ASIO 插件集成到 Unity 工程中时，在初始化过程中发生崩溃。
