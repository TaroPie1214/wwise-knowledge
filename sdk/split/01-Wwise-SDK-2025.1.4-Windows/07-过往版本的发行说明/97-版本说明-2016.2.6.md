版本说明 2016.2.6

以下各节列举并描述 Wwise 版本 2016.2.5 和版本 2016.2.6 之间的变化。

# 平台 SDK 的变化

- **PS4 SDK:** PS4 VSI integration minimal support is now Visual Studio 2015.
- **Switch SDK:** 已更新至 SDK 3.5.1.

# 漏洞修复

- **WG-34210** 已解决：当连通单声道到 LFE 总线并且 2D 定位启用声像摆位器时，发生内存损坏。
- **WG-34226** 已解决：Harmonizer Effect 在一定条件下可能在音频信号中生成非法数值。
- **WG-34237** 已解决：（PS4）当使用多声道压缩器远程连接到游戏时，发生内存损坏。
- **WG-34284** 已解决：（Android）在特定硬件（mobiistar, Nova, Play Spark）初始化时出现被零除。
- **WG-34349** 已解决：（PS4）在创建硬件实例失败后发生断言。
- **WG-34370** 已修复：(Xbox One) 假如之前为音频包调用了 DecodeBank，会无法播放 XMA 音频。
- **WG-34376** 已解决：当编辑 Wwise 数值时，在 Multi-Editor 中发生崩溃。
- **WG-34378** 已解决：Game Object Profiler 中的空内存访问崩溃报告。
- **WG-34504** 已解决：在使用精确到采样点的过渡，并且媒体在多个包中被复制时，卸载包会导致罕见崩溃。
- **WG-34643** 已解决：当 `AK::InitSettings::bUseSoundBankMgrThread` 被设置为 `false` 时，`UnloadBank` 有时会返回错误码。
- **WG-34648** 已修复：在耗尽内存时， `CAkLimiter::Remove()` 可能会发生崩溃。
- **WG-34973** 已修复：（PS4）振动出现内存泄漏。
- **WG-35086** 已修复：（iOS 11）AAC 解码器故障。
- **WG-35245** 已修复：(Android) 在 Android 上，Unity 无法正常使用 Crankcase REV。
