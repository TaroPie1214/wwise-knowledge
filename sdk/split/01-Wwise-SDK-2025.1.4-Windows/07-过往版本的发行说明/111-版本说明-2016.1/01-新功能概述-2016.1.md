新功能概述 2016.1

# Ambisonics 支持

Wwise 现在支持导入 ambisonic 媒体源文件（AMB或WAV）、以 3D 方式旋转 ambisonics 源和通过 ambisonics 特定总线配置输出 ambisonics。而且通过使用新的 Wwise Recorder 插件，可以将 ambisonics 输出保存到 ambisonic 文件。

# Wwise Recorder插件

Wwise 推出了新效果器插件。在总线上插入 Wwise Recorder 插件效果来记录以 WAV、WEM 或 AMB 文件方式通过的音频信号。这些记录文件可在任何声音编辑器中打开或导入到 Wwise 工程中。也可以在游戏执行中记录，然后通过外部源对象在游戏中播放。

# 离线解压缩

`PrepareBank()`现在有在加载 SoundBank 时自动解压缩 Vorbis 文件并在 PCM 中缓存它们的选项。这可以在未来播放时节省 CPU，对 CPU 吃紧和重复率高的声音（例如自动武器）的平台特别有用。

# 工作流程改进

工作流程改进后，在 Wwise 设计工具中的工作变得更加容易、更加快速：

- **Project Explorer Browser 增强功能**：Project Explorer Browser 视图现在有加快对象查找的搜索字段。它还可以记住上次选择的对象和展开/折叠状态。
- **表列配置**：现在整个 Wwise 中都可以对表格中的列进行隐藏、显示和排序操作。
- **可以向键盘快捷方式指定新的 Wwise Authoring 命令**：通过键盘快捷方式和控制设备会话可访问新的Wwise命令：
  - 对象：
    - Open In External Editor (1 to 5)
    - Open In Wwise Wave Viewer
  - 工作组：
    - Add (WAV)
    - Add (Work Unit)
    - Checkout (WAV)
    - Checkout (Work Unit)
    - Commit-Submit (WAV)
    - Commit-Submit (Work Unit)
    - Diff (WAV)
    - Diff (Work Unit)
    - Revert (WAV)
    - Revert (Work Unit)
    - Update (WAV)
    - Update (Work Unit)
  - [了解如何将命令指定到快捷方式。](https://www.audiokinetic.com/library/?source=Help&id=keyboard_shortcuts)
  - [Learn how to assign a command to a control surface control.](https://www.audiokinetic.com/library/?source=Help&id=creating_control_surface_bindings#creating_binding_for_keyboard_shortcut)

# 深刻的 API 变化

对 Wwise API 的重大返工带来了几大改进：

- **插件编码改进**：代码修改功能使开发人员更容易注册和链接插件，而对先前工作无影响或影响很小。例如，Wwise 插件现在可轻易地添加到 Unity 工程，无需再重新编译 DLL。请参阅 [重要迁移说明 2016.1](whatsnew_2016_1_migration.html) 了解有关插件动态和静态库系统变化的更多信息。
- **扩大了系统 API 支持**：除 DirectSound、WASAPI 和 XAudio2 外，其它音频系统输出现在也可以使用音频设备共享集定义。这些新的共享集可以实现第三方的开发，也可以结合新开放的 Sink 插件框架实现您自己的开发。
- **MIDI API 改进**：Wwise 2016.1 中提供了新的 MIDI 函数。这些函数可以通过 API 向声音引擎发布 MIDI 事件，进一步提高开发游戏内生成音乐的能力。请参阅`PostMIDIOnEvent()` 了解更多信息。

# 移除转码设置限制

在 iOS、tvOS 和 Android 平台上，Wwise 现在支持多声道声音。

# 支持高 DPI

Wwise 现在支持例如 4K 和 5K 显示器上的高象素密集度显示。
