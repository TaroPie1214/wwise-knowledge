# 其他改进

- **WG-55188** (wp.py) 脚本默认为 Python 3。
- **WG-57641** 针对内存溢出添加了调试工具。
- **WG-60203** (wp.py) 为 Switch 和 PS5 添加了对 Visual Studio 2019 解决方案的支持。

# 漏洞修复

- **WG-53467** 已修复：WwiseConsole 在 macOS 上消耗 100% 的 CPU 资源。
- **WG-55462** 已修复：在 Mac 上将 Keyboard Shortcuts and Commands 中创建的命令文件显示成了灰色（不可用）。
- **WG-57483** 已修复：在清理 SoundBank 后针对暂停的 Interactive Music 对象调用 `StopAll` 操作时发生崩溃。
- **WG-58028** 已修复：在删除 Sound Voice 时，Contents Editor 发生崩溃。
- **WG-58713** 已修复：(Mac Authoring) 无法在自定义命令中正常运行当前工作目录。
- **WG-59100** 已修复：在 macOS 上无法使用 Unicode 字符导入音频文件。
- **WG-59273** 已修复：无法在 macOS Monterey 12.3 上启动 Wwise。
- **WG-59359** 已修复：有时会过早退出 WwiseConsole。
- **WG-59569** 已修复：WwiseConsole 中有时会忽略主循环的超时状态。
- **WG-60072** 已修复：(Mac Authoring) Game Object 3D Viewer 中没有文本。
- **WG-60083** 已修复：在取消停靠前端的 REV 2 插件时出现卡顿。

# 社区报告的漏洞修复

- **WG-57768** 已修复：在反复生成 SoundBank 时，SoundBanksInfo.xml 中 ExternalSources 的顺序不一致。
- **WG-57931** 已修复：在连接 Wwise Profiler 时，SoundEngine 崩溃或挂起。
- **WG-58185** 已修复：在使用重定向输出时，自定义命令会在程序前面添加 Change Working Directory (CWD)。
- **WG-58199** 已修复：有些随 Wwise 一起发布的动态库被报告为潜在恶意软件。现在会在适用情况下使用 Audiokinetic 的证书对其进行签名。
- **WG-58792** 已修复：Music Track 中所含的 Music Clip 可能会引用并非子级 Music Track 对象的源文件。
- **WG-58802** 已修复：`AkBufferPosInformation` 结构中存在未初始化的变量。
- **WG-59602** 已修复：在启动新的实例时，有些 SoundSeed Woosh Randomizer 选项无法正常工作。
- **WG-60262** 已修复：在对流播放 Opus 文件寻址时可能会发生崩溃。
- **WG-60284** 已修复：在为 CrankCase 生成内容时可能会出现竞争危害，并导致在生成 SoundBank 后使用过时的信息。
- **WG-60299** 已修复：在对流播放文件寻址且寻址目标在预取窗口中时可能会发生崩溃。
