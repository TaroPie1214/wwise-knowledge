# API 改进

- **WG-51210**  `增大了 SetGameObjectAuxValues` 和 `SetGameObjectOutputBusVolume` 可接受的取值范围。新的倍数范围介于 0 和 16 之间，在超出该范围时会对数值加以限制。

# 其他改进

- **WG-51201** 将默认监控队列大小增大到了 1024 kb。

# 漏洞修复

- **WG-49304** 已修复：在导入 WAAPI 或制表符分隔的音频文件时会忽略指定的音频源名称。
- **WG-49918** 已修复：Mastering Suite 坐标图中轴的值域与其对应属性的值域不一致。
- **WG-49940** 已修复：在 Mastering Suite 中实时调节参数时听到杂音。
- **WG-49974** 已修复：Mastering Suite 转换曲线图没有按照对数标度（底数为 10）显示比率。
- **WG-49975** 已修复：Mastering Suite 中 Limiter 参数滑杆的标度不够细化。
- **WG-50165** 已修复：在将某些 DirectInput 设备连接到电脑时，设计工具出现短暂的 UI 卡顿。
- **WG-50829** 已修复：在对象特别多时无法滚动到 Assigned Object 列表 (Switch Container) 的末尾。
- **WG-50834** 已修复：Mastering Suite 内 Master Volume 模块中的 Channel Link 按钮不显示工具提示。
- **WG-50889** 已修复：无法通过右键单击总线和辅助总线并选择 Find all references 来调节衰减曲线。
- **WG-50980** 已修复：因 Spatial Audio Diffraction 代码路径被零隔开而导致音频中断。
- **WG-51024** 已修复：在将 Virtual Folder 作为 Music Switch 过渡规则的 Source 或 Destination 时，Source 或 Destination 的值被视作等同于 None。
- **WG-51035** 已修复：Mastering Suite 中的鼠标光标在隐藏行为上存在不一致。
- **WG-51049** 已修复：在导入 WAAPI 或制表符分隔的音频文件时选择替换模式会创建重复的 Event 动作。
- **WG-51050** 已修复：(Spatial Audio) 在启用 Output Bus 滤波器的情况下，衍射和透射 LPF/HPF 值偏低会导致音量突然增大。
- **WG-51066** 已修复：在更改主题时无法正确更新 Mastering Suite 调色板。
- **WG-51166** 已修复：在通过延迟型 I/O 实现方式准备媒体时触发断言。
- **WG-51188** 已修复：(Spatial Audio) 在三角形的平面与 Portal 轴平行的情况下，Portal 和平面之间的交叉计算出现错误，进而触发断言并导致 Portal 开口出错。
- **WG-51192** 已修复：在使用较大的 Spread 和 Focus 值对多声道声音进行 3D 定位时发生不符合预期的声像定位行为。
- **WG-51208** 已修复：对于某些 Sound Engine 缓冲区大小设置，iZotope Trash Box Modeler 可能无法正常工作。
- **WG-51222** 已修复：在 Mastering Suite 中手动编辑频段数量会导致分频频率出错。
- **WG-51239** 已修复：在采用 24 位编码时，ASIO Input 插件没有音频输出。
- **WG-51259** 已修复：Spatial Audio Room 导致在每次更新时都会对 Voice Graph 进行不必要的重新评估。
- **WG-51303** 已修复：在平台的音频帧大小不是 4 的倍数时，Mastering Suite 出现错误。
- **WG-51314** 已修复：在 Wwise 设计工具中切换 Effect 的 Enable 复选框时，Mastering Suite 发生内存泄漏。
- **WG-51404** 已修复：Wwise 将 Virtual Folder 下的所有总线全部设为了引用 System Audio Device。
- **WG-51444** 已修复：在 Mastering Suite 中链接或取消链接声道可能会导致相对音量前后不一致。

# 社区报告的漏洞修复

- **WG-51498** 已修复：AK::SoundEngine 命名空间引用页中缺少函数。
