# 重要的迁移说明2015.1

|  |
| --- |
| Wwise SDK 2025.1.4 - Windows |

重要的迁移说明2015.1

**WG-4849** 添到了对多个平台版本的支持。

新功能现在可让您为每个目标平台创建多个平台。

- 在迁移时，您的项目将自动为工程中所使用的每个现有平台创建一个自定义平台。我们将保留原来的平台名称。 命令行的脚本编写方式在迁移后仍可以正常工作。生成的 SoundBank 路径将保持原样。但是现在注意，如果您选择重命名或添加自定义平台，您的自定义工具可能需要相应地更新，才能支持您创建的新平台。
- 如果您先前设置了键盘快捷方式来快速访问平台，则这些快捷方式可能会丢失。

  注解
  :   这取决于用户的机器。它不会连接到项目本身。
- 在迁移时，如果您的项目有被禁用平台的残余数据，则这些数据将从工程中移除。如果这些数据有用，则请确保迁移前在 Wwise 的先前版本中激活已被禁用的平台。

**WG-26052** 移除了 AkCopyStreamedFiles.exe 工具。

Wwise在出厂时提供了一个新的命令行工具“CopyStreamedFiles.exe”，它支持最新的 SoundBank XML元数据格式变化。

- 在迁移时，您的生成后步骤将自动转换，以使用这一新工具。

**WG-26233** 迁移通过 [PrepareEvent()](namespace_a_k_1_1_sound_engine_a2e6ebb779470b43d78984c27f6a0c238.html#a2e6ebb779470b43d78984c27f6a0c238) API 加载的媒体

为了通过不存储每个声音的偏移量和 SoundBank 标识符来节省运行时的内存，我们更改了 [PrepareEvent()](namespace_a_k_1_1_sound_engine_a2e6ebb779470b43d78984c27f6a0c238.html#a2e6ebb779470b43d78984c27f6a0c238) API 查找媒体文件的方式。其结果是，使用 [PrepareEvent()](namespace_a_k_1_1_sound_engine_a2e6ebb779470b43d78984c27f6a0c238.html#a2e6ebb779470b43d78984c27f6a0c238) API 按需加载媒体的游戏将必须调整它们的 SoundBank 组织。在Wwise 2015.1中，通过 [PrepareEvent()](namespace_a_k_1_1_sound_engine_a2e6ebb779470b43d78984c27f6a0c238.html#a2e6ebb779470b43d78984c27f6a0c238) 加载的媒体必须以文件系统可访问的松散文件方式存在。如果这些媒体文件包含在 SoundBank 中，它们将被无法再被 SoundBank Manager 找到。迁移的方法是，从所有 SoundBank 中移除将通过PrepareEvent()加载的媒体文件。CopyStreamedFiles 实用程序在默认情况下配置为将任何 SoundBank 中不包含的松散文件（和流播放文件）复制到目标位置。

请参阅《安装与迁移指南》了解有关将工程迁移到 Wwise 新版本的一般建议。
