版本说明 2016.2.2

以下各节列举并描述Wwise版本2016.2.1与版本2016.2.2之间的变化。

# 平台 SDK 的变化

- **WG-32419** Linux：Crankcase REV 插件现已可用。

# 漏洞修复

- **WG-30561** 已解决：Meter FX 中除数为 0 异常。
- **WG-30641** 已解决：当 `IReadBytes::ReadString` 缓冲大小不足时，返回值不正确，并且内部状态错误。
- **WG-31068** 已解决：（PS4）当断开耳机和重连耳机时出现问题。
- **WG-31287** 已解决：卸载包含 MIDI 的 SoundBank 无法停止声音，而是继续保持等待。
- **WG-31663** 已解决：Game Parameter Simulation Value 在控制设备绑定时不可用。
- **WG-31703** 已解决：导入文件夹不遵守对象层级结构。
- **WG-31895** 已解决：Custom References 和 Properties 在某些对象类型上无法链接到 Multi Editor 或与其取消链接。
- **WG-31907** 已解决：在 Multi Editor 中时，点击参考属性会错误地替换当前数值。
- **WG-32019** 已解决：当在 Music Transition 源/目标浏览器中未选取任何对象时，发生崩溃。
- **WG-32021** 已解决：如果 Switch Containers 未同步（`WALSwitchCntr::SetSwitchGroup），连接时会发生崩溃。`
- **WG-32026** 已解决：Interactive Music PlayBack Speed（互动音乐回放速度）不总依据 RTPC 更新。
- **WG-32076** 已解决：在 Project Settings 中更改自定义属性之后， WCUSTOMPROPERTIES 文件内容为空。
- **WG-32192** 已解决：当在多个次级输出中播放时，Advanced Profiler 的 Voice选项卡中 Game Object 显示不正确。
- **WG-32250** 已解决：`AK::StreamMgr::IAkIOHookDeferred::Cancel()` 只为已知的传输被调用一次，即使它将 `io_bCancelAllTransfersForThisFile` 从 true 改为 false。
- **WG-32309** 已解决：由未被引用的调制器导致的 `LoadBank()` 故障。
- **WG-32320** 已解决：ambisonics 输入导致 Convolution Reverb 崩溃。
- **WG-32357** 已解决：当一个 ambisonic 阶数高于 IR 阶数的总线使用 Convolution Reverb 时，干声级别不应用于额外的声道。
- **WG-32396** 已解决：当使用大量高优先级并且已进行声像摆位的对象时，SCE Audio 3D plug-in 中 Ambisonics 可能出故障。
- **WG-32417** 已解决：（iOS）使用 iOS 10.3 Beta 版本的情况下，sink 初始化导致崩溃。
- **WG-32421** 已解决：当用 `AkChannelEmitter` 来使用 `SetMultiplePositions` 时，导致输入声道映射错误。

# 其它更改

- **WG-31785**: 为 Wwise 帮助添加了 **Simplified Chinese（简体中文）** 的 User Preferences、Documentation、Language 选项。
