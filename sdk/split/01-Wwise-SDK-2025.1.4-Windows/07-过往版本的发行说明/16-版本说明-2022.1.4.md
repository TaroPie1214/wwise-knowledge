# 新增功能

- **WG-64101** 添加了新的声音引擎 API `AK::SoundEngine::SetJobMgrMaxActiveWorkers`。在对声音引擎中的 Job Manager 进行初始化后，可使用此 API 来动态修改 Job Manager 所发布工作请求的最大数量，而无需重新初始化声音引擎。

# 漏洞修复

- **WG-59939** 已修复：Object Tab 内的 Secondary Editor 的 Minimize 和 Maximize 图标不够清楚。
- **WG-62367** 已修复：(Spatial Audio) 在开始和停止播放声音时，Game Object 3D Viewer 中没有显示散布声锥。
- **WG-63907** 已修复：Schematic 视图中高亮显示的节点有透明边框。
- **WG-63930** 已修复：(Spatial Audio) 一个表面跟另一表面完全重合时，反射声断断续续且不稳定。
- **WG-64263** 已修复：有时不会立即更新对与 RTPC 绑定的 Switch 对应的曲线所作的更改。在有些情况下，会使用原有曲线，直到重新加载。
- **WG-64614** 已修复：(Spatial Audio) 在内存不足时可能会发生崩溃。
- **WG-64661** 已修复：在快速切换 Object Tab 时可能会发生崩溃。
- **WG-64663** 已修复：在内存不足的情况下，AkMediaEntry 可能会触发断言。
- **WG-64718** 已修复：没有应用 Voice Volume 和 Makeup Gain 总线属性上的调制器。
- **WG-64742** 已修复：(Spatial Audio) 在计算 Room 内的衍射射线时，由于射线长度计算出现错误，导致随机射线投射引擎的性能下降。
- **WG-64781** 已修复：在音乐对象的上级对象指定了 MIDI Target 时，SoundBank 没有包含音乐对象的 MIDI Target 层级结构。
- **WG-64957** 已修复：在工程加载过程中，Media ID 验证不够及时。这只会影响 Wwise 设计工具中的本地播放。
- **WG-64982** 已修复：在将鼠标悬停于列表标题上时没有显示工具提示。
- **WG-65017** 已修复：在设置 `AkJobMgrSettings::fnRequestJobWorker` 的情况下运行声音引擎时，`AK::JobMgr::WorkUntilDone` 可能会发生死锁。

# 社区报告的漏洞修复

- **WG-61946** 已修复：有些内存地址没有充分对齐。
- **WG-63863** 已修复：(WAAPI) 在针对处于版本控制之下的工程执行导入操作时，并非所有对话框都会在自动化模式下被隐藏。
- **WG-64015** 已修复：在执行 Perforce 版本控制操作的过程中向 Wwise 发布多项“关闭”命令会引发崩溃。
- **WG-64424** 已修复：在正在播放的声音上添加使用内置参数的 RTPC 曲线会导致无法正确累加曲线的效果。
- **WG-64495** 已修复：在内存不足的情况下执行 `Ak3DAudioMixer::TriageAndMix` 会引发崩溃。
- **WG-64496** 已修复：在 Music Switch Container 包含的 Switch 目标并非前者在工程层级结构下的子对象时，没有将 Music Switch Container 正确打包到 SoundBank 中。
- **WG-64632** 已修复：在 Project Explorer 中单击对象时可能会发生崩溃。
- **WG-64669** 已修复：在 Music Playlist Container 的 Transition 没有验证关键属性值时发生死锁。
- **WG-64683** 已修复：SoundBank 中的 GameSync 弃用设置对 Interactive Music 对象不起作用。
- **WG-64849** 已修复：对于有些带有 Listener Relative Routing 的总线结构，有时会一次将两条 Master Audio Bus 错误地发送到同一 Output Device 两次，导致出现音频丢失或其他错误。
- **WG-64881** 已修复：`AK::JobMgr::Internal::JobPool_initEntry` 可能会发生死锁。
- **WG-64898** 已修复：没有将 Music Stinger 父对象打包到 SoundBank 中。
- **WG-64919** 已修复：在执行 Stop All 动作后，`CAkURenderer::RemoveFromActiveNodes` 中存在无效内存访问，导致发生崩溃。
- **WG-64949** 已修复：在首次播放前没有设置初始 Switch 时，Continuous Switch Container 有时会持续播放额外的声音。另外，在同一游戏对象上播放两个不同实例时切换会导致只播放一个实例。
