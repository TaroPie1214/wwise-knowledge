# 版本说明 2018.1.4

|  |

| Wwise SDK 2025.1.4 - Windows |

版本说明 2018.1.4

以下各节列举并描述 Wwise 版本 2018.1.3 和版本 2018.1.4 之间的变化。有关早期版本的信息，请参阅 [过往版本的发行说明](previousreleases.html) 章节。此版本为漏洞修复版本，因此对所有 2018.1.x 版本来说，工程、音频包和 API 都是兼容的，不会有任何变动。

|  |  |
| --- | --- |
|  | **备注:** **插件开发人员：** 此 Wwise 版本支持通过 Wwise 2018.1 构建的插件。对于通过旧版 Wwise 构建的插件，开发人员必须使用兼容版本重新编译并重新打包。 |

# 平台 SDK 的变化

- **Android NDK**：已更新至 NDK r17c。
- **Lumin SDK**：已更新至 SDK 0.18.0。
- **Switch SDK**：已更新至 SDK 6.4.0。

# 新功能

- **WG-36301** 添加了新的付费插件 Wwise SoundSeed Grain。它是一个可添加到 Wwise 工程中的粒子合成器。SoundSeed Grain 方便灵活地对音频文件实施粒化和变换。如需了解更多信息，请参阅 [Wwise SoundSeed Grain](https://www.audiokinetic.com/library/edge/?source=Help&id=wwise_soundseed_grain_plug_in) 帮助页面。
- **WG-39124** 新增了免费的**实验性**插件 Wwise ASIO。它使用 Steinberg Audio Stream Input/Output (ASIO) 连接 ASIO 兼容设备。如需了解更多信息，请参阅 [Wwise ASIO Output](https://www.audiokinetic.com/library/edge/?source=Help&id=wwise_asio_output_plug_in) and the [Wwise ASIO Input](https://www.audiokinetic.com/library/edge/?source=Help&id=wwise_asio_input_plug_in) 帮助页面。
- **WG-39329** Source Editor 中现在会显示音频文件标记。
- **WG-40481** 新增了免费的**实验性**插件 Wwise Channel Router。它方便将多条采用不同声道配置的总线混音并输出到单个输出设备。如需了解更多信息，请参阅 [Wwise Channel Router](https://www.audiokinetic.com/library/edge/?source=Help&id=wwise_channel_router_plug_in) 帮助页面。

# 行为变化

- **WG-39476** 通过将声音引擎配置为使用任务计划程序，提升了 Xbox One XMA 声部的处理性能；先并行处理非 XMA 声部，再并行处理 XMA 声部。

# 其它更改

- **WG-39571** (Android) 现在将 Integration Demo 构建成了标准的 Gradle 工程，可使用 Android Studio IDE 直接打开。
- **WG-39757** 重新命名了 Source Editor 中的以下两个选项卡：   
  - **General Settings** 改成了 **Source**。
  - **Source Settings** 改成了 **Conversion**。
- **WG-40031** 将 .gitignore 文件添加到了插件生成器中。

# 漏洞修复

- **WG-32443** 已修复：(PS4) 在运行 SCE Bed 时调用 suspend 会引发崩溃。
- **WG-39022** 已修复：在 Nexus 9 设备上运行 Integration Demo 时发生崩溃。
- **WG-39390** 已修复：Perforce 出现故障，在重新添加之前删除的文件后仍会显示删除标记。
- **WG-39857** 已修复：UI 出现故障，导致 Wwise 在应用部分插件（包括 McDSP Limiter 和 Futzbox）时发生随机崩溃。
- **WG-39870** 已修复：在编辑器中移动 RTPC 光标时发生罕见崩溃。
- **WG-39871** 已修复：在处理某些耗时的进程时关闭 Wwise 两次会引发崩溃。
- **WG-39896** 已修复：在移除几何构造后，AkReflect 中短暂显示受影响的镜像声源名称。
- **WG-39917** 已修复：(PS4) PS4 控制器上的 Motion 效果减弱。
- **WG-39937** 已修复：在 Spatial Audio 中，三角形太大会导致浮点溢出和不符合预期的行为。
- **WG-39941** 已修复：在将 7.1 文件转码为 Opus 格式时会反转环绕声和后置声道。
- **WG-39998** 已修复：在通过远程桌面连接时，假如正在加载工程布局，会无法初始化 GameProfiler3D 视图。
- **WG-40011** 已修复：效果器和源插件的默认 Property Editor 中不显示引用。
- **WG-40036** 已修复：在较响的声音启用了辅助发送时，干声和湿声路径中的 HDR 衰减不均衡。
- **WG-40074** 已修复：在断开监控的情况下，假如游戏生成大量错误，稍后可能会出现堆栈溢出情况（仅 Profile 和 Debug 版本中）。
- **WG-40077** 已修复：假如未选中任何一行，Capture Log 中可能会不显示位置标志。
- **WG-40097** 已修复：即便指定了别的设备，Microsoft Spatial 输出也会继续选择默认输出设备。
- **WG-40152** 已修复：无法将 Wwise 连接到不同子网中的主机。
- **WG-40234** 已修复：在插件引用不止一个媒体项时， [GetPluginMedia](class_a_k_1_1_i_ak_plugin_context_base_a36514f7827919ee6c85a56295141297a.html#a36514f7827919ee6c85a56295141297a) 的行为不稳定。
- **WG-40330** 已修复：(PS4) 扬声器配置始终为 7.1。
- **WG-40349** 自定义属性现在可以支持其他数字类型。另外，在从 Real64 转换为 Real32 值时新增了精度损失警告；在从 int64 转换为超出容许范围的 int32 值时新增了数据丢失错误。
- **WG-40372** 已修复：在播放与游戏对象绑定的声音时，假如将 Spatialization 设为 None，并在 MultiSource 中为该游戏对象指定多个位置，会导致音量无限大。
- **WG-40504** 已修复：父对象设置了 Listener Relative Routing 时，SoundBank 中的对象不会沿用。
