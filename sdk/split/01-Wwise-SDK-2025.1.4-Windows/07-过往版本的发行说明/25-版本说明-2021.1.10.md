# 行为改进

- **WG-60412** 作为 System Audio Object 发送到输出设备的多声道 Audio Object 现在具有与被摆位到 Main Mix 或 Passthrough Mix 的同等多声道源类似的音量。这一改进会将多声道 System Audio Object 的音量降低几个分贝（具体取决于其声道数）。比如，对于立体声源，为 -3 dB；对于四声道源，为 -7 dB。在实践中，这意味着任何为此专门针对 3D Audio 对混音所做的修正都可被移除。

# 性能改进

- **WG-54292** (Spatial Audio) 针对 Room 游戏对象减少了冗余路径数。

# 漏洞修复

- **WG-58695** 已修复：在移除 Voice Data 性能分析数据并执行 Start Capture 时可能会发生崩溃。
- **WG-59298** 已修复：在 `AK_SpeakerVolumeMatrix` 回调中对音量矩阵所作的更改只在下一音频帧生效。
- **WG-59312** 已修复：由于在 Audio Object 位置变化时声像摆位不连续，导致在 Output Device 上混音到 Main Mix 或 Passthrough Mix 的 Audio Object 出现可闻的噼啪声。
- **WG-59864** 已修复：若在使用 WAAPI 脚本时关闭客户端，Wwise 撤消命令会不起作用。
- **WG-60068** 已修复：在结合单声道声音同时使用 Focus 和 Spread 时，Audio Object 3D View 中 Spread 圆圈的视觉大小有误。
- **WG-60103** 已修复：在分配新的效果器时运行声音引擎会导致内存泄漏。
- **WG-60348** 已修复：`SetGameObjectRadius` API 调用没有返回其 Game Object ID。
- **WG-60428** 已修复：作为 System Audio Object 单独发送到输出设备的四声道 Audio Object 出现定位错误。
- **WG-60510** 已修复：`ComputeSphericalVBAPGains` 返回的音量不太精准。
- **WG-60564** 已修复：在注册游戏对象时内存分配失败，可能进而导致内存泄漏。
- **WG-60579** 已修复：`SetGameObjectToPortalObstruction` 和 `SetPortalToPortalObstruction` 对象和值没有输出到 API Call。
- **WG-60686** 已修复：在重新加载 Work Unit 后查看 Music Switch Container 时无法正确刷新 Property Editor。
- **WG-60833** 已修复：对于有些 Pitch 值，虚声部上的 Time Stretch 效果器 (Play from elapsed time) 可能会导致 CPU 用量突然增加。
- **WG-60861** 已修复：在发送空的 `AkAcousticSurface` 名称时发生崩溃。
- **WG-60899** 已修复：在 Column Configuration 包含 Sizes Preview/Total Size 时，Query Editor 发生崩溃。
- **WG-60919** 已修复：在 Music Segment Editor 中剪切后无法粘贴音乐片段。
- **WG-60938** 已修复：Time Stretch 效果器的尾音中产生的采样有误。
- **WG-60967** 已修复：在复制或剪切 Music Event Cue 时，Wwise 发生内存泄漏。

# 社区报告的漏洞修复

- **WG-59458** 已修复：在一个工程中使用多个 Mastering Suite 插件时出现音频失真。
- **WG-59586** 已修复：在使用滑杆调节 Convolution Reverb 属性时可能会发生崩溃。
- **WG-59913** 已修复：在 MIDI 环境下播放音乐时，在 `AK::SoundEngine::ClearBanks` 之后马上调用 `AK::SoundEngine::StopAll` 会发生崩溃。
- **WG-59929** 已修复：Portal 声障对房间底噪不起作用。
- **WG-60215** 已修复：Audio Input 插件无法在 Wwise 设计工具中产生声音。
- **WG-60243** 已修复：在音频设备长时间断开时发生崩溃。
- **WG-60352** 已修复：针对取消链接的平台将效果器插件打包在了 SoundBank 中。
- **WG-60373** 已修复：(Spatial Audio) 在穿越不同的中间 Room 时，经由两条可能路径传播到听者所在 Room 的房间底噪只能通过其中较短的一条路径听到。
- **WG-60415** 已修复：在 macOS 上无法使用 Unicode 字符导入音频文件。
- **WG-60576** 已修复：在 Property Editor 中有 Music Switch Container 时重新加载 State Group Work Unit 会发生崩溃。
- **WG-60592** 已修复：在所用 Modulator 使用与内置属性绑定的 RTPC 时发生崩溃。
- **WG-60618** 已修复：Address Sanitizer 报告了与函数 `AK::DeviceEnumeratorHelper()` 相关的错误。
- **WG-60699** 已修复：在 Music Clip 上设置颜色的情况下打开工程时发生崩溃。
- **WG-60739** 已修复：在工程下定义Custom Properties 时无法在 WAQL 中进行访问。
- **WG-60823** 已修复：在外部编辑器中更改音频源后使用 Audio Source Editor 上的曲线菜单时发生崩溃。
- **WG-60946** 已修复：在通过 WAAPI 或 Tab Delimited Import 导入附加音频文件源时没有将音频源设为活跃状态。
- **WG-61077** 已修复：在通过关联到 RTPC 并设有内置值（如距离）的 Switch Group 播放 Music Switch Container 时发生崩溃。
