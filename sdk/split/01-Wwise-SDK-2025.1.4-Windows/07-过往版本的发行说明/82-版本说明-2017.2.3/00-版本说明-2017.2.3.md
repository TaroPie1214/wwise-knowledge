版本说明 2017.2.3

以下各节列举并描述 Wwise 版本 2017.2.2 和版本 2017.2.3 之间的变化。有关早期版本的信息，请参阅 [过往版本的发行说明](previousreleases.html) 章节。如果使用联机帮助，您还可以从目录列的顶部选择别的 Wwise 版本。此版本为补丁版本，因此对所有 2017.2.x 版本来说，工程、音频包和 API 都是兼容的，不会有任何变动。

# 性能变化

- **WG-36930** 缩短了工程加载时间（搜索索引生成速度比以前快 1.25 倍）。

# 漏洞修复

- **WG-36649** 已修复：在使用多个 SCE Audio 3D Sink 时会导致设计工具运行速度变慢。
- **WG-36668** 提升了 Reflect 延迟线处理声源的效率。
- **WG-36742** 已修复：在连接时 `CAkStateAware::SetStateProperties` 出现故障。
- **WG-36749** 已修复：`ResetAll` 速度很慢。
- **WG-36812** 已修复：(PS4) `CAkSinkPS4` 忽略 `AkPlatformInitSettings::uNumRefillsInVoice`。
- **WG-36818** 已修复：LPF 和 HPF 控件更新导致块大小为 2,048 的音频出现可察觉的失真，同时导致块大小为 256 和 512 的音频出现控制精度问题。
- **WG-36893** 已修复：在所设寻址表恰巧读到文件结尾附近时，从音频包读取文件可能会导致 Vorbis 发生崩溃。
- **WG-36935** 已修复：为 Bus Volume、Output Bus Volume、LPF 或 HPF 添加 State 会使总线中的声音混合在一起。
- **WG-36954** 已修复：在创建新的工程时，不能正确添加 Audio Device ShareSet。
- **WG-36958** 已修复：在播放多个音乐实例时忽略了 **change occurs at** 状态。
- **WG-36977** 已修复：在播放的声音没有输出至设备时发生崩溃。
- **WG-36985** 已修复：在调用 `AK::SpatialAudio::SetPortal()` 时，Portal 声障值重置为 0。
- **WG-37000** 已修复：在初始化引擎后立即加载 Init.bnk 时，可能会出现竞争危害，继而引发崩溃。
- **WG-37010** 已修复： `SetImageSource` 会生成大量声源，直至内存池内存耗尽。
- **WG-37065** 已修复：在停止游戏时， `AK::SoundEngine::Term` 可能发生崩溃，继而导致 Unity Editor 崩溃。
- **WG-37075** 已修复：对于有些 Portal 朝向，不能正确定位听者相邻 Room 的 Room 游戏对象。
- **WG-37116** 已修复：生成前/后步骤宏中的路径不规范。
- **WG-37117** 已修复：SoundBank 元数据文件中替代声部媒体 (Use reference language as stand-in) 的时长信息不正确。
- **WG-37123** 已修复：External Source 输出路径不规范。
- **WG-37130** 已修复：同一 Room 内所有 Portal 的 Portal 声障值全部被设为最靠近听者的 Portal 的 Portal 声障值。
- **WG-37131** 已修复：不能设置 Room 游戏对象的墙壁声笼值，因此也不会应用于 Room 底噪。
- **WG-37162** 已修复：在听者慢慢进入另一 Room 时，假如该 Room 具有不同的墙壁声笼值，那么取自听者所在 Room 的墙壁声笼值就可能会不连续。
- **WG-37169** 已修复：声音传播系统没有考虑发声体与最近 Portal 之间的距离，导致距离和衍射计算基于的路径不是最佳路径。
