# 平台 SDK 改进

- **Xbox One XDK**：已更新至 XDK 10.0.17134.5076（2018 年 7 月 qfe13）。
- **Stadia SDK**：已更新至 SDK 1.45。

# 行为改进

- **WG-48499** PulseAudio (Linux/Stadia)：现在支持使用 `AkPlatformInitSettings::sampleType` 在 `PA_SAMPLE_FLOAT32LE` 和 `PA_SAMPLE_S16LE` 之间进行选择。在 Stadia 上，默认使用 `PA_SAMPLE_S16LE`。不再使用 `PA_STREAM_INTERPOLATE_TIMING` 和 `PA_STREAM_ADJUST_LATENCY`。

# 其他改进

- **WG-48818** 现在改为分发 vc\_150 和 vc\_160 版的 Wwise 插件 dll。

# 漏洞修复

- **WG-46449** 已修复：(Android) 在运行老版固件的 Vivo Y51A 手机上，音频输出含混不清。
- **WG-46627** 已修复：在 Wwise 中实时编辑自定义属性时没有更新游戏中的值。
- **WG-46745** 已修复：在卸载 PCK 文件时可能会发生崩溃。
- **WG-47646** 已修复：Capture Log 无法正确滚动到光标位置。
- **WG-47765** 已修复：Profile 中 Stereo Delay 插件 5.1 声道的音频输出无效。
- **WG-47810** 已修复：在有 Game Object 被注销的情况下停止播放处于 Continuous 模式的 Switch Container 可能会引发崩溃。
- **WG-48224** 已修复：在没有给 Portal 命名（名称字段为空）的情况下调用 `AK::SpatialAudio::SetPortal` 可能会引发崩溃。
- **WG-48235** 已修复：`FilePackager.Console` 无法打包设为流播放的音频文件。
- **WG-48287** 已修复：Linux 和 Stadia 上忽略了 AkThreadProperties::dwAffinityMask。
- **WG-48379** 已修复：在调用 `AddOutputCaptureMarker` 后马上终止声音引擎会引发轻微的内存泄漏。
- **WG-48392** 已修复：Advanced Profiler 视图内 Listeners 选项卡的 Volumes 列中显示了错误的值。
- **WG-48396** 已修复：在初始化声音引擎时，Stadia SDK 1.44 发生崩溃。
- **WG-48417** 已修复：在启用 `AkInitSettings::bDebugOutOfRangeCheckEnabled` 系统时，错误地将管线中的下一效果器指定为 Out Of Range 样本的源。
- **WG-48440** 已修复：在 Windows 1709 上，MS Spatial Audio 插件发生崩溃。
- **WG-48442** 已修复：在通过命令行生成 External Source 时显示了错误的警告。
- **WG-48503** 已修复：`AkArray::BinarySearch` 中越界读取内存。
- **WG-48776** 已修复：在播放的声音具有自定义属性时触发断言。
- **WG-48822** 已修复：在循环流播放 Opus 声部时播放中止。
- **WG-48855** 已修复：在长时间未连接 Wwise 的情况下运行游戏会触发断言 (AkBehavioralCtx.cpp)。
- **WG-48976** 已修复：在上次发布补丁时意外地移除了 iOS 中断处理程序保护代码。
- **WG-49030** 已修复：(Stadia) 在 `AkPlatformInitSettings::sampleType` 等于 `AK_INT` 时，5.1 Surround Audio 声道模式发生崩溃。

# 社区报告的漏洞修复
