版本说明 2016.1.4

以下各节列举并描述Wwise版本2016.1.3与版本2016.1.4之间的变化。

# 平台 SDK 的变化

- **PS4 SDK**: 已更新至 SDK 4.008.101
- **Vita SDK**: 已更新至 SDK 3.570

# 漏洞修复

- **WG-30971** 已解决：（Vita 硬件）效果器插件未被注册。
- **WG-30984** 已解决：rumble （隆隆声）在设备变更后接收所有音频信号并且永不停止。
- **WG-30986** 已解决：当牵涉到 Blend Container 时，声音可能无法恰当地连接到 Auxiliary Bus。
- **WG-30992** 已解决：（PS4）使用 SCE Audio3d Bed Output 音频设备时会忽略 Master Audio Bus 的音量。
- **WG-31008** 已解决：当 Blend Container 在播放时，关闭 Game Simulator 会导致 ASSERT（断言）。
- **WG-31093** 已解决：在 Harmonizer 和 TimeStretch 中可能导致内存损坏。
- **WG-31100** 已解决：（Xbox One）如果音高改变，在匮乏出现后暂停声音会导致 XMA 断言。
- **WG-31106** 已解决：插件 DLL 错误不显示。
- **WG-31161** 已解决：无用错误日志 *Audiokinetic Wwise：无法创建 XAudio2 主声部*。
- **WG-31230** 已解决：当使用带有多层子持续容器的触发率容器时，可能崩溃。
- **WG-31268** 已解决：当将一个带有匿名声道设置的声音连接到标准声道设置的总线时， `CAkSpeakerPan::GetSpeakerVolumes()` 内发生挂起。
- **WG-31349** 已解决：当原始素材需要缩混时，可能导致 SoundBank 完好性问题。
- **WG-31428** 已解决：当 SoundBank Settings 中的 *Use SoundBank names（使用 SoundBank名称）* 选项被取消时，无法生成更多的 SoundBank。
- **WG-31435** 已解决：大型工程的载入‘完成’阶段花的时间太长（自从 2016.1.2 版本开始)。
- **WG-31440** 已解决：（Windows）即使其他的中间件已经载入了 XAudio2.7，但在 Windows 10 上使用它还是可能导致问题。
- **WG-31476** 已解决：在 Wwise Wave Viewer 工具中的 Playback 损坏。
- **WG-31524** 已解决：在 `CAkTransitionManager::ProcessTransitionsList` 中可能发生崩溃。
- **WG-31532** 已解决：在工程载入时，混合音轨会重置所有的参数值范围区域。
- **WG-31537** 已解决：（Xbox One）在回收旧声部时，未知 XMA 声部状态导致 XMA 断言。
- **WG-31542** 已解决：当回放某些带有交叉淡变循环区域的原始文件时，设计工具崩溃。
- **WG-31617** 已解决：在 `AK::DSP::ApplyGainRamp` 内发生罕见崩溃。
- **WG-31685** 已解决：（Android）在设备上设置最大采样率为 48 kHz 会错误报告成更高的数值。
- **WG-31766** 已解决：（Xbox One）当采样率不是 48 kHz 时，带有 XMA的 `GetSourcePlayPosition()` 返回的位置无效。
- **WG-31768** 已解决：当缓冲仍在进行时，`AK::SoundEngine::GetBufferStatusForPinnedEvent()` 和 `AK::IAkStreamMgr::GetBufferStatusForPinnedFile()` 返回的值可能会显示 100。

# 其它更改

- **WG-31559** 在 Capture Log 中为故障添加了一条发送性能分析数据的信息。
