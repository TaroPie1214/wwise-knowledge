# 性能改进

- **WG-55919** 将 rpmalloc 更新到了 1.4.3 版本。此次更新包含对长时间运行的进程的总计预留内存大小的改进。
- **WG-56533** 提升了由全局作用域 RTPC 驱动的 Switch Container 的性能。

# 其他改进

- **WG-56430** 为改善一般工程的内容显示效果，调整了 Performance Monitor 中各个数值的默认最小和最大范围。
- **WG-57325** 取消了对 Unreal Engine 4.25 的支持。现在支持的最低版本为 4.26。

# 漏洞修复

- **WG-48627** 已修复：在重命名处于版本控制系统之下的工程中的 Work Unit 时可能会发生崩溃。
- **WG-53891** 已修复：在构建主题时发生编译错误。
- **WG-55883** 已修复：在使用 File Manager 移动源文件后媒体转码出现错误。
- **WG-55889** 已修复：在实施性能分析时切换布局可能会引发崩溃。
- **WG-55922** 已修复：WwiseConsole `generate-soundbank` 操作不支持 `–output`、`–source-file` 或 `–source-by-platform` 选项。
- **WG-56184** 已修复：检视某些对象会导致 Music Track 执行 Change End Position 操作。
- **WG-56307** 已修复：在使用颜色选择器时可能会发生崩溃。
- **WG-56377** 已修复：使用了 32 位扇区 `StreamMgrModule.h::AkFileDesc::uSector` 保存文件包条目。
- **WG-56423** 已修复：(Spatial Audio) Portal 返回 100% 的 Aperture 值，导致在听者处于 Portal 正中位置时对 Spread 值没有任何影响。
- **WG-56487** 已修复：在对象之间切换或添加过渡时，Music Transition 列表展开/折叠设置和滚动位置丢失。
- **WG-56508** 已修复：在播放零延迟声音时，Opus 播放流出现匮乏。
- **WG-56625** 已修复：在从 Suspend 过渡到 Active 状态时，`AK::SoundEngine::WakeupFromSuspend` 调用不起作用。
- **WG-56729** 已修复：在处于 Continuous 模式的 Blend Container 的 Limit Instances 属性应用有 RTPC 时，在容器播放过程中卸载 SoundBank 会引发崩溃。
- **WG-56879** 已修复：(wp.py) 对于采用不同配置的 Authoring 条目，`generate-bundle` 操作赋予了 `"name"` 字段完全相同的值。
- **WG-56973** 已修复：在离线渲染期间终止 SoundEngine 时死机。
- **WG-57026** 已修复：`AK::IAkMixerInputContext::GetPannerPosition` 返回了错误的 Z 值。
- **WG-57069** 已修复：wp.py 基架中 tasks.json 的格式有误。
- **WG-57253** 已修复：Advanced Profiler 的 Streams 选项卡中显示了 ID 而非流名称。

# 社区报告的漏洞修复

- **WG-54889** 已修复：WAAPI 无法为 WebSocket 分片提供恰当的支持（无法在 Python 中执行负载较大的 RPC 调用）。
- **WG-55303** 已修复：对于包含高度声道的总线，Listener-Relative Routing 的性能不佳。
- **WG-55786** 已修复：在使用 `AK::SoundEngine::RegisterBusMeteringCallback` 时发生崩溃。
- **WG-55808** 已修复：在声音播放过程中更改作用于 Modulator LFO 属性的 Game Parameter 时发生崩溃。
- **WG-56351** 已修复：在驱动 Blend Container 的 Modulator 具有受 RTPC 控制的属性时，`CAkRTPCMgr::AkRTPCSubscription::PushUpdate` 引发崩溃。
- **WG-56429** 已修复：(Spatial Audio) 在听者和声源之间存在一个或多个 Room 时，没有将听者所在 Room 的透射损失值应用于房间底噪和混响的直接透射路径。
- **WG-56562** 已修复：在 SoundBank 生成过程中执行生成前和生成后操作时，WAAPI 调用的处理速度很慢。
- **WG-56645** 已修复：在从之前的解码错误恢复时重置 XMA 硬件导致 GDK 发生崩溃。
- **WG-56664** 已修复：在参数由 MIDI Velocity 驱动时触发断言并引发崩溃。
- **WG-56680** 已修复：在加载大量 Switch Container 且 RTPC 驱动的 Switch 的值发生变化时性能出现下降。
- **WG-56717** 已修复：`CAkParameterTarget::SetRootNode` 函数可能会触发错误的断言。
- **WG-56794** 已修复：在进入 Portal 时发生崩溃。
- **WG-56865** 已修复：在对 Audio Device 效果器（如 Mastering Suite）实施性能分析时发生崩溃。
- **WG-57117** 已修复：(Spatial Audio) 在使用 Portal 连通两个 Room 时，有时会报告 Game Object 处在错误的 Room。
- **WG-57204** 已修复：在使用 XMA 的情况下通过内存指针而非使用 SoundBank ID 或文件名卸载 SoundBank 时发生崩溃。
