社区插件开发流程

本主题列出了在开发者为 Wwise 构建社区插件时的主要建议、要求和最佳做法。请查看列表并参阅链接的文档以确保所开发的插件符合 Audiokinetic 的相关要求。

- [关键技术要求](plugin_requirements.html). 查看社区插件相关技术要求（即所开发插件能做什么、不能做什么）。
- [音频插件](effectplugin.html). 本节及其所有子主题与创建音频插件的开发者相关。以下主题和小节尤其有用：
  - [使用开发工具](effectplugin_tools.html). 本节阐释了如何使用 wp.py（使用 Python 编写的工具）来创建插件并根据需要加以构建。
  - [编写音频插件的设计工具部分](effectpluginwwise.html). 本节阐释了如何编写音频插件的设计工具部分。
  - [编写音频插件的声音引擎部分](effectpluginsoundengine.html). 本节阐释了如何编写音频插件的声音引擎部分。
  - [示例：开发低通滤波器插件](code_effectplugin_runthrough.html). 本主题举例展示了如何为音频插件（本例中为低通滤波器）编写代码。
  - [测试音频插件](plugin_tests.html). 本主题列出了一系列确保插件与 Wwise 兼容的准则及可供开发者运行的测试。
  - [Wwise 源插件和效果器插件故障排除指南](plugin_troubleshooting.html). 本主题阐释了如何解决插件开发者经常遇到的各种问题。
- [版本控制插件](source_control.html). 本节及子主题阐释了如何创建插件以将 Wwise 与版本控制系统（如 Perforce 或 Subversion）集成。
- [Audiokinetic Launcher 的插件打包格式](plugin_packaging.html). 本主题阐述了通过 Audiokinetic Launcher 及插件商店分发插件所需的打包格式。
